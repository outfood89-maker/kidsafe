# [Claude Code 작업 지시서] 화남(😡) 체크인 대화 수정

> 문제: "그림 그리기 하면서 화났네" — 아이는 '화남'과 '그림그리기'를 **따로** 골랐는데, 키디가 "그림 때문에 화났다"는 **없는 인과를 지어냄.** 사실 왜곡이라 반드시 고친다.
> 너가 앞서 제안한 3가지 방향 맞음. 아래대로 진행하되, 우선순위와 추가 확인 2건을 지킬 것.

---

## 1. [주력] 기분 정보를 하루/볼것 받아주기에 넘기지 않기

하루·볼것 Haiku 받아주기의 `priorAnswers`에서 **기분(mood_today)을 제외**하고 한 일·볼 것만 넘긴다.
→ 키디가 기분을 모르면 "화 때문에 그림" 같은 인과를 **만들 재료 자체가 없어진다.** 이게 진짜 해결책.

## 2. [보조] 프롬프트에 인과 결합 금지 명시

`_react_system`에 추가: **"기분과 한 일/볼 것을 원인-결과로 엮지 마라. 아이가 화났어도 그건 그림 때문이 아니다. 한 일과 볼 것에만 반응하고, 기분은 언급하지 마라."**
→ 단, 이건 1번의 보조다. 1번(재료 제거)이 주 방어, 2번은 혹시 기분 정보가 새도 막는 이중 안전망. **1번 없이 2번만 하면 안 됨.**

## 3. [포즈] 화남 세션은 전 단계 포즈를 차분하게 통일

화남(😡)이면 하루/볼것 받아주기 포즈도 `jump` 대신 차분하게(`reactionPose`가 mood 참조).

**+ 추가 확인 (중요):** 화남 세션의 **모든 단계** 포즈가 일관되게 차분한지 점검.
- 기분 받아주기(비트2): sad/차분 — 지금 실제로 그런가? (보고는 sad였지만 화면은 점프였음 → 확인)
- 후속질문·마무리(비트3·4): 포즈가 뭐야?
- 하루/볼것 받아주기(비트5): 이번에 jump→차분으로 고치는 곳
→ 화남 세션 동안 키디가 **한 번이라도 점프/신난 포즈가 나오면 안 됨.** 단계마다 포즈가 따로 놀지 않게, mood가 부정이면 전 단계 차분 포즈로 통일.

---

## 4. [확인 후 수정] "동화 보면서 기분 풀어봐" 출처

이미지의 마무리 대사 "그림 그리다가 화났는데 동화 보면서 기분 풀어봐" 처리:

- 이게 **Haiku 생성**이면 → 1·2번으로 자동 해결됨(인과 결합 제거 + 프롬프트 금지).
- 이게 **로컬 고정 템플릿**(kiddyLines.js)이면 → 1·2번으로 안 잡힘. **그 문자열을 직접 고쳐야 함.**

먼저 출처를 확인하고, 로컬 템플릿이면 아래 원칙으로 교체:
- "기분 풀어봐"는 **위로 원칙 위반**임 — 화난 감정을 빨리 치우라는 말. 우리 원칙은 "공감 + 키디가 곁에"만, 감정을 덮거나 해결하려 들지 않음.
- 교체 예: "화났구나. 키디가 옆에 있어 줄게 💚" (인과 안 엮고, 감정 안 덮고, 곁에만)

---

## 수정 후 기대 결과 (화남 세션)

- 받아주기(비트2): 기분에만 반응 → "화났구나… 속상했지 💚" (그림 얘기 안 꺼냄)
- 하루(비트5): 활동에만 반응 → "그림 그렸구나!" (화남과 안 엮음)
- 마무리: "화났구나. 키디가 옆에 있어 줄게 💚" (감정 안 덮음)
- 포즈: 세션 내내 차분 (점프 0)

## 검증

- 😡 + 한 일(그림그리기)로 체크인 한 바퀴 돌려서:
  - 키디가 "그림 때문에 화났다"는 식으로 **인과를 엮지 않는지**
  - "기분 풀어봐" "동화 보면 괜찮아" 같은 **감정 덮는 위로가 안 나오는지**
  - 모든 화면에서 키디가 **점프 안 하고 차분한지**
- 막히면 멈추고 보고.
