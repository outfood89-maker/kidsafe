# [작업지시서 AD-7] 부모 '둘러보기 모드' — 예시 가족 핸즈온 투어

*작성: 컨트롤 타워 2026-07-06 · 근거: 팀장 AD-7 발부(오너 설계: 가상 세팅 핸즈온 투어) · **착수는 AD-6 커밋 후**(ParentDashboard·책장 컴포넌트 의존)*
*7/8 리허설 개시 시 일시정지. 브랜치 격리·main 무접촉·7/14 전 머지 금지 불변.*

## §0. 전제 (오너 설계 원칙 — 전부 필수)

1. **정적 시드(LLM 0)**: 예시 가족 데이터는 프론트 내장 JSON(`client/src/utils/tourSeed.js`) — **클라이언트 메모리에만 주입.** 서버·DB 호출 0, localStorage(diaryStore 포함) 기록 0, **종료 시 흔적 0.**
2. **실제 컴포넌트로 렌더** — 모형 스크린샷·가짜 화면 금지. 실제 대시보드 섹션에 시드 데이터를 흘려 렌더 + 코치마크 오버레이로 유도.
3. **정직 장치(필수)**: 투어 중 상시 상단 배너 `미리보기 — 예시 가족의 화면이에요.` (verbatim). 이 배너가 없는 프레임이 한 순간도 없어야 함.
4. 시드 가족 이름은 **데모 계정(하늘·바다)과 겹치지 않게**(심사 시드와 혼동 방지) — 작업자 픽, 아동 노출 아님(부모 화면 전용).
5. ⚠️ **정거장 안내 카피는 팀장 게이트 대상** — 이 브리프에는 스탬프가 없다. **구현 = 구조·시드·오버레이 뼈대까지. 안내 문구는 상수 자리(placeholder 키)만 만들고, '필요 문구 목록'(정거장 4 + 진입 제안 + 재진입 라벨)을 산출해 보고** → 팀장 일괄 스탬프 후 반영 커밋. 임시 문구 하드코딩 금지.

## §1. 시드 (`tourSeed.js`)

- 구성: 예시 아이 1명 프로필(이름·나이 6~7), 주간 리포트(편지·감정 흐름 — 💜 배너가 발화하는 형태의 체크인 분포), 대화의 씨앗 1~2개, 그림일기 엔트리 3~4건(2개월 분산 — 월 그리드가 보이게, 1건은 도장 없음=체험용), 자녀 설정 표시값.
- 데이터 형태는 **각 섹션이 실제로 소비하는 응답 스키마와 동일**하게(리포트·씨앗·책장 렌더가 분기 없이 그대로 먹도록). 시드 내용 문안(편지 예문 등)도 부모 노출 텍스트이므로 **문구 목록에 포함해 팀장 게이트**.

## §2. 투어 모드 (ParentDashboard)

- `tourMode` state + 시드 주입: 투어 중 각 정거장 섹션의 데이터 소스를 시드로 대체(**API 호출 자체를 건너뜀** — fetch 가드). 실제 데이터와 혼합 렌더 금지(전면 시드).
- **정거장 4 (순서 고정)**: ① 키디의 한 주(편지+💜ばnner) → ② 대화의 씨앗 → ③ 가족 책장 — **시드 일기에 도장 찍어보기 체험**(AD-6 도장 UI 그대로, 단 **메모리 상태에만 반영** — diaryStore 무기록) → ④ 자녀 설정.
- 코치마크 오버레이: 반투명 배경 + 대상 섹션 하이라이트 + 안내 카피 자리 + [이전/다음/그만보기]. 정거장 이동 시 mainTab도 함께 전환(실 화면 그대로).
- **종료 CTA**: 마지막 정거장 후 `이제 우리 아이 프로필을 만들어볼까요?` (verbatim — 팀장 지시문 기확정) → 자녀 설정(프로필 생성) 유도. 종료·중도 이탈 시 tourMode 해제 + 시드 전부 폐기(메모리) + 실 데이터 재조회.
- **진입**: 부모 대시보드 **최초 진입 시 제안**(localStorage `parentTourSeen` 플래그 — 부모 측이라 허용, 이 플래그 하나만 기록) / **재진입**: 설정(자녀 설정 탭)에 "둘러보기 다시 하기" 진입점(라벨도 게이트 목록에).
- 아이 쪽 튜토리얼 없음(설계 확정) — 키즈 표면 무접촉.
