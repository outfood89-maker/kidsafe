# [Claude Code 작업 지시서] 데모 전 폴리시 패스 — ParentDashboard

> C(부모 리포트) 화면이 잘 떴고, 윤리선·감정 설계도 살아 있음(확인됨). 데모 전 **다듬기 5건**을 우선순위대로 친다.
> **백엔드 데이터 로직 자체는 신뢰하되, 표시·정합성·톤만 손본다.** 막히면 멈추고 보고.

우선순위: **①조사 → ②안전 모순 → ③영상 개수 → ④메뉴 가림 → ⑤감정 색** (+선택 ⑥ 편지 등장)

---

## ① 조사 처리 중앙화 (최우선 — 가장 눈에 띔)

**증상:** 섹션 제목이 "해인**가** 나누고 싶어한 것" → 문법 오류. '해인'은 받침이 있어 "해인**이가**"가 맞다.
지금 `{이름}가`처럼 조사가 하드코딩돼, **받침 있는 이름(한국 이름 대부분)에서 전부 틀린다.**
(반면 "해인**을**", "해인**만의**"는 맞게 나옴 → 처리가 일관되지 않음.)

**해결:** 이름 받침 유무로 조사를 골라주는 **헬퍼 하나로 중앙화**하고, 화면의 모든 `{이름}+조사`를 이걸로 교체.

```js
// utils/josa.js
const hasBatchim = (word) => {
  if (!word) return false;
  const code = word.charCodeAt(word.length - 1);
  if (code < 0xAC00 || code > 0xD7A3) return false; // 한글 음절이 아니면
  return (code - 0xAC00) % 28 !== 0;                 // 종성 인덱스 0 = 받침 없음
};

// 받침 있으면 first, 없으면 second
export const josa = (word, withB, withoutB) =>
  `${word}${hasBatchim(word) ? withB : withoutB}`;

// 사용 예
josa(name, "이가", "가")   // 해인이가 / 지우가
josa(name, "을", "를")     // 해인을 / 지우를
josa(name, "은", "는")     // 해인은 / 지우는
josa(name, "이", "가")     // 해인이 / 지우가  ("정말 멋있어요" 앞 등)
josa(name, "만의", "만의")  // 받침 무관
```

**적용 위치(찾아서 전부):** 섹션 제목 "OO이가 나누고 싶어한 것", 닫는 말, 비밀 줄, 그리고 `kiddyMessage`처럼 **백엔드가 만든 문장 안의 이름 조사**는 백엔드 소관이니 그건 따로 보고(프론트에서 못 고침).

## ② 영상 안전 모순 점검 (데이터 정합성)

**증상:** "🌿 대부분 안전했어요" 리스트에 "원숭이가 한반도에서는 절대 살 수 없는 이유**(0%...?)**" 같은 항목이 섞임.
그 `(0%...?)`가 **(a) 실제 유튜브 제목 일부**인지, **(b) 안전 점수(0%)가 제목 표시에 새어든 것**인지 확인.

**해결:**
- (b)라면: 제목 렌더에서 점수 텍스트 분리(버그). 그리고 **안전 점수 0%면 '위험'인데 '대부분 안전' 집계에 든 것 자체가 모순** → insights 집계에서 안전 색 기준(90+ 초록 / 70–89 노랑 / ≤69 빨강) 반영 확인.
- (a)라면: 표시는 OK지만, **저점수 영상이 '안전' 요약에 포함되는지**는 여전히 점검. "대부분 안전" 문구는 실제 안전 비율에 근거해야 함.
- 모호하면 멈추고 보고(실제 insights 응답의 점수 필드를 같이 확인).

## ③ 영상 개수 ↔ 리스트 불일치

**증상:** "영상 7개"라는데 아래 나열은 12개 이상.
**해결:** 카운트 소스와 리스트 소스를 일치시킨다. 같은 기간·프로필 집계인지 확인. 리스트를 상위 N개로 자르는 경우 "영상 N개"가 **전체 수**인지 **표시 수**인지 라벨을 명확히("영상 12개" 또는 "최근 7개 표시").

---

## ④ 플로팅 '메뉴' 버튼이 콘텐츠 가림

**증상:** 민트색 '메뉴' 버튼이 스크롤 시 감정 흐름 '화' 타일·이야기 카드를 덮음.
**해결:** z-index/위치 조정. 콘텐츠와 안 겹치게 하거나, 스크롤 시 카드 내용을 가리지 않는 위치로. (헤더에 고정이면 카드 영역과 분리.)

---

## ⑤ 감정에 경고색 빼기 (설계 폴리시 — 톤 일관성)

**원칙:** *감정에는 경고색(빨강)을 쓰지 않는다.* 슬픔을 차분한 블루그레이로 둔 것과 같은 이유 — 빨강이면 부모가 '문제'로 읽는다. 리포트는 안심이 목적.

**해결:** "화남 N" 카운트 칩 + '화' 요일 타일의 **붉은 톤을 차분한 주황/테라코타로 톤다운.** 빨강(`#C84B47` 류) 금지. 슬픔(블루그레이)과 화남(차분한 주황)이 나란히 '경고 아닌 감정'으로 보이게.
※ 단, ②의 **영상 안전 '위험'(빨강)** 은 그대로 둔다 — 그건 진짜 경고가 맞음. **감정 빨강만 제거.**

---

## ⑥ (선택) 편지 등장 애니메이션

데모 클라이맥스가 "부모가 리포트를 **여는 순간**". 키디 편지(히어로 카드)가 로드 시 **0.3–0.4초 fade-in + 살짝 위로 떠오름**. 폰트·레이아웃 불변, 애니메이션만. `prefers-reduced-motion` 존중.
