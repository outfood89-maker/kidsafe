## §2. 진입 — 체크인 공유 선택 직후

- 위치: DailyCheckin의 공유 선택(`chooseShare`) 처리 완료 후, 기존 마무리(reward/closing)와 자연 연결되는 지점 — 기존 체크인 흐름·비밀 약속 멘트를 **한 글자도 건드리지 말 것** (체크인 현행 유지 = 상위 판정).
- 제안 카피(§7-①): 기본 vs **😢😡 변주** — 그날 체크인 mood가 슬픔/화남이면 변주 사용(강요 금지). "안 할래/쉴래" → §7-① 응답 멘트 후 종료, **아무것도 저장하지 않음, 재촉·알림 없음.**
- **R5 진입 조건:** 일기 제안은 **당일 체크인 완료 시에만** (뼈대 재료 보장 — 체크인 스킵한 날은 제안 자체가 안 뜸).
- **R8 거절 빈도 하향(강요 금지의 구현):** "안 할래" **3일 연속 → 격일 제안 / 7일 연속 → 주 2회**. 아이가 가족 책장에 자발 방문하면 기본 빈도로 리셋. 거절 카운트·마지막 제안일은 프로필별 localStorage(`diaryStore.js`에 함께).
- 두 채널 구분(설계 v2 §2·§7): 그림일기 흐름은 체크인의 비밀 약속과 별개 채널 — 체크인 비공개 답변이 일기에 흘러들면 안 됨. **일기 재료로 쓰는 체크인 데이터 = mood(기분)·what_did_today(한 일) 2개뿐**, 후속답·비공개 텍스트 사용 금지.

## §3. 질문 플로우 — 3층 (설계 v2 §3)

**1층 · 고정 뼈대 (신규 질문 1개뿐):** 날짜·요일 자동 / **날씨 칩** ☀️맑음 ☁️구름 🌧️비 ⛄눈 + **"모르겠어"**(R3 — 선택 시 날씨 문장 생략) (+말하기) / 기분·한 일 = **그날 체크인 재사용, 재질문 금지.**

**2층 · 오늘의 질문 1개:** 회전 풀 8종(§7-②) 중 랜덤, **최근 3일 사용 질문 중복 회피**(프로필별 localStorage에 최근 사용 qid 기록). 풀 필터 3중:
- **R1 감정 태그:** 맑은 날 전용 2종("제일 재미있었던 순간"·"뭘 제일 잘했어") / 전천후 6종. **체크인 😢😡인 날은 전천후 풀에서만 회전** (흐린 날 전용 질문은 없음 — 감정 탐색은 체크인의 몫, 일기는 추억 채널).
- 연령: 질문 6("무슨 소리")은 **6세 이상 전용.**
- **R2 "없었어" 출구 상시:** 모든 회전 질문 칩 끝에 `음… 오늘은 없었어!` 고정. 선택 시 해당 문장은 일기에서 **생략**(뼈대 3문장으로 일기 성립), 키디 고정 반응: **"그런 날도 있지!"** (§7-⑧)

답 = 칩(질문별 칩 셋은 §7-② 표) + 말하기.

**3층 · 그림 참여 질문 (매일 고정, 마지막):** "오늘 그림에 꼭 넣고 싶은 거, 하나만 골라줘!" — **칩은 그날의 답에서 자동 생성**(한 일·2층 답·날씨 등에서 명사 후보) + 말하기. 답은 문장에 넣지 않고 `child_pick` 필드로 저장만 (고도화의 그림 프롬프트 재료).

**연령 사다리 (v0 범위):** 4~5세 = 칩만(말하기 버튼 숨김) / 6세+ = 칩+말하기. (8세+ 자유 문장은 범위 밖.)

**말하기 안전 (필수):** 모든 자유 발화 transcript는 칩 답과 달리 **`safetyLexicon.screen_text` 경유** — ① high 판정: 고정 응답(A/B) calm 표시·음성 + `createCareSignal(profileId, "high")`(fire-and-forget) + **해당 텍스트는 일기 어디에도 유입 금지**(칩 재선택으로 복귀) ② soft: 고정 응답 C만, 신호 없음, 텍스트 미유입 ③ 정상: 답으로 사용. KiddyRoom·ChatWidget과 동일 계보.

## §4. 조립 — `diaryAssembler.js` (LLM 0, 코드 결정적)

입력: `{ date, weather, mood, didToday, rotatingQ: {qid, answer}, childPick }` → 출력: 문장 배열(5~6문장). 조사 처리는 기존 josa 유틸 사용(○○+랑/이랑, 을/를, 이/가 등 — 하드코딩 금지).

| 슬롯 | 템플릿 | 상태 |
|---|---|---|
| 날씨 | ☀️ "오늘 날씨는 맑았어요." 등 4종 | **스탬프 대기 (§7-③ 후보)** |
| 기분 | 체크인 5종 mood → 문장 | **스탬프 대기 (§7-③ 후보)** |
| 한 일 | "오늘은 ○○을 했어요." | **스탬프 대기 (§7-③ 후보)** |
| 오늘의 질문 | 설계 v2 §3 표 조립 문장 8종 ("○○랑 같이였어요" 등) | 확정 (설계 v2 verbatim) |
| 마무리 | 좋은 날 "오늘도 참 좋은 하루였어요" / 흐린 날(😢😡) "내일은 더 웃을 수 있으면 좋겠어요" | 확정 (설계 v2 verbatim) |

- **생략 규칙:** 날씨 "모르겠어"(R3)·회전 질문 "없었어"(R2) → 해당 문장 생략. 뼈대 3문장(날짜는 표기, 기분·한 일)만으로도 일기 성립.
- **R6 인용 프레임 — 말하기 답 전용 (문장 파손 방지, 필수):** 자유 발화 원문에는 **어미·조사를 직접 결합하지 말 것.** 인용 조립만: `{질문 리드}, "{원문}" 하고 이야기했어요.` — 질문 리드는 해당 질문 조립 문장의 ○○ 앞 구간을 기계적으로 사용(예: 맛있었던 것 → `제일 맛있었던 건, "딸기 아이스크림이랑 김밥" 하고 이야기했어요.`). 따옴표 안 내용과 무관하게 문법이 성립한다. **칩 답만 조사 결합 템플릿 사용.** (P 스크리닝 경유는 §3 그대로.)
- **R4 마무리 풀 구조:** 마무리 문장은 풀 배열 + 최근 2회 사용 기록(중복 회피 로테이션)으로 구현 — v0 풀은 확정 2종(좋은 날/흐린 날)뿐이라 사실상 기분 선택이지만, 구조는 풀로(고도화 확장 대비).
- 재료 밖 창작 0 — 답에 없는 단어가 문장에 나타나면 버그다.
- 단위 테스트: 조립 함수는 순수 함수로 — 기분 5케이스 + 생략 2케이스(R2·R3) + 인용 프레임 1케이스 스냅샷 검증 스크립트 포함.

## §5. 낭독 → 간직

1. 조립 완료 → 낭독 전 멘트(§7-④) → **TTS로 완성 일기 전문 낭독**(`voice.enqueue` — 문장별 이어붙임), 화면엔 일기 텍스트 표시(크림 톤 '종이' 카드 허용 — 설계 v2 §6-5, 작품 지면 예외).
2. **그림 자리 플레이스홀더:** 이미지 없음 — 자리 박스 + 플레이스홀더 카피(§7-⑤, 스탬프 대상). 그림 생성 시도·API 호출 코드 금지.
3. 마무리 선택(§7-⑥): v0은 자랑 기능이 없으므로 **전 기분 공통 '간직하기' 계열** 사용 — ⚠️ 대본 v1의 기본 마무리("자랑하러 갈까?")와 다르므로 **v0 한정 대체를 스탬프 요청에 포함**(§7 참조). "간직하기" → 저장 + 완료 멘트 / "안 할래" → 저장 0으로 종료.

## §6. 저장·가족 책장 — 서버·DB 무접촉 (오너 SQL 0)

**v0 저장 = localStorage** (`diaryStore.js`로 추상화 — 고도화 머지 시 Supabase 테이블+라우터로 교체할 것을 주석 명시):
- 키: `diary_v0_<profileId>` / 엔트리: `{ id, date, sentences[], moodEmoji, childPick, keptAt }`
- **불변식 (설계 v2 §7 — 검증 대상):** ① '간직하기' 선택분만 저장 — 중간 이탈·"안 할래"는 흔적 0 ② 음성 원문(transcript) 미저장 — 저장물은 조립 완성문+선택값뿐 ③ 위기 텍스트 유입 없음 ④ **찢어버리기 = 페이지 단위 즉시 완전 삭제**(확인 대화 §7-⑦ → 삭제 → 복구 불가), 부모 삭제 기능 없음 ⑤ 간직·찢기에 배지·보상·알림 연결 금지.

**가족 책장 (`/family-shelf`):**
- 진입: 키디의 방 안 **상자/책장 오브젝트**(작은 버튼 — 브랜치 전용 ADD, KiddyRoom 기존 로직 무접촉) + 일기 간직 직후 "보러 가기".
- 열람: **월별 '한 권' 묶음** — 월 헤더(그 달 최다 기분 이모지 표지) + 날짜순 페이지 목록 → 페이지 상세(일기 전문 + 그림 플레이스홀더 + 찢어버리기 버튼).
- 라우트는 App.jsx에 ProtectedRoute로 추가 (브랜치 전용).
