# [Claude Code 작업 지시서] kiddyMessage 이름 조사 토큰화

> 부모 리포트의 키디 한마디(`kiddyMessage`, Haiku 생성)에서 아이 이름 뒤 조사를 정확히 처리한다.
> 원칙: **"사실은 코드가, 분위기만 LLM이."** 톤은 Haiku, 조사(문법 규칙)는 프론트 `josa.js`가 보장.
> LLM에 "조사 맞춰줘"라고 시키지 않는다(확률적 → 가끔 틀림). 이름을 토큰으로 빼고 프론트가 치환.
>
> ※ 프론트엔 이미 `utils/josa.js`(받침 판별 헬퍼)가 있음 — 그걸 재사용.

작업: **백엔드 1 + 프론트 1.** 막히면 멈추고 보고.

---

## 1. 백엔드 (`reports.py`) — 이름을 토큰으로 출력

kiddyMessage 생성 시, Haiku에게 **실제 이름을 주지 말고** 플레이스홀더 `{{CHILD}}`를 쓰게 한다.

- 프롬프트에서 아이 이름이 들어가던 자리를 `{{CHILD}}`로 교체.
- 시스템/지시 프롬프트에 명시: **"아이를 부를 때는 `{{CHILD}}` 라고만 써라. 조사(이/가, 을/를)는 평소대로 뒤에 붙여라. 단 실제 이름은 절대 직접 쓰지 마라. 예: `{{CHILD}}이 이번 주에도 마음을 들려줬어요`, `{{CHILD}}을 안아주세요`."**
- 백엔드는 이 문자열을 **치환하지 않고 그대로** 응답에 실어 보낸다. (치환은 프론트 몫 — 조사 규칙이 프론트에 있음)

**검증:** 응답 `kiddyMessage`에 `{{CHILD}}`가 들어 있는지 확인. 실제 이름이 박혀 나오면 프롬프트가 안 먹은 것 → 프롬프트 강화 후 재시도.

---

## 2. 프론트 — `{{CHILD}}`를 josa로 치환

`kiddyMessage`를 화면에 그리기 직전, `{{CHILD}}` + 바로 뒤 조사를 잡아 이름+올바른 조사로 교정한다. 기존 `josa.js` 재사용.

```js
import { josa } from "./josa";

export function renderKiddyMessage(message, name) {
  const pairs = [
    [/\{\{CHILD\}\}이가|\{\{CHILD\}\}가/g, () => josa(name, "이가", "가")],
    [/\{\{CHILD\}\}이\b/g,              () => josa(name, "이", "가")],   // 주격 이/가
    [/\{\{CHILD\}\}은\b|\{\{CHILD\}\}는/g, () => josa(name, "은", "는")],
    [/\{\{CHILD\}\}을\b|\{\{CHILD\}\}를/g, () => josa(name, "을", "를")],
    [/\{\{CHILD\}\}와|\{\{CHILD\}\}과/g,   () => josa(name, "과", "와")],
    [/\{\{CHILD\}\}/g,                   () => name],   // 조사 없이 단독으로 남은 경우
  ];
  let out = message;
  for (const [re, fn] of pairs) out = out.replace(re, fn);
  return out;
}
```

- `kiddyMessage` 렌더하는 곳을 `renderKiddyMessage(msg, profileName)`로 감싼다.
- 화면에 `{{CHILD}}`가 **그대로 노출되면 실패**(치환 누락 케이스) → 어느 조사가 빠졌는지 보고.

---

## 3. 검증 (받침 있는 이름으로)

- **해인(받침 ○):** "해인이 들려줬어요", "해인을 안아주세요" 등 전부 정상인지.
- **지우(받침 ✕):** "지우가 들려줬어요", "지우를 안아주세요" 정상인지.
- 키디 편지 본문에 `{{CHILD}}`가 안 새고, 조사가 다 맞는지.

## 막히면 멈추고 보고

- Haiku가 `{{CHILD}}` 지시를 무시하고 실제 이름을 박을 때 → 프롬프트 강화 필요, 보고.
- josa로 못 거르는 희귀 조사(에게/한테 등)가 나올 때 → 케이스 추가 위해 보고.

*처리 후, 해인으로 띄운 부모 리포트의 키디 편지에서 조사가 다 맞는지 + 토큰이 화면에 안 새는지 확인.*
