# [Claude Code 작업 지시서] J — 영상 후 키디 한마디 (학습 넛지)

> 영상이 끝나면 키디가 **짧고 임팩트 있는 한마디**를 준다: 방금 본 주제의 재미 팩트 1개 + 영어 단어("따라해봐!"). **TTS로 읽어준다.**
> 예: "{이름}아! 사자 영상 재밌었어? 사자는 가족끼리 무리 지어 살아. 아 참, 사자는 영어로 'Lion'이야. 따라해봐 — 라이언!"
>
> 🚨 **최상위 안전선 (절대):** 사실은 **검증된 팩트 뱅크(코드)에서만** 나온다. **LLM(Haiku)이 사실을 생성하지 않는다.** 이 프로젝트의 뼈아픈 규칙("사실은 코드가, 분위기만 LLM이")이 여기 그대로 적용된다. 키디가 틀린 지식을 4~7세에게 주면 신뢰가 무너진다.
> **막히면 임의로 우회하지 말고 멈추고 보고.**

## 0. 확정 사실 (실제 코드 검증됨)

- **완료 화면이 이미 있다 → 새 오버레이가 아니라 기존 자리 채우기.** [VideoPlayer.jsx](../client/src/components/VideoPlayer.jsx):
  - 플레이어 = `react-youtube`, **`onEnd`(영상 끝) 작동 중** ([:534](../client/src/components/VideoPlayer.jsx#L534)) → `videoEnded=true`. 새 트리거 불필요.
  - **(A) 비연속 종료 완료화면** ([:459-490](../client/src/components/VideoPlayer.jsx#L459)): 키디 success + "다 봤어! 재미있었어? 🎉" + "소감 말하기"(챗봇) + "목록으로". ← 여기 문구를 팁으로 교체.
  - **(B) 연속재생 검수 대기 카드** ([:345-454](../client/src/components/VideoPlayer.jsx#L345)): 영상 끝 → preview(3s) → analyzing(검수) → scored(5s) → 자동재생. ← 이 구간에 키디 팁 말풍선 추가.
- **주제 신호 = `video.title`.** (VideoPlayer에 `video.title` 있음. 아이가 고른 watchKeyword는 여기 안 넘어오므로 title로 판단.)
- **음성 = `useKiddyVoice` 재사용** ([hooks/useKiddyVoice.js](../client/src/hooks/useKiddyVoice.js)). 영어 발음은 브리프 I2의 글↔말 분리 원리 그대로(화면=영어, 음성=한글표기).

## 1. 🚨 안전 원칙 (설계의 심장)

- **사실은 검증된 팩트 뱅크에서만.** LLM 사실생성 금지. 팩트 뱅크의 문장은 **사람이 검증한 것만**(Freddie 소관).
- **제목 매칭이 조금 느슨해도 안전한 이유:** 팩트를 '그 대상에 대해 일반적으로 참인 한 문장'으로만 쓴다("사자는 무리 지어 살아"). 그러면 "사자와 토끼" 같은 영상에서 사자로 매칭돼도 **틀린 지식은 안 나간다**(사자 사실은 여전히 참).
- **매칭 실패 시 폴백은 사실을 지어내지 않는다** → 기존 "다 봤어! 재미있었어?" 무해한 한마디로.
- 4~7세: **한 문장, 쉬운 말, 무섭거나 복잡한 사실 금지.**

## 2. 데이터 — 팩트 뱅크 (신규 파일)

신규 파일 예: `client/src/utils/kiddyTips.js`. 엔티티(구체 대상) 단위로 큐레이션.

```js
// ⚠️ fact 는 반드시 '사람이 검증한 사실'만. LLM이 지어낸 문장 금지.
//    fact 는 그 대상에 일반적으로 참인 짧은 한 문장(4~7세용). enShow=화면 영어, enSpeak=음성 한글표기.
export const KIDDY_TIPS = [
  { keywords: ["사자", "라이언"],        fact: "사자는 가족끼리 무리 지어 살아", enShow: "Lion",     enSpeak: "라이언",   emoji: "🦁" },
  { keywords: ["공룡", "티라노", "공룡들"], fact: "아주 옛날 지구엔 공룡이 살았대",   enShow: "Dinosaur", enSpeak: "다이노소어", emoji: "🦕" },
  { keywords: ["자동차", "자동차들", "카"], fact: "자동차는 바퀴가 굴러서 앞으로 가", enShow: "Car",      enSpeak: "카",       emoji: "🚗" },
  { keywords: ["우주", "로켓", "행성"],    fact: "우주는 아주아주 넓고 별이 많아",   enShow: "Space",    enSpeak: "스페이스",  emoji: "🚀" },
  // … 인기 카테고리(kidTopics.js) 대표 대상부터 20~30개. Freddie가 사실 검증·확장.
];

// 제목에서 첫 매칭 엔티티를 찾는다(없으면 null). 부분문자열 포함으로 단순 매칭.
export function detectTip(title = "") {
  const t = String(title);
  for (const tip of KIDDY_TIPS) {
    if (tip.keywords.some((k) => t.includes(k))) return tip;
  }
  return null;
}
```

- **커버리지:** `kidTopics.js`의 인기 카테고리(동물·공룡·자동차·우주·바다동물·곤충·직업 등) 대표 대상부터 ~20~30개. **팩트 문장은 Freddie가 최종 검증.**
- 여러 키워드가 매칭되면 **첫 항목**(배열 앞=인기순) 우선. 단순하게.

## 3. 대사 — 순수 템플릿 (LLM 0)

`kiddyLines.js` 패턴대로 로컬 템플릿. 팩트 뱅크 값을 채운다. **LLM 안 씀.**

```js
// 팩트 있을 때 (몇 가지 변형 중 랜덤 — 매번 같은 말 방지)
// "{voc}! {enShowKo} 영상 재밌었어? {fact}. 아 참, 영어로는 '{enShow}'! 따라해봐 — {enSpeak}!"
// 표시(화면)엔 enShow(영어), 음성(TTS)엔 enSpeak(한글표기)를 쓰도록 문자열을 분리 구성.
```

- **화면 텍스트**: "…영어로는 **Lion**! 따라해봐 — 라이언!" (영어 철자 노출 OK)
- **TTS 텍스트**: 같은 대사지만 "Lion" 자리를 **enSpeak(라이언)**로 바꿔 `voice.speak()`에 넘김. (CLOVA가 영어 철자를 깨서 읽는 것 방지 — I2 글↔말 분리와 동일 원리.)
- **팩트 없을 때(폴백)**: 기존 "다 봤어! 재미있었어? 🎉" 유지. **사실 지어내지 않음.**

## 4. 배치 (두 곳, 로직은 공유)

**(A) 비연속 종료 완료화면** ([:459](../client/src/components/VideoPlayer.jsx#L459), 그리고 portrait 패널 [:615](../client/src/components/VideoPlayer.jsx#L615)):
- "다 봤어! 재미있었어?" 자리를 **팁 한마디로 교체**(detectTip 매칭 시). 매칭 없으면 기존 문구.
- 아래 **"소감 말하기"(챗봇)·"목록으로" 버튼은 그대로 유지** — 회귀 금지. (팁 다음에 아이가 소감으로 이어짐. 이 자리는 나중에 STT가 들어갈 곳.)
- 팁 뜰 때 `voice.speak(팁_TTS텍스트, "bright")` + 다시듣기 버튼.

**(B) 연속재생 검수 대기** ([:345-454](../client/src/components/VideoPlayer.jsx#L345)):
- 검수 카드 **위에 키디 팁 말풍선** 추가. 검수 진행 UI(다음영상 카드·그래프·카운트다운)는 아래 유지.
- ⚠️ **타이밍:** 팁은 `videoEnded` 시작 시 뜨고 **preview→analyzing→scored 구간 내내 유지**. **검수(analyzing) 하위단계에 종속시키지 말 것** — 검수가 캐시로 순식간에 끝나면 팁이 잘린다.
- ⚠️ **음성 겹침:** 다음 영상 자동재생(`onPlayNext`) 직전에 팁 TTS 정지. (useKiddyVoice가 새 speak/언마운트에 정지하지만 여기선 명시적으로 확인.)

## 5. 검증 (구현 후 반드시)

**정탐:**
- 제목에 "사자" 포함 영상 → 완료화면에 "사자는… 영어로 Lion! 따라해봐 — 라이언!" + 음성이 **"라이언"으로** 읽힘(영어 철자 안 깨짐).
- 연속재생 ON: 영상 끝 → 검수 대기 중 **키디 팁이 뜨고**, 검수가 빨라도 팁이 안 잘림.

**폴백/안전:**
- 팩트 뱅크에 없는 주제(제목 매칭 없음) → 기존 "다 봤어! 재미있었어?"만, **사실 안 지어냄.**
- 화면엔 영어(Lion), 음성엔 한글(라이언) 분리 확인.

**회귀:**
- 기존 완료화면의 **"소감 말하기"·"목록으로" 버튼, 시청시간 저장(persistHistory), 시간초과 화면**이 안 깨졌는지.
- 연속재생 자동 진행(preview 3s→검수→scored 5s→자동재생)이 팁 추가로 안 깨졌는지.
- 다음 영상 재생 시 팁 음성이 **안 겹치는지.**

## 6. 막히면 멈추고 보고

- 팩트 뱅크에 **검증 안 된 사실**을 넣어야 할 것 같으면 → 넣지 말고 보고(Freddie 검증 필요).
- 연속재생 카드 레이아웃이 팁 말풍선까지 넣기엔 빡빡하면 → 갈아엎기 전에 보고.
- 검수 타이밍과 팁 노출이 안 맞으면(잘림/겹침) → 임의 조정 말고 보고.
- 대사에 LLM을 쓰고 싶어지면 → **쓰지 말 것.** 안전선이다.

*구현 후, 사자(또는 팩트 뱅크에 있는 주제) 영상으로 (A)비연속·(B)연속재생 양쪽에서 팁이 뜨는지 + 영어는 화면 영어·음성 한글인지 + 기존 버튼/저장 회귀 없는지 확인해 컨트롤 타워에 보고.*
