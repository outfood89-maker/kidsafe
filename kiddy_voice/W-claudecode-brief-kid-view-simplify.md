# [Claude Code 작업 지시서] W — 스트레치 1: 분석화면 아이 뷰 간소화 (G0 통과 개방, 기획서 1)

> **목적:** 7축 분석 그래프는 부모·심사위원용 증거이지 아이의 UX가 아니다. 아이에게는 "봐도 되는지"만 크고 단순하게 (UX 25점 — 아동 눈높이 설계의 증거). 분석력 과시는 '자세히 보기'로 보존.
> **대원칙: 순수 표시 계층 변경.** 재생 게이팅 로직·분석 데이터·API **무변경**. VideoModal은 데모 핵심 동선 — 회귀 스모크 필수 (팀장 명시).
> **카피 팀장 확정본 — verbatim, 임의 수정 금지.**
> **일정:** 스트레치 1 완료 후에만 스트레치 2(키디의 방) 개방. 막히면 멈추고 보고.

## 1. 변경 — [VideoModal.jsx](../client/src/components/VideoModal.jsx)

**아이 기본 화면 (모달 열림 직후):**
- 큰 신호등(기존 등급 색·이모지 활용) + 키디 한 줄:
  - 재생 가능(`canPlay` true, :177): **"키디가 미리 봤어! 안심하고 봐도 돼 🦕"**
  - 차단(canPlay false): **"이건 키디가 아직 안심 못 했어. 다른 거 보러 가자!"**
- 기존 7축 점수 카드 영역(:282~ 안전도 점수 + 참고 그룹)은 **"자세히 보기" 접기/펼치기 안에** — 기존 컴포넌트 그대로 재사용, 접힘 상태(state 1개)만 추가. 삭제·재작성 금지 (efadd94 재발방지 — 조건부 로직 보존).
- 점수 피드백("이 점수가 이상한 것 같아요" :300~)은 자세히 보기 안으로 같이 접힘 (분석을 보는 사람용).
- **부모 진입 경로가 있으면 기본 펼침 유지** — VideoModal이 부모 화면(ParentDashboard 등)에서도 쓰이는지 먼저 확인, 쓰이면 prop으로 분기. 안 쓰이면 아이 기본 접힘만.

## 2. 지키는 것 (변경 금지)

- `canPlay` 판정(:177)·재생 버튼 동작·연속재생/큐 연결 — 로직 한 줄도 변경 금지
- 정밀분석 업그레이드 로직(:91), 신고 접수(:61), 분석 API 호출 — 무변경
- 키디 대사는 Typewriter 관례 따를 것 (기존 모달 내 키디 대사 패턴 있으면 동일하게)

## 3. 검증 — 회귀 스모크 (팀장 필수 지정)

- **재생:** 안전 영상 클릭 → 새 간소 화면 + 재생 정상
- **게이팅:** 위험 영상 → 차단 카피 + 재생 불가 유지
- **연속재생:** 재생목록·추천에서 다음 영상 인라인 검수 흐름 기존과 동일
- **자세히 보기:** 펼치면 기존 7축·피드백 그대로 (diff에서 기존 렌더 코드 삭제 0)
- 모바일 뷰 1회 / build ✓
