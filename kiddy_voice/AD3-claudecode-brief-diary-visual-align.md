## §1. 시각 기준 요약 (목업 4컷의 문법)

- **배경:** 다크 에메랄드 라디얼(`radial-gradient` 상단 밝은 톤 → #0c1f1d 계열). 전용 풀화면.
- **헤더:** 좌측 `‹ 그만하기`(플로우 중) / 우측 여백 / 중앙 문패. 완성·책장 계열은 `‹ 가족 책장` 식 맥락 백버튼.
- **키디:** 화면당 **1회, 크게, 상단 중앙** (현행 KiddyImg PNG 사용 — 목업의 🦕 이모지를 PNG로 대체하는 것은 동일 문법으로 간주). 아래 **말풍선 카드**(질문 텍스트, Typewriter 유지).
- **다시 듣기:** 말풍선 아래 소형 pill `🔊 다시 듣기` → 현재 질문 TTS 재발화.
- **칩:** **2열 그리드, 이모지 위+라벨 아래의 큰 칩**(목업 .chip 문법). 말하기 버튼은 전폭(grid-column 1/-1).
- **진행 점:** 하단 중앙 5점 — `1=날씨 · 2=오늘의 질문 · 3=그림 참여 · 4=완성 확인 · 5=간직/완료`. 현재 단계만 활성(길쭉한 민트).
- **완성 페이지(③):** 크림 종이 카드 — 상단 `날짜 | 날씨: ☀️ · 기분: 🙂` 라인(점선 하단 보더), 그림 영역, 문장들(밑줄 라인 문법).
- **가족 책장(④):** 월별 '한 권' = **2열 그리드 크림 카드**(좌측 컬러 바·📕 큰 이모지·`{n}월의 이야기`·`N개의 이야기 · 쓰는 중/완성!`). 하단 안내 한 줄("한 장 한 장이 우리 가족의 추억이 돼요").

## §2. 시정 1 — 질문 전용 화면 분리 (한 화면 원칙)

**대상:** `client/src/components/DiaryFlow.jsx` 렌더부(현행 ~190행대 `overlay`/`sheet` 상수와 그 사용처).

- 현행 **바텀시트/센터 오버레이(max 520 시트) → 전용 풀스크린 화면**으로 교체: `fixed inset-0 z-50` + §1 배경, 체크인(DailyCheckin)과 같은 문법 — 큰 1화면 내 자결, 스크롤 이어붙이기 금지.
- 헤더 추가: 좌 `‹ 그만하기` = 기존 `onClose` 경로 재사용(“배경 클릭 닫기”는 제거 — 전용 화면엔 배경이 없음. 단 onClose 로직 자체는 불변). 중앙 문패 = §6-보류1의 `DIARY_TITLE` 상수.
- 각 스텝(weather/rotating/pick)은 §1 문법: 키디 크게 중앙 → 말풍선(kiddyLine) → 🔊 다시 듣기 → 칩 2열(이모지+라벨) → 말하기 전폭 → 진행 점.
  - 날씨 칩 이모지는 `WEATHER_CHIPS` 라벨에 이미 포함(☀️ 등) — 라벨을 이모지/텍스트로 분리 표시(문구 자체는 불변).
  - `SafetyBanner`(위기 고정응답)·`CRISIS_RETURN_HINT` 표시 위치·문구 불변 — 말풍선 아래 카드로 유지.
- result(완성)·done·entry(제안: §4로 이동함)도 같은 풀화면 문법. result의 크림 종이 카드에 `날씨 · 기분` 라인 추가(§1 — 데이터는 이미 props에 있음: weather 상태·checkinMood).
- 🔊 다시 듣기: 현재 kiddyLine을 `voice.speak(kiddyLine, "bright")` 재발화(신규 헬퍼 1개 허용). 낭독 중이면 중복 enqueue 방지(voice.stop 후 재발화).

## §3. 시정 2 — 키디 반복 아이콘 제거 (화면당 1회)

- **DiaryFlow:** 현행 '소형 키디(56px) + 한 줄' 옆배치 → §1 문법(큰 키디 중앙 1회 + 말풍선)으로. 그 외 자리에 키디 아이콘 추가 금지.
- **FamilyShelf(그림일기 홈):** 키디 중복 정리 —
  - 빈 책장 상태: 상단 '오늘 일기 쓰기' 카드의 KiddyImg(48)와 빈 책장 안내의 KiddyImg(140)가 **동시 노출 = 반복** → 카드 쪽 키디 제거(카드는 📖 이모지·텍스트만, KidHome 타일과 문법 통일은 §6-보류3 참조), 화면의 키디는 빈 책장 안내 1회만.
  - 브릿지·찢음 확인 상태: 각각 키디 1회 — 현행 유지(정합).
- **KidHome 타일:** 말하기 연습 배너(키디)와 그림일기 타일(키디)이 나란히 = 메인 화면에 키디 2회. 타일 쪽 KiddyImg를 📖 큰 이모지로 교체 검토 — **§6-보류3(팀장 회신 대기), 회신 전 현행 유지.**

## §4. 시정 3 — 자동 진입 금지 → 제안 단계

**해석(팀장 확인 대기 — §6-보류4, 회신 전 이 해석으로 구현):**
- **R5/R8 제안 경로(체크인 말미):** 현행 `diaryFinish` → DiaryFlow 오버레이 **자동 팝업** 금지. 대신 **reward(완료) 화면 안에 제안 요소**를 노출: 기존 ENTRY 제안 카피 + `좋아!`/`안 할래` 칩(전부 기존 스탬프 — DiaryFlow entry 스텝의 문구·분기·통계 기록을 그대로 이관). `좋아!` 탭 → DiaryFlow 전용 화면 진입(weather부터, entry 스텝 생략). `안 할래` → 기존 declineAndClose와 동일 기록(recordProposalResult(false)) 후 정상 완료.
  - 구현 위치: `DailyCheckin.jsx` reward 블록(диaryFinish 관련 ~530행대) — 자동 `setDiaryOpen(true)` 제거, 제안 UI 렌더로 대체. DiaryFlow에 `startAt="weather"` 류 prop 1개 허용(entry 생략용).
- **브릿지 diaryIntent 경로:** 아이가 방금 명시적으로 원한 진입("좋아!"를 이미 탭) = **연속 진행 유지**(체크인 완료 → 바로 DiaryFlow weather부터). 제안을 또 묻지 않음(이중 제안 금지).
- **FamilyShelf '오늘 일기 쓰기' 경로:** 아이가 버튼을 직접 탭 = 명시적 의도 → DiaryFlow weather부터(제안 생략, selfInitiated 통계 미기록 불변).
- **DiaryFlow entry 스텝 코드는 삭제하지 말고 `{false && ...}` 비활성 보존**(사유 주석).
