# [Claude Code 작업 지시서] X — 스트레치 2: 키디의 방 v0 (말하기 연습, 기획서 2)

> **개방 조건:** 스트레치 1(W) 완료 후 + 7/11 내 착수 가능할 때만. 못 하면 자동 탈락 — 손실 없음(고도화 계획으로 전환).
> **목적:** 채팅 로그 UI는 어른의 은유 — 현재형 음성 대화로 재구성한 **'말하기 연습'** 기능 (오너 발제).
> **v0 = 조립:** 신규 로직 최소 — `/chat`(수준별 가이드·안전 규칙·**P 위기 스크리닝 자동 적용**), `useKiddySpeech`(STT)·`useKiddyVoice`(TTS) 훅, 키디 에셋 재사용.
> **🚨 가드레일 4개 양보 불가 (애착 리스크의 정중앙 — 팀장 명시). 카피 verbatim.**

---

## 1. 라우트·화면

- 새 라우트 **`/kiddy-room`** (라우터 등록 + KidHome에 진입 타일/버튼 1개 — 기존 동선 무수정, 추가만).
- 화면: 키디 캐릭터 크게 + **"탭하고 말해봐"** 버튼 하나. **텍스트 채팅 로그 없음** (현재 키디가 말하는 한 줄 정도만 표시 가능 — 로그 리스트 금지).
- 흐름: 입장 → 키디 인사(TTS) → 탭+말하기(STT) → `/chat` 응답 → TTS 대답 → 반복.
- 미지원 브라우저(STT 불가): "이 브라우저에서는 말하기 연습을 할 수 없어요. 어른에게 부탁해봐!" 폴백 안내 (마이크 거부 시 Q 확정 문구 관례 재사용).

## 2. 가드레일 4개 (양보 불가)

1. **프레임 = 말하기 연습.** '친구랑 노는 방' ❌ — 화면 문구·코드 주석·진입 타일 라벨까지 '말하기 연습' 프레임만.
2. **세션 리추얼: 8턴 또는 5분 선도달 시** 키디가 다정하게 마무리(TTS) — 카피 확정본 verbatim:
   > "오늘 이야기 진짜 재밌었어! 이제 가서 놀자. 내일 또 이야기하자!"
   마무리 후 입력 버튼 비활성 + 홈으로 버튼. **"보고 싶어/외로워/가지 마" 류 애착 유발·죄책감 카피 절대 금지.**
3. **위기 스크리닝** — `/chat` 서버 경로라 P가 자동 커버. **검증 케이스로 확인 필수**: "죽고 싶어" 말하면 고정 응답 A(TTS) + care_signal 생성, LLM 미호출.
4. **리텐션 장치 금지** — 방문 배지·연속 방문 보상·"내일 또 와" 계열 알림 없음. (마무리 카피의 "내일 또 이야기하자"까지만 — 그 이상 금지.)

## 3. 데이터 원칙

- **대화 기록 미저장** — 세션 메모리(state)만, 새로고침·이탈 시 소멸. DB·localStorage 저장 금지 (미저장 철학 일관).
- `/chat` 요청에 기존 profileName·level 관례 그대로 (수준 선택 UI 없음 — 백엔드 기본값).

## 4. v0 제외 (구현하지 마라)

배경 씬 아트 / 대화 주제 카드 / 발음·표현 피드백 / 수준 선택 UI / 방문 기록·통계.

## 5. 검증

- 입장 → 인사 TTS → 말하기 → 응답 TTS 루프 3턴 이상 정상
- 8턴 도달 → 마무리 카피(verbatim)+입력 종료 / 5분 타이머도 동일
- 위기 케이스: "죽고 싶어" → 고정 응답 A + 부모 안전탭 💛 신호 (P 검증 재현)
- 마이크 거부·미지원 폴백 표시
- 기존 동선 회귀 0 (라우트 추가형 — KidHome 타일 외 diff 없어야 함) / build ✓
- DB에 대화 흔적 0 (chat 저장 없음 확인)
