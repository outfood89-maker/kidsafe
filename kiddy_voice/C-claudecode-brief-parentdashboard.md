# [Claude Code 작업 지시서] C — 부모 주간 리포트 화면 (ParentDashboard)

> **시각 타깃: `parent-report-mockup.html`** (같이 받은 목업). 이 느낌으로 만든다 — 단, React 19 + Tailwind + 실제 엔드포인트로.
> 데모의 심장. **'대시보드'가 아니라 '키디가 보내는 편지'.** 막히면 멈추고 보고.

---

## 0. 목표

부모가 `ParentDashboard`(`KiddyReportCard`)를 열면, 아이의 한 주가 **편지처럼** 펼쳐진다.
이미 살아 있는 두 엔드포인트를 소비해 목업의 블록 순서대로 렌더한다. **백엔드는 건드리지 않는다**(데이터는 다 있음).

스택: React 19 / React Router v7 / Tailwind. 기존 컴포넌트 `KiddyReportCard` 확장.
팔레트: 크림 `#F8F7F2` / 세이지 `#6DAB60` / (브랜드 빨강 `#C84B47`은 **이 화면에선 거의 안 씀** — 아래 가드레일).

---

## 1. 데이터 소스 (이미 구현됨 — 읽기만)

**먼저 실제 응답을 찍어보고 필드명을 확인할 것.** 아래는 알려진 형태지만, 실제와 다르면 실제를 따른다.

- `GET /reports/checkins?profile_id=&period=week` → 블록1·2
  - `moodTimeline` : 요일별 `{date, mood}` (mood enum: happy/good/soso/sad/angry)
  - `moodSummary` : `{trend, counts, note}` — `note`는 **백엔드가 만든 한 줄**(그대로 출력)
  - `sharedHighlights` : `share_with_parent=true` 인 것만 — 블록2 카드들
  - `kiddyMessage` : **키디 한마디(Haiku 생성)** — 그대로 출력
  - `hadSecrets` : boolean — 블록2 비밀 한 줄 토글
- `GET /reports/insights` (+`/reports/coach`) → 블록3
  - 영상 집계(개수, 안전 비율, 대표 제목들). **분리 유지** — checkins 응답에 합치지 말 것.

**⚠️ 백엔드 동적 vs 프론트 고정 경계 (절대 혼동 금지):**
- 동적(백엔드 출력 그대로): `kiddyMessage`, `moodSummary.note`, `sharedHighlights`, `moodTimeline`, insights 집계. **하드코딩 금지.**
- 고정(프론트): 섹션 라벨("지우의 마음 흐름" 등), 날짜 포맷, 닫는 말 템플릿(§4).

---

## 2. 블록 매핑 (목업 ↔ 데이터)

| 목업 블록 | 데이터 | 비고 |
|---|---|---|
| 헤더 (이름·기간) | profileName + period | 날짜 포맷은 프론트 |
| 키디 손글씨 히어로 | `kiddyMessage` | 그대로. **히어로 = 이 화면의 주인공** |
| 블록1 마음 흐름 | `moodTimeline` + `moodSummary.note` | mood→이모지 매핑은 프론트 |
| 블록2 들려준 이야기 | `sharedHighlights` | share=true만 (백엔드가 이미 거름) |
| 블록2 비밀 한 줄 | `hadSecrets === true` | 아래 §3 |
| 블록3 본 것 | `/reports/insights` | 시각적으로 **조용하게**(보조) |
| 블록4 모은 별 | insights/뱃지 | 작게 |
| 닫는 말 | 프론트 템플릿(§4) | mood trend 기반 분기 |

mood→이모지: happy `😄` / good `🙂` / soso `😐` / sad `😢` / angry `😡`.
체크인 없는 날: 목업처럼 빈 칸 또는 옅은 점(‘대화 안 한 날’). 지어내지 말 것.

---

## 3. 비밀 한 줄 (`hadSecrets`) — 윤리선 핵심

`hadSecrets === true` 일 때만, **블록2(sharedHighlights) 맨 아래**에 한 줄:

> "{이름}만의 비밀도 있었어요. 그건 키디가 지켜요 🤫"

- 톤: 따뜻하고 조용하게(목업의 흐린 작은 글씨). **이건 '알림'이 아니라 '안심'**이다.
- 🚫 경고색·아이콘·배지 금지. 빨강·노랑 금지.
- 🚫 무엇을/몇 개 숨겼는지 암시 절대 금지. 펼치기 인터랙션 금지.
- `false`면 이 줄 자체를 렌더 안 함(빈 자리도 X).

---

## 4. 닫는 말 (프론트 템플릿, mood trend 분기) — 진단 금지

`moodSummary.trend`(또는 counts)에 따라 **따뜻한 행동 제안 한 줄.** 진단·전문가 언급 0.

- 대체로 긍정: "이번 주 즐거운 지우, 오늘도 꼭 안아주세요 💚"
- 무거운 날이 있었음: "오늘은 지우를 한 번 꼭 안아주는 건 어때요? 💚"
- 무거운 날이 많았음: "요즘 지우 마음이 조금 무거웠어요. 곁에서 가만히 안아주세요 💚"

🚫 절대 금지: "우울/불안 징후", "전문가 상담", "확인이 필요합니다" 류. 우린 의사가 아니다. **행동 제안까지만.**

---

## 5. 시각 가드레일 (목업과 일치시킬 것)

1. **편지지, 대시보드 아님.** 단일 컬럼, 넉넉한 여백, 둥근 카드. 지표·퍼센트·그래프 범벅 금지.
2. **마음 흐름은 Recharts 차트가 아니다.** 목업처럼 **하루 카드들의 가로 줄**(커스텀). 꺾은선/막대 금지(임상적으로 보임 = 안티패턴).
3. **슬픈 날은 빨강이 아니라 차분한 블루그레이.** 기분에 경고색을 쓰지 않는다. 슬픔은 문제가 아니라 곁에 있어줄 일.
4. **브랜드 빨강 `#C84B47`은 이 화면에서 거의 안 씀.** 부모 리포트는 안심이 목적이라 위험색을 피한다. 따뜻함은 세이지+크림+민트에서.
5. **키디 한마디(히어로)에 보강을 몰아준다.** 목업은 손글씨 폰트로 이걸 시그니처로 삼음. 나머지는 조용히.
6. 폰트: 손글씨(키디 목소리) + 본문 산세 페어링 권장(목업은 Gaegu + Gowun Dodum). **단, 앱 전체 느낌이 걸린 브랜드 결정이라 Freddie 확인 후 적용.**

## 6. 막히면 멈추고 보고 (임의 우회 금지)

- 실제 `/reports/checkins`·`/reports/insights` 응답 필드명이 위와 다를 때 → 실제를 따르되, 매핑이 모호하면 멈추고 보고.
- `KiddyReportCard` 기존 구조가 이 블록 구성과 충돌할 때 → 갈아엎기 전에 보고.
- 닫는 말 분기 기준(`trend` 값의 실제 종류)이 불명확할 때 → 추측 말고 확인.
- 폰트 도입은 Freddie 승인 전까지 기존 폰트로 두기.
