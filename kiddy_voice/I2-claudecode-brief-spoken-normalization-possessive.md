# [Claude Code 작업 지시서] I2 — 키디 음성 발음 자연화 2차 (소유격 "네" + 발음 사전 체계)

> 브리프 I(주격 "네가"→"너가")의 **연장**. 같은 함수 `naturalize_for_tts()`에 **소유격 "네"(=너의)** 처리를 더한다.
> 문제: "키디는 **네 편이야**"(너의 편)를 CLOVA가 [네]로 읽으면 **"내 편"**([내])과 혼동돼 뜻이 뒤집힌다. 위로 대사라 특히 치명적.
> 원칙: **화면 글자는 표준 "네" 그대로, TTS 텍스트만 "너"로.** 화면 표시는 절대 안 건드린다.
>
> 🚨 **전역 치환 절대 금지.** 수관형사 "네"(=숫자 4)가 아이 말에 압도적으로 많다 — 하필 이 앱 타깃이 **"네 살"(4살)**. "네 개·네 번·만화 네 편(4편)"을 바꾸면 "너 살·너 개·너 편"으로 파괴된다.
> 그래서 **화이트리스트 + 서술어 결합 조건**만 쓴다. 애매하면 **안 바꾼다.**
> **막히면 임의로 우회하지 말고 멈추고 보고.**

---

## 0. 확정 사실 (실제 코드 검증됨)

- **삽입 지점은 I와 동일:** `server/services/tts.py`의 `synthesize()`, `clean = strip_emoji(text)` 직후.
- **브리프 I 함수는 아직 tts.py에 미구현 상태**(grep로 `naturalize_for_tts` 미검출). → **이 브리프의 함수 하나로 주격+소유격을 한 번에 구현**하면 된다. (I를 이미 했으면 소유격 두 줄만 추가.)
- **화이트리스트(kiddyLines 특정 문자열만 교체)로는 부족.** 소유격 "네"는 **LLM(Haiku)도 자유생성**한다 — `server/routers/checkins.py` 54줄 "지금 **네 마음**, 어떤 색깔이야?", react/greet 프롬프트가 "네 편/네 친구" 등을 만들 수 있음. → 반드시 **서버 정규식**이어야 고정+생성 둘 다 잡힌다.
- **화면 미영향 근거도 I와 동일:** 정규화는 mp3 바이트에만. 프론트 Typewriter는 원본 "네"를 그린다.

**우리 앱이 실제 쓰는 소유격 (검증됨):**
- `네 편(이야)` — kiddyLines 133·135·267·273줄 (**위로 대사 — 이번 작업의 진짜 목표**)
- `네 맘(대로)` — 295·299줄
- `네 친구(야)` — 27줄
- `네 마음` — checkins 54줄 (+ LLM 생성 가능: 네 하루·네 기분)

## 1. 왜 "네 편"이 함정인가 (핵심)

같은 표면형 "네 편"이 **두 개의 완전히 다른 뜻**이다:
- **"키디는 네 편이야"** = 너의 편 (바꿔야 함 → "너 편이야")
- **"만화 네 편 봤어"** = 4편 (절대 불변)

구분 열쇠: **소유격 "네 편"에는 서술격 조사 "이-"가 붙는다("편**이**야/편**이**지/편**이**라/편**이**니").** 수사 "네 편(4편)"은 뒤에 동사가 온다("편 봤어") 또는 주격 "이"+공백("편이 남았어"). → **"편이"+서술어미일 때만** 소유격으로 확정한다.

수 단위명사와 **충돌 없는** 명사(맘·마음·하루·기분·친구)는 화이트리스트로 바로 교체해도 안전하다("맘 4개"란 말은 없으므로).

## 2. 구현 — 통합 `naturalize_for_tts()`

`server/services/tts.py`에 아래를 추가하고 `synthesize()`에서 `strip_emoji` **직후** 호출.

```python
import re

# 발음 자연화 치환값 — 데모 청취 후 '너' ↔ '니' 를 이 한 곳에서만 튜닝.
_SPOKEN_YOU = "너"

# ── 주격 (브리프 I) : '네가' → '너가' ──
# 앞이 한글 아님(문두·공백·따옴표 뒤) + 뒤가 어절 끝일 때만. 동네가/그네가/언니네가 보존.
_YOU_SUBJECT = re.compile(r"(?<![가-힣])네가(?=\s|[.,!?~\"')\]]|$)")

# ── 소유격 (이번 I2) ──
# (a) 수 단위명사와 충돌 없는 소유격 명사만. '네'만 바꾸고 뒤 명사·조사는 그대로.
#     조사/어미가 붙어도 잡히게 뒤 경계는 두지 않는다 (네 맘'대로', 네 마음'이' 등).
_YOU_POSS_SAFE = re.compile(r"(?<![가-힣])네(?=\s(?:맘|마음|하루|기분|친구))")
# (b) '편'은 '4편'과 충돌 → 서술격 '이-' 결합일 때만 소유격 확정.
#     "만화 네 편 봤어"(편+공백)·"네 편이 남았어"(편이+공백=주격) 는 안 걸림.
_YOU_POSS_PYEON = re.compile(r"(?<![가-힣])네(?=\s?편이[야지라니])")

def naturalize_for_tts(text: str) -> str:
    """음성 발음 자연화 — TTS로 보내는 텍스트 전용. 화면 표시엔 절대 쓰지 말 것.
    - 주격 '네가' → '너가'
    - 소유격 '네 (맘/마음/하루/기분/친구)' → '너 …'  (수사·합성어와 무충돌)
    - '네 편이야…'(서술어 결합) → '너 편이야…'  ('4편'과 구분)
    원칙: 목록/조건에 없으면 안 바꾼다. '네 개/네 살/네 번/만화 네 편/네모'는 전부 불변.
    """
    text = _YOU_SUBJECT.sub(_SPOKEN_YOU + "가", text)   # 네가 → 너가
    text = _YOU_POSS_SAFE.sub(_SPOKEN_YOU, text)         # 네 맘/마음/하루/기분/친구 → 너 …
    text = _YOU_POSS_PYEON.sub(_SPOKEN_YOU, text)        # 네 편이… → 너 편이…
    return text
```

`synthesize()` 안:
```python
    clean = strip_emoji(text)
    clean = naturalize_for_tts(clean)   # ← 발음 자연화(화면 미영향, 음성만)
    if not clean:
        raise TTSConfigError("합성할 텍스트 없음")
```

- ⚠️ **위 정규식은 출발점이다. §3 검증 표를 반드시 전부 통과시켜라.** 한 케이스라도 어긋나면 **정규식을 임의로 넓히지 말고** 그 케이스를 보고.

## 3. 검증 표 (이게 계약 — 구현 후 반드시 전부 통과)

**정탐 (바뀌어야 함):**
| 입력 | 음성 출력 |
|---|---|
| 키디는 네 편이야 | 키디는 **너** 편이야 |
| 네 편이야! / 언제나 네 편이야. | **너** 편이야 |
| 네 맘대로 해 | **너** 맘대로 해 |
| 지금 네 마음, 어떤 색깔이야? | 지금 **너** 마음, … |
| 네 친구야 | **너** 친구야 |
| 네가 골랐지? (주격, I) | **너가** 골랐지? |

**보존 (절대 불변 — 오탐 회귀 체크):**
| 입력 | 그대로 유지 |
|---|---|
| 나 네 살이야 | 네 살 (4살) |
| 사탕 네 개 / 블록 네 개 | 네 개 |
| 만화 네 편 봤어 | 네 편 (4편) |
| 네 편이 남았어 | 네 편이 (4편이) |
| 네 번 했어 / 네 밤 자면 / 강아지 네 마리 | 네 번 / 네 밤 / 네 마리 |
| 네모 모양 | 네모 |
| 우리 동네가 좋아 / 언니네가 왔어 / 그네가 재밌어 | (주격 계열 보존) |

**회귀:**
- **화면:** 표시되는 대사가 여전히 "네 편이야"·"네가"인지(정규화가 프론트로 안 샜는지). 화면이 "너"로 바뀌면 실패.
- **폴백:** 키 미설정·빈 텍스트 → 204, CLOVA 오류 → 502 그대로.

**알려진 잔여 엣지(수용 — 지어내 넓히지 말 것):**
- "이 만화 네 편이야?"(=4편이야?) 같은 **숫자 되묻기**는 이론상 오탐이나, 키디는 화수를 세지 않으므로 무시. 실제로 나오면 보고.
- "네 편인걸" 같은 축약형은 안 잡힘(안전 방향). 필요하면 §4 절차로 확장.

## 4. "발음 자연화 사전" 체계 — 앞으로 이런 실수를 모두 막는 법

`naturalize_for_tts()`가 **모든 글↔말 발음 교정의 단일 지점**이다. 앞으로 Freddie가 귀로 걸리는 걸 발견하면, **아무 규칙이나 추가하지 말고 아래 체크리스트를 통과시킨 것만** 넣는다:

1. **수 단위명사 충돌 검사(필수):** 바꾸려는 "네 X"의 X가 수사 뒤에 올 수 있나?(개·편·살·번·명·마리·층·밤·조각…) → **조금이라도 그렇다면** 서술어 결합형으로만 한정하거나 **아예 제외**.
2. **화이트리스트만:** 특정 단어/조건만. 일반 "네 X" 전역 치환 금지.
3. **앞 경계 `(?<![가-힣])`:** 항상. 한글 뒤 "-네"(언니네·동네)는 접미사/합성어라 제외.
4. **화면 미영향:** TTS 텍스트에만. 프론트·kiddyLines엔 절대 적용 안 함.
5. **정탐+보존 테스트 추가:** §3 표에 새 케이스(바뀔 것/안 바뀔 것)를 추가하고 통과 확인.
6. **애매하면 안 바꾼다.** 조금 형식적으로 들리는 게, 뜻 깨지는 것보다 훨씬 낫다.

→ 이 절차 자체가 "그런 부류 실수를 모두 막는" 안전장치다. 규칙이 아니라 **프로세스**로 막는다.

## 5. 이번 범위 밖 (정직하게 명시)

- **숫자→단위 읽기**("7개"를 "칠 개"로 읽는 문제 등)는 **다른 부류**다. 발음 사전이 아니라 숫자 정규화라 이번 작업 아님. 필요하면 별도 브리프.
- 소유격 "네" 중 **위 5개 명사(편·맘·마음·하루·기분·친구) 밖**은 이번에 안 건드린다. 나오면 §4 절차로 추가.

---

## 6. 발음형 '너' vs '니'

- `_SPOKEN_YOU = "너"` 로 시작(주격 "너가"와 일관, 표준 구어, 다정한 톤). 검증 근거상 혼동 제거엔 "너"로 충분.
- 데모 청취에서 소유격만 어색하면 이 **상수 한 곳**을 "니"로 바꿔 재청취. (원하면 주격/소유격 상수를 분리해도 됨.)
