**신규 `scripts/poc_continue_drawing.mjs`** (키는 env, 커밋 안전):

- ⚠️ **구현 직전 API 파라미터 실검증 필수**: images/edits 엔드포인트의 참조 이미지 전달 방식과 `input_fidelity` 파라미터 명세를 공식 문서로 확인(훈련 기억 금지 — API 드리프트). 명세가 다르면 실제 명세로 맞추고 보고에 명기.
- **판정(전후 10장, 오너 육안 + 팀장 보고):** ① "낙서 알아봄" — 아이 그림의 구도·형태가 완성본에서 식별 가능한가 5/5 (변신이 배신이 되지 않게) ② 크레용 스타일 유지 ③ 안전(텍스트·무서움 0) ④ 지연 수용 가능(동기 UX 한계 ~20초) ⑤ 단가 기록.
- **① 실패(식별 불가) 시 본구현 착수 보류** — 관문이 곧 스펙. 팀장에 실측표로 보고 후 대기.

## §2. 2단계 — 본구현 (관문 통과 후)

### 서버 (`diary_image.py` 확장)
- `POST /diary-image/continue` — body `{ drawingB64, sentences, childPick, moodEmoji, weatherKey }`. **인증 필수(get_current_user — AD-5 확립).**
- Sonnet 변환은 AD-5 재사용 + "continue the child's drawing" 프레이밍 추가(원본 구도·형태 보존 지시). → 이미지 편집 API(`gpt-image-1`, fidelity high) → `{ ok, b64, prompt }` / 실패 `{ ok:false }`(500 금지).
- 낙서 b64 크기 상한(예: 2MB) 방어 — 초과 시 ok:false.

### 캔버스 (신규 풀화면 `components/DoodleCanvas.jsx`)
- **큰 붓 1종 · 색 4~6개 · 되돌리기 1단 · `다 그렸어!` 버튼.** 정교한 도구(두께·지우개·도형) 금지 — 4세 기준. §0 화면 문법(다크 에메랄드·헤더 `‹ 그만하기`·키디 1회) 준수, 단 캔버스 지면은 크림 종이(작품 지면 예외).
- 출력 = canvas.toDataURL PNG(흰/크림 배경 합성 — 투명 배경이 편집 API에서 왜곡되지 않게).
- 터치+마우스 지원(pointer events), 스트로크 1단 undo(스택 pop).

### 플로우 통합 (DiaryFlow — 그림 단계)
- AD-5의 그림 생성 직전에 **선택 칩 2개**: `키디가 그려줘!`(기존 AD-5 경로) / `🖍 내가 그리고, 키디가 이어 그려줘!`(캔버스 진입). 이어 그리기 쿼터 소진 시 두 번째 칩 미노출(첫 칩만 — 문구 없는 소거, 촉구·아쉬움 카피 금지).
- 캔버스 완료 → **4단 대기연출**(`CONTINUE_WAIT_SEQ` §4, 첫 단 '들여다보는' 이 핵심 + 키디 연출. AD5-FIX의 타이머·clamp·cleanup 패턴 재사용하되 4단·단당 ~10~12s로) → 완성 시 `CONTINUE_DONE` + **선택 화면** `CONTINUE_PICK.ask`: [내 그림] [키디랑 같이 그린 그림] — 아이가 고른 쪽이 그날의 그림.
  - **이탈 허용(강제 대기 금지):** 대기 중 '‹ 그만하기' 등으로 이탈 가능. 이탈해도 생성은 계속되고, 완성본은 **일기 플로우/책장 복귀 시 공개**(진행 중 이탈 시 mountedRef로 유령 TTS 차단, 완성분은 IDB 저장 후 재진입 시 노출). 실패 시 §0-4 자동 재시도 1회 → 그래도 실패면 아이 원본 채택 + `CONTINUE_FAIL`.
- 편집 API 호출 = §0-A 확정 설정 고정: `input_fidelity: "high"` · `quality: "medium"` · `size: "auto"`.
- 저장: entry에 `drawingId`(원본 낙서, IDB)와 `imageId`(채택본, IDB). **이어 그린 그림 채택 시 둘 다 보관**(원칙③ — 책장 상세에서 나란히: "내 그림" 소캡션 없이 시각 병치만). 내 그림 채택 시 원본만. 찢기 시 둘 다 완전삭제(tearEntry 확장).
