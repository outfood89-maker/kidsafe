# 시연 영상 시나리오 v2 — 두 벌 (A안: 키디의 방 포함 / B안: 미포함)

*작성: 컨트롤 타워 2026-07-05 · v1(7/2) 대체 · 용도: **7/9 팀장·오너 '키디의 방 컷' 포함 결정** → 7/10 가녹화 → 7/12 최종*
*총 길이: A안 4~5분 / B안 3.5~4.5분. 내레이션·편집은 오너·팀장 몫 — 이 문서는 화면·클릭·타이밍만.*
*구조 원칙 유지: 기능 나열이 아니라 **하루의 흐름** — 아이의 낮 → (놀이) → 부모의 저녁.*

## 0. 녹화 전제 — v1 대비 전부 해소됨 ✅

v1의 전제 7개(비밀 멘트·랜딩 문패·씨앗·시드·데모 계정·시청 약속·배지 개편) **전부 구현·배포 완료.** 신규 전제만 확인:

| # | 전제 | 상태 |
|---|---|---|
| 1 | 랜딩 v2 "만남의 흐름" (AB+AC 준공) | ✅ 배포됨 |
| 2 | W 신호등 아이 뷰 / U 💜 배너 / X 키디의 방(Z 비주얼·정문) / Y 인증 가드 | ✅ 배포됨 |
| 3 | 시드 리프레시 (7/11) — 녹화일(7/12) 기준 💜·편지 발화 상태 | ⬜ 7/11 실행 |
| 4 | TTS 음성 포함 여부 — 7/6 강의실 음향 확인 결과 | ⬜ (미확인 시: 음성 포함 + 전 대사 자막 — v1 보험 유지) |
| 5 | (A안만) 키디의 방 마이크 실녹음 환경 — 조용한 공간 + 마이크 품질 | ⬜ 7/10 가녹화에서 판정 |

**녹화 계정 규칙 (v1 승계 + 강화):**
- 체크인 장면은 하루 1번 게이트 → **예비 프로필 2~3개** 준비, 테이크마다 교체. ⚠️ **해인 프로필 사용 금지** (U 시드 오염). 데모 계정은 부모 화면 조회 장면에만.
- **이름 연속성 (팀장 보강):** 예비 프로필들의 **이름을 전부 '하늘'로 생성** (오너 본인 계정 안에 — 시드 무오염). 장면 2의 키디 호칭("하늘아~")과 장면 4의 편지 주인("하늘")이 같은 아이로 보여야 한다 — 이름이 다르면 시청자에겐 다른 아이의 이야기가 된다.
- 데스크톱 전체화면·알림 끄기·커서 강조. 모바일 연출은 반응형 모드 통일.

## 장면 3.5 — 키디의 방 (A안 전용, 목표 ~30초) ★7/9 결정 대상

> **서사 위치 근거:** 영상을 다 본 다음 = "화면이 꺼져도, 키디는 곁에 있어요"(랜딩 ❸ 카피와 수미상응). 검수 앱이 아니라 관계 앱임을 실물로 증명하는 컷.

| 클릭/행동 | 화면에 보여야 하는 것 | 메모 |
|---|---|---|
| 3.5-1. 하단 '키디' 탭 | **키디의 방 입장** — 배경(글로우·별) + 키디 인사 음성 | Z 비주얼 반영본 |
| 3.5-2. 🎤 콕 → 아이가 한마디 (**대본: "공룡은 왜 없어졌어?"** 류 아이다운 질문) | 듣는 중 → **생각 중(키디 search 클립 전환)** → 키디 답변 타이핑+음성 | **구도 = 인사 → 아이 질문 → 키디 답** (팀장 지정). 실녹음 — 조용한 환경 필수 |
| 3.5-3. '‹ 홈으로' 복귀 | — | 8턴 리추얼까지 보여줄 시간은 없음 — 내레이션으로 "기록도 점수도 남지 않는 연습" 한 줄 처리 권장 |

**A안 리스크와 보험:** 마이크 실녹음이 최대 변수 — ① STT 오인식 NG 시 재입장(세션 리셋)로 재테이크 ② 가녹화(7/10)에서 2테이크 내 성공 못 하면 **B안으로 자동 전환** ③ 아이 목소리 섭외 불가 시 어른 목소리로 하되 질문 대본은 아이답게.
