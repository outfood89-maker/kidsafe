# [Claude Code 작업 지시서] kiddyMessage 이름 조사 — 플레이스홀더 토큰화

> 키디 한마디(`kiddyMessage`, Haiku 생성)의 이름 조사를 정확히 처리한다.
> **원칙: "사실은 코드가, 분위기만 LLM이."** 톤은 Haiku, 조사(문법 규칙)는 프론트의 `josa.js`가 보장한다.
> LLM에 "조사 맞춰줘"라고 시키지 않는다(확률적 → 가끔 틀림). 대신 이름을 토큰으로 빼고 프론트가 치환.

작업은 **백엔드 1 + 프론트 1**, 경계 분명. 막히면 멈추고 보고.

---

## 1. 백엔드 (`reports.py`) — 이름을 토큰으로 출력

**바꾸는 것:** kiddyMessage 생성 시, Haiku에게 **실제 이름을 주지 말고** 플레이스홀더 `{{CHILD}}`를 쓰게 한다.

- 프롬프트에서 아이 이름이 들어가던 자리를 `{{CHILD}}` 토큰으로 교체.
- 시스템/지시 프롬프트에 명시: **"아이를 부를 때는 반드시 `{{CHILD}}` 라고만 써라. 실제 이름이나 조사(이/가, 을/를)를 붙이지 마라. 예: `{{CHILD}} 이번 주에도 마음을 들려줬어요`."**
- 즉 Haiku 출력에는 `{{CHILD}}`가 그대로 남는다(조사 없이). 백엔드는 이 문자열을 **치환하지 않고 그대로** 응답에 실어 보낸다.
- ⚠️ 백엔드에서 `{{CHILD}}`를 실제 이름으로 바꾸지 말 것. 치환은 프론트 몫(조사 규칙이 프론트에 있음).

**검증:** 응답 `kiddyMessage`에 `{{CHILD}}`가 (보통 1~2회) 들어 있는지 확인. 이름·조사가 박혀 나오면 프롬프트가 안 먹은 것.

---

## 2. 프론트 — `{{CHILD}}`를 josa로 치환

`kiddyMessage`를 화면에 그리기 직전, `{{CHILD}}` 토큰을 이름+조사로 치환한다. **기존 `utils/josa.js` 재사용.**

핵심: 토큰 **뒤에 오는 조사**를 보고 알맞게 붙인다. 두 가지 처리 방식 중 하나로 구현(Claude Code 판단, 단순한 쪽 권장):

**방식 A — 토큰에 조사 종류를 실어 보냄 (백엔드도 약간 관여):**
`{{CHILD:이가}}` `{{CHILD:을}}` 처럼 토큰에 조사쌍 힌트를 넣게 하고, 프론트가 파싱해서 `josa(name, ...)`. → 정확하지만 프롬프트가 복잡.

**방식 B — 프론트에서 토큰+뒤따르는 조사를 정규식으로 처리 (권장, 백엔드 단순):**
백엔드는 `{{CHILD}}` 다음에 조사를 그대로 쓰게 두고(예: `{{CHILD}}이 들려줬어요`, `{{CHILD}}을 안아주세요`), 프론트가 **`{{CHILD}}` + 바로 뒤 조사**를 잡아 교정.

```js
import { josa } from "./josa";

// {{CHILD}} 뒤에 붙은 조사를 보고 이름+올바른 조사로 치환
export function renderKiddyMessage(message, name) {
  const pairs = [
    [/\{\{CHILD\}\}이가|\{\{CHILD\}\}가/g, () => josa(name, "이가", "가")],
    [/\{\{CHILD\}\}이\b/g,  () => josa(name, "이", "가")],   // 주격 '이/가'
    [/\{\{CHILD\}\}은\b|\{\{CHILD\}\}는/g, () => josa(name, "은", "는")],
    [/\{\{CHILD\}\}을\b|\{\{CHILD\}\}를/g, () => josa(name, "을", "를")],
    [/\{\{CHILD\}\}와|\{\{CHILD\}\}과/g,  () => josa(name, "과", "와")],
    [/\{\{CHILD\}\}/g, () => name],  // 조사 없이 단독으로 남은 경우
  ];
  let out = message;
  for (const [re, fn] of pairs) out = out.replace(re, fn);
  return out;
}
```

- `kiddyMessage` 렌더하는 곳에서 `renderKiddyMessage(msg, profileName)`로 감싸기.
- ⚠️ 백엔드가 방식 B를 따르려면 §1 프롬프트를 "토큰 뒤에 평소대로 조사를 붙여라(`{{CHILD}}이`, `{{CHILD}}을`). 단 이름은 절대 직접 쓰지 마라"로 조정.
- **둘 중 실제로 채택한 방식에 맞춰 §1 프롬프트 문구를 정확히 맞출 것**(A면 토큰에 힌트, B면 토큰+조사). 혼용 금지.

---

## 3. 검증 (받침 있는 이름으로)

- 해인(받침○): "해인이 들려줬어요", "해인을 안아주세요" 등 전부 정상인지.
- 지우(받침✕): "지우가 들려줬어요", "지우를 안아주세요" 정상인지.
- 화면에 `{{CHILD}}`나 `{{CHILD:...}}`가 **그대로 노출되면 실패**(치환 누락) → 어느 조사 케이스가 빠졌는지 보고.

## 막히면 멈추고 보고

- Haiku가 `{{CHILD}}` 지시를 자꾸 무시하고 실제 이름을 박을 때 → 프롬프트 강화 필요, 보고.
- josa로 못 거르는 희귀 조사(에게/한테 등)가 kiddyMessage에 나올 때 → 케이스 추가 위해 보고.
