## §1. 서버 — 프롬프트 변환(Sonnet) + 이미지 생성(gpt-image-1-mini)

**신규:** `server/routers/diary_image.py` + **`server/main.py`에 import + `include_router(prefix="/diary-image")` 등록**(CLAUDE.md 규칙 — 누락 시 404 사고 전례).

- `POST /diary-image/generate` — body `{ sentences: [..], childPick, moodEmoji, weatherKey }` (Pydantic, Optional 필드는 `Optional[...]` — 422 사고 전례).
- 1단계 Sonnet 변환: `AsyncAnthropic`(reports.py 패턴, 모델 env `IMAGE_PROMPT_MODEL` 기본 `claude-sonnet-5`). **시스템 프롬프트 = 대본 §2 verbatim 3블록**: 고정 스타일 블록 + 장면 변환 규칙(일기 문장 소재만·문장 밖 창작 금지·인물 일반화·child_pick 필수 등장·키디 아기공룡 동반·😢 차분하되 무섭지 않게) + 안전 제약 말미 고정. 출력 JSON `{"prompt": "..."}` 하나만 — 파싱 실패 시 §2 완성 예시 구조로 코드 조립 폴백(LLM 실패가 그림 실패로 번지지 않게).
- 2단계 OpenAI Images: `gpt-image-1-mini`, `OPENAI_API_KEY` env, size 1024×1024(또는 mini 기본), **응답 b64 반환** `{ ok, b64, prompt }`. httpx 등 async 호출, 타임아웃 넉넉히(30s+). 외부 응답 접근은 `.get()` 방어(KeyError→500 전례).
- 실패(어느 단계든): `{ ok: false }` — 프론트가 폴백 카피 처리. 서버는 500 던지지 말 것.

## §2. 프론트 — 저장(IndexedDB) + 플로우 통합

- **신규 `utils/diaryImageStore.js`**: 이미지 전용 IndexedDB 헬퍼(외부 의존 0, raw IDB ~40줄: put/get/delete(id)). **localStorage에 base64 넣지 말 것** — 한 달치면 5MB 쿼터 초과. IDB 불가 환경은 조용히 플레이스홀더 폴백.
- entry에 `imageId`(선택 필드)만 추가 — diaryStore 직렬화 불변식 영향 없음. **찢기 시 IDB 이미지도 함께 삭제**(완전삭제 불변식).
- `utils/api.js`에 `generateDiaryImage(payload)` 헬퍼(Axios, BASE_URL).
- **DiaryFlow result 스텝 순서(대본 §1 '조립·낭독' verbatim 순서)**: 낭독(READ_INTRO+enqueue, 기존) → 그림 대기 카피 `IMG_WAIT` 표시+TTS+로딩(플레이스홀더 자리에서) → 성공 시 `IMG_DONE` 표시+TTS+이미지 렌더 → KEEP.ask(기존). 실패 시 `IMG_FAIL` 표시+TTS+플레이스홀더 유지 → KEEP.ask 그대로(텍스트 저장 불변).
- 간직 시: 이미지 있으면 IDB 저장+entry.imageId. FamilyShelf 페이지 상세·완성 페이지에서 imageId 있으면 이미지 렌더, 없으면 기존 IMAGE_PLACEHOLDER + (오늘 페이지 한정) **재시도 버튼**(대본: "그림 재시도 버튼은 책장에서") — 재시도는 §3 재생성 한도와 별개(최초 생성 실패의 복구).

## §3. 그림 다시 그리기 + 오늘 일기 다시 만들기

- **다시 그리기**: 완성 화면(result)·오늘 페이지 상세에 `🎨 그림 다시 그려줘` — 같은 일기로 재생성, **하루 2회 한도**(diaryStore meta `regen: {date, count}` — 날짜 바뀌면 리셋). 소진 시 버튼 대신 `REGEN_OUT` 카피 표시. 재생성 성공 시 기존 이미지 IDB 교체.
- **다시 만들기**: **오늘 페이지 상세 한정** `다시 만들래` → 확인 다이얼로그 `REMAKE.confirm`(버튼 `REMAKE.yes`/`REMAKE.no`) → yes 시: 오늘 엔트리+이미지 삭제(tear와 동일 경로) → `startWrite()` 재사용(체크인 재사용 원칙 — 미체크인이면 브릿지). **새 플로우 '간직' 완료 시에만 대체** — 중도 이탈하면 이전 일기는 이미 삭제된 상태임을 확인 다이얼로그가 정직하게 말하고 있음("지금 일기는 사라지고") → 스펙 그대로 선삭제.
- 다시 만들기 후 최초 생성 1회는 재생성 한도와 별개(새 일기의 첫 그림). 한도는 재생성(`🎨`)에만.

## §7. 부록 — AD-4 시정 1건: 방 오브젝트 공간감 (KiddyRoom)

오너 스크린샷 판정: 현행 책장 오브젝트가 부속물처럼 작음. 방향 = **방을 '사물이 놓인 공간'으로**:
- 오브젝트 터치 타겟 **≥64px**, 이모지 크기 상향(≈48~56px 시각).
- **받침/선반감**: 오브젝트 아래 얇은 선반 바(또는 바닥 타원 그림자) + drop-shadow — '떠 있는 아이콘'이 아니라 '놓인 사물'로.
- 라벨 병기 유지(`📚 가족 책장`), idle 미세 애니메이션 허용(2.5~3s 부유/기울임 — KiddyImg 키프레임 재사용 가능).
- **오브젝트 존 구조**: 향후 오브젝트 추가(마이크 등) 상정 — 위치·받침·라벨을 공통 처리하는 존(배열 매핑) 구조로. 현재 항목은 책장 1개.
- 목업 미커버 화면 → §0 문법 준용 + 헤드리스 검증(오브젝트 렌더·탭·calm 가드 회귀), 실화면은 머지 패킷.
