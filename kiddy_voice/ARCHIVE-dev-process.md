## 1. 멀티에이전트 협업 구조 (차별화 서사의 핵심)

```
오너 (Freddie) ── 결정·중계·실행(SQL·배포·테스트)
 ├─ 자문 팀장 클로드 (Cowork) ── 전략·심사기준 분석·우선순위·카피 검수 (메모 1~4호)
 ├─ 클로드코드 2 = 컨트롤 타워 ── 설계·코드 검증·작업지시서(브리프)·리뷰 (구현 코드 작성 안 함)
 └─ 클로드코드 1 = 작업자 ── 브리프 기반 구현·"막히면 멈추고 보고"
```
- **브리프 핸드오프 프로토콜**: 컨트롤 타워가 `파일:줄` 앵커가 박힌 "붙여넣기용 완결 지시서"를 작성 → 오너가 작업자 창에 전달 → 작업자가 구현+검증 보고 → 컨트롤 타워가 diff 전수 리뷰(삭제 줄 포함) → 승인/재작업.
- **게이트 체계**: 카피(아동 대상 문구)는 팀장 검수 게이트 / 코드 변경 중 윤리 관련(비밀 저장 등)은 사전조사 보고 게이트 / DB 조작은 오너 수동 실행.
- **오너 제품 발견 사례 (2026-07-03):** Y 검수 중 오너가 발견 — 차단·숨김된 영상이 부모에게 흔적 없음, **"검수가 잘될수록 조용해지는 역설."** 팀장이 고도화 1순위 승인(설계 원칙 3개 확정, [ROADMAP-고도화.md](ROADMAP-고도화.md)). 오너·AI 역할 분리 속에서 제품 통찰은 오너에게서 나온 대표 사례.
- **'적대적 리뷰' 사례 (팀장 지정 기록, 2026-07-03):** 키디의 방(X) 출고 전, 컨트롤타워가 리뷰 에이전트 19개(4개 관점 → 발견별 반박 검증)를 돌려 **"위기 신호 유실 레이스"를 사전 발견** — 아이가 세션 종료 직전 위기 발화를 하면 부모 안전 신호가 조용히 사라지는 결함. **실제 위기가 와야만 드러나는 종류라 사전 발견이 아니면 영영 몰랐을 버그.** 확정 결함 8건 전부 출고 전 수정. "AI가 만들고 AI가 공격해보고 사람이 결정한다"는 품질 체계의 실물 증거.
- **'기계 검증으로 리허설 범위 축소' 사례 (팀장 권장 기록, 2026-07-03):** 리허설(7/8~10) 전에 컨트롤타워가 신규 표면 3종의 **판정 로직을 실코드 직접 호출 시뮬레이션으로 사전 검증** — 재생 게이트 진리표 18/18 · 위기 사전 17/17 · 감정 패턴 수학 5/5 전부 PASS. 덕분에 사람 리허설은 로직 재확인이 아니라 **화면·연결·실기기(음성 루프, 위기 발화→부모 카드, 브라우저 폴백)만** 보도록 범위를 좁힘([REHEARSAL-신규표면-테스트가이드.md](REHEARSAL-신규표면-테스트가이드.md)). "기계가 검증할 수 있는 것은 기계가, 사람은 사람만 볼 수 있는 것을" — 한정된 리허설 시간을 고위험 영역에 집중시킨 운영 사례.
- **'멈춤의 규율' 사례 (발표 실물 증거):**
  - **madeForKids 인증 우회 에스컬레이션 (2026-07-03):** W(아이 뷰 간소화) 작업 중 작업자가 스코프 밖 구멍 발견 — 인증 영상이 연령·상업성 가드까지 전부 우회(`canPlay = certified || …`). 임의 수정 않고 선택지 3안(A 유지/B 인증통과 폐지/C 절충)으로 에스컬레이션 → 팀장이 (C) "분석을 강제하지 않되, 아는 위험은 무시하지 않는다" 확정 (B는 무료 티어 구조 파괴, A는 서사 구멍) → Y 지시서로 변환.
  - **ASMR 에스컬레이션 (2026-07-02):** 작업자가 언어 감사 중 ASMR·먹방류가 검수를 통과하는 것을 **실측 2건으로 발견** → 임의 수정 대신 **구조적 갭("유해성 검수 ≠ 연령 적합성")으로 규정하고 선택지 3안(a/b/c)을 만들어 방향 결정을 요청** → 팀장이 (a)+(c) 승인·(b) 기각 + 연령 기준 5축 확정 → 컨트롤 타워가 T 지시서로 변환. 발견자가 고치지 않고 체계가 결정한다.
  - 그 외: Sonnet 5 502(작업자 보고→컨트롤타워 실호출 진단→M3), 비밀 저장 게이트(B 조사 후 착수), 위기 카피 게이트(P 검수 후 verbatim), "볼 것" 시제 오염(FRAME_HINT 규칙화).
- 헌장: [control-tower-role.md](../control-tower-role.md)

## 3. AI 활용 아키텍처 (신청서 'AI 활용' 항목 재료)

- **역할별 지능 배치**: 아이 실시간 상호작용(챗봇·체크인 반응·인사) = Haiku(빠름·저비용) / 부모 주간 리포트·AI 코치 = **Sonnet 5**(깊은 종합 분석). `REPORT_MODEL` 환경변수.
- **"사실은 코드가, 분위기만 LLM이"**: 수치·일정·감정 카운트는 서버가 결정적으로 계산, LLM은 톤 한 문장만. 근거 밖 창작 금지 프롬프트 + 로컬 템플릿 폴백.
- **가드레일 프롬프트 설계 사례** (원문은 코드에 보존):
  - 리포트: 비공유 데이터 프롬프트 미투입(구조적), `{{CHILD}}` 토큰(이름 조사 정확성은 프론트 josa), 진단·미래예측 금지
  - 대화의 씨앗: 공유분·감정 집계에만 근거, 캐묻기 금지, 4~7세 열린 질문
  - 부정 감정(😢😡) 받아주기: **LLM 미사용, 사람이 쓴 고정 템플릿** (가장 예민한 순간엔 검증된 문장)
  - 캐싱·버전 무효화: 프롬프트 버전 상수(v1→v4)로 캐시 자동 갱신 — 비용 절약+일관성

### 6-2. FRAME_HINT — 질문 유형별 시제/틀 고정 (`server/routers/checkins.py`, Haiku)

### 6-3. 위기 신호 사전 (`server/safety_lexicon.py` ↔ `client/src/utils/safetyLexicon.js` 동일 유지) — **LLM 0**

가장 예민한 순간엔 LLM을 쓰지 않는다. 사람이 쓰고 팀장이 검수한 확정 사전 + 고정 응답만 사용(임의 수정 금지). 판정 순서와 오탐 방어가 핵심:

- **① 폭력 피해** (`한테 맞았`, `때렸어` …) → 고정 응답 B + 부모 신호. **예외 문맥 미적용** — "게임하다가 아빠한테 맞았어"는 진짜 신호. (`한테 맞았`로 한정해 "퀴즈 맞았어"·"주사 맞았어" 오탐 방지)
- **② 자해·죽음** (`죽고 싶`, `사라지고 싶`, `자살` …) → 고정 응답 A + 부모 신호. 단 게임·공룡·좀비·동화 등 **3인칭/놀이 문맥이면 통과** (오탐 방지).
- **③ 외로움·불안** (`나만 미워`, `혼자인 것 같` …) → 고정 응답 C만, 부모 알림 없음.
- 매칭은 **공백 제거 후 부분일치**("죽고싶어" 붙여쓰기 우회 방지) — 단 `자살`만 원문 매칭("이모는 혼자 살아"→"혼자살아" 경계 오탐 방지, 팀장 발견).
- 위기 발화 텍스트는 **어디에도 저장되지 않는다** — 부모 신호 테이블(care_signals)에 텍스트 컬럼 자체가 없음(구조적 불가).

아이용 고정 응답 A(자해·죽음 계열) 원문:

```

```
