## 1. 멀티에이전트 협업 구조 (차별화 서사의 핵심)

```
오너 (Freddie) ── 결정·중계·실행(SQL·배포·테스트)
 ├─ 자문 팀장 클로드 (Cowork) ── 전략·심사기준 분석·우선순위·카피 검수 (메모 1~4호)
 ├─ 클로드코드 2 = 컨트롤 타워 ── 설계·코드 검증·작업지시서(브리프)·리뷰 (구현 코드 작성 안 함)
 └─ 클로드코드 1 = 작업자 ── 브리프 기반 구현·"막히면 멈추고 보고"
```
- **브리프 핸드오프 프로토콜**: 컨트롤 타워가 `파일:줄` 앵커가 박힌 "붙여넣기용 완결 지시서"를 작성 → 오너가 작업자 창에 전달 → 작업자가 구현+검증 보고 → 컨트롤 타워가 diff 전수 리뷰(삭제 줄 포함) → 승인/재작업.
- **게이트 체계**: 카피(아동 대상 문구)는 팀장 검수 게이트 / 코드 변경 중 윤리 관련(비밀 저장 등)은 사전조사 보고 게이트 / DB 조작은 오너 수동 실행.
- 헌장: [control-tower-role.md](../control-tower-role.md)

## 3. AI 활용 아키텍처 (신청서 'AI 활용' 항목 재료)

- **역할별 지능 배치**: 아이 실시간 상호작용(챗봇·체크인 반응·인사) = Haiku(빠름·저비용) / 부모 주간 리포트·AI 코치 = **Sonnet 5**(깊은 종합 분석). `REPORT_MODEL` 환경변수.
- **"사실은 코드가, 분위기만 LLM이"**: 수치·일정·감정 카운트는 서버가 결정적으로 계산, LLM은 톤 한 문장만. 근거 밖 창작 금지 프롬프트 + 로컬 템플릿 폴백.
- **가드레일 프롬프트 설계 사례** (원문은 코드에 보존):
  - 리포트: 비공유 데이터 프롬프트 미투입(구조적), `{{CHILD}}` 토큰(이름 조사 정확성은 프론트 josa), 진단·미래예측 금지
  - 대화의 씨앗: 공유분·감정 집계에만 근거, 캐묻기 금지, 4~7세 열린 질문
  - 부정 감정(😢😡) 받아주기: **LLM 미사용, 사람이 쓴 고정 템플릿** (가장 예민한 순간엔 검증된 문장)
  - 캐싱·버전 무효화: 프롬프트 버전 상수(v1→v4)로 캐시 자동 갱신 — 비용 절약+일관성
- **검수 파이프라인**: 키워드 Tier0 → 캐시 → 자막+썸네일 Vision 정밀분석(Tier2), 채널 학습. 60초 쇼츠 필터.
