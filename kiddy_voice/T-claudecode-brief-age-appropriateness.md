## 1. (a) 장르 소프트 플래그 — 즉시 (배틀 가드 패턴 재사용)

기존 [analyze.py:577](../server/routers/analyze.py#L577) `_battle_guard`(제목 기반 ageRating 강제 보정)와 동일한 방식. [safetyFilter.js:49](../client/src/utils/safetyFilter.js#L49)가 `ageRating > 프로필 나이`면 숨기므로 서버만 고치면 파이프라인이 알아서 거른다.

**키워드 최소셋 (팀장 확정 — 임의 추가·변경 금지, 변경은 팀장 검수 대상, P 사전과 동일 규율):**
```python
# 연령 상향(10+) 장르 신호 — 유해가 아니라 '아동 문법 부재' 장르 (T 브리프 §1, 팀장 확정셋)
AGE_GENRE_KEYWORDS = ["asmr", "먹방", "mukbang"]   # 매칭은 소문자 변환 후 부분일치 (ASMR 대소문자 무관)
# 예외 — 제목에 아동 문맥 동반 시 상향 제외 (아이용 자장가 ASMR은 적합할 수 있음. P EXCLUDE 패턴 재사용)
AGE_GENRE_EXCLUDE = ["동요", "자장가", "동화", "키즈", "kids"]
```
- **동작:** 제목(소문자 변환)에 AGE_GENRE_KEYWORDS 매치 **and** AGE_GENRE_EXCLUDE 미매치 → `ageRating`을 **최소 10으로 상향** (기존 값이 더 높으면 유지). 점수(violence 등)는 건드리지 않는다 — '후보 제외'가 아니라 연령 상향.
- **적용 지점 = 배틀 가드와 동일 3곳:** Tier0+1 검색 카드 2곳([analyze.py:653](../server/routers/analyze.py#L653)·[:692](../server/routers/analyze.py#L692) 배틀 안전장치 옆) + Tier2 결과 보정(`_battle_guard` 호출 자리와 동일 층위). 배틀 가드 로직 자체는 변경 금지 — 별도 함수(예: `_age_genre_guard`)로 나란히.
- ⚠️ 이미 캐시된 영상은 (a)만으론 안 바뀜 — Tier0+1 캐시는 재검색 시 갱신되는지 확인, 아니면 (c) 재분석과 함께 해소됨을 보고에 명시.

## 2. (c) Tier2 연령 축 정식화 — 기한 7/9 (R 시드 전 필수)

Tier2 정밀분석 프롬프트의 ageRating 가이드([analyze.py:397](../server/routers/analyze.py#L397) 부근)에 **팀장 확정 기준 5축을 verbatim으로** 반영:

> **연령 상향 기준 (ageRating 판정 시 반드시 고려):**
> ① 인지 부하: 전문 용어·복잡한 규칙·빠른 전개 (e스포츠·리뷰·시사)
> ② 파라소셜·응대형 톤: 속삭임, 시청자 개인 응대, "너만을 위한" 화법 (ASMR류)
> ③ 성인 지향 장르·정서: 먹방, 소비 자극 중심, 로맨스·공포 분위기
> ④ 광고·상업성 밀도: 제품 홍보가 본질인 콘텐츠
> ⑤ 아동 문법 부재: 아동용 어휘·말속도·시각 언어가 아닌 콘텐츠
>
> **원칙:** 유해 요소가 없다는 것과 아동에게 적합하다는 것은 다르다. 애매하면 차단이 아니라 연령 상향.

- 기존 ageRating 값 체계(3/5/7/10)·응답 JSON 형태는 그대로 (프론트 spread 호환 — CLAUDE.md 규칙).
- **⚠️ 캐시 무효화 (2026-07-03 정정 — 작업자 실코드 확인):** 재분석 트리거는 ~~prompt-rules.json mtime~~ → **DB `prompt_rules.updated_at`**(Phase 3c에서 DB 이전, analyze.py:756-767이 캐시 analyzedAt과 비교). 배포 후 오너가 `UPDATE prompt_rules SET updated_at = now() WHERE id = 1;` 실행 → high 캐시는 접근 시 lazy 재분석. Tier0+1 캐시는 룰 체크를 안 타므로 장르 키워드 매치분 표적 삭제 SQL 별도 실행 (컨트롤타워 처방). analyze.py:755 낡은 주석도 함께 정정.
- 완료 보고에 **"R 시드 착수 가능"** 여부 명시. 7/9 초과 예상 시 즉시 중단·보고.

## 3. 연속재생 큐 게이팅 — 검증 케이스 추가 (수정은 이미 반영됨)

PlaylistModal→KidHome `playQueue` 연결(이미 구현)은 승인. **팀장 조건: 재생목록 큐도 다음 영상 인라인 검수 게이팅을 동일하게 통과해야 함 — 큐를 넘긴다고 검수를 우회하면 안 된다.** 확인 사항:
- 재생목록에서 진입한 큐의 다음 영상도 기존 인라인 검수(analyzing→scored/blocked)를 타는지 코드로 확인 (경로 분기 없이 같은 playQueue 메커니즘인지).
