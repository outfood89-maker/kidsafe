## 0. 회의 배경

오너 발제로 일기 방향 빌드업 개시. 이번 회의에서 일기 설계의 가장 어려운 두 질문을 먼저 풀었다:

1. **"4살에게 '일기'라는 개념 자체가 어렵다"** (오너 제기)
2. **"부모가 일기를 보면 정말 도울 수 있을 텐데, 비밀 규약과 충돌한다"** (오너 제기)

전제 판정: **체크인은 현행 유지** — 지금 앱에서 가장 완성도 높은 표면이므로 손대지 않는다. 일기는 체크인 위에 세운다.

## 1. 확정 기둥 4개

### 기둥 ① — 체크인 재사용 원칙: "쓰는 일기가 아니라, 모이는 일기"

일기를 별도의 새 활동으로 만들지 않는다. 체크인이 이미 매일 수집하는 것(기분 + 오늘 한 일 + 후속 이야기)이 일기의 원재료다. v1 = 체크인 끝에 키디가 오늘 이야기를 정리해 들려주고 "간직할래?" 한 번 묻는 것. **아이의 추가 부담 0에서 시작**하고, 아이가 더 말하고 싶은 날만 덧붙인다.

### 기둥 ② — 주도권 사다리 (연령별)

4살의 개념 문제는 가르쳐서 푸는 게 아니라 은유와 주도권 설계로 푼다:

| 연령 | 방식 | 특징 |
|---|---|---|
| 4~5세 | **키디가 대필** | '일기'라는 단어 자체를 쓰지 않음(은유 후보: "오늘의 이야기 상자"). 키디가 체크인 재료로 2~3문장을 만들어 **음성으로 읽어주고**, 아이는 "간직할래/안 할래"만 선택. 글자·쓰기 없음. 표지는 그날의 기분 이모지 |
| 6~7세 | **함께 쓰기** | 키디가 뼈대를 만들고 아이가 칩으로 살 붙임 ("누구랑 있었어?") |
| 8세~ | **스스로 쓰기** | 말하기(STT)로 직접 덧붙임. '일기'라는 말을 이때 처음 도입 |

서사 자산: **"아이의 첫 일기는 키디가 대신 써주고, 마지막엔 아이가 혼자 씁니다."** (제품이 아이와 함께 자라는 구조 — 발전가능성 서사)

### 기둥 ③ — 4층 프라이버시 모델 (부모 열람 딜레마의 해법)

**논리 전제 (회의 합의):** 부모가 다 볼 수 있는 일기는 부모의 목적에도 자멸적이다. 아이가 열람을 인지하는 순간 일기는 표현이 아니라 공연이 되고, 발달 가치와 부모가 원한 진짜 신호가 함께 사라진다(관찰자 효과). "열람 vs 비밀"은 잘못된 양자택일 — 체크인의 해법("말은 아이의 것, 안부는 가족의 것")을 확장한다.

| 층 | 내용 | 접근 |
|---|---|---|
| 1층 | 일기 원문 | **아이만.** 부모 접근 구조적 불가 (hadSecrets 계보) |
| 2층 | 존재·흐름 | **항상 부모에게.** "이번 주 이야기 3번 남겼어요" + 감정 날씨. 내용 없는 신호 |
| 3층 | 내용 | **아이가 고른 것만, '선물' 프레임으로.** "이 이야기, 엄마아빠한테 선물할래?" — 공유가 추출이 아니라 아이의 기쁨이 되는 구조. 페이지 단위 |
| 4층 | 위기 | **P 체계 그대로.** 내용 아닌 존재-신호 + 대화 권유. 유일한 예외 |

**'관리'의 재정의:** 부모가 관리하는 것은 **기능과 안전**(일기 on/off, 안전 신호 수신)이지 내용이 아니다. 내용의 문은 아이만 연다.

**아이에게 정직 원칙 유지:** 일기 화면에 "엄마아빠는 네가 선물한 것만 볼 수 있어" 명시 (비밀 약속 멘트 2탄, 카피는 팀장 게이트).

### 기둥 ④ — 금지선 2개 (설계 불변식)

1. **비공유 일기에서 파생물 추출 금지.** "비공유 내용에서 주제만 뽑아 대화의 씨앗을 만들자"는 유혹이 반드시 온다 — 주제 추출도 간접 유출이다. 씨앗은 지금처럼 **공유분 + 감정 흐름에서만.**
2. **선물(공유)에 보상 금지.** 선물에 배지·리워드를 붙이는 순간 매수가 된다. 공유 무보상 원칙(배지 개편 N 브리프)과 동일선.

## 2. 체크인 관련 판정 (이번 회의 부속 결정)

- **현행 유지 확정.** 프리즈 존중, 심사 출고 품질.
- 고도화 후보로만 기록: ① **빠른 안부 모드** (리추얼 피로 관리 — 어떤 날은 기분 1탭만, 긴 버전은 주 2~3회) ② **감정 어휘 확장** (무서워·속상해·뿌듯해 등 — 연령 게이트 필수, 감정 사전과 한 몸)
- 두 후보 모두 일기 설계와 연동해 함께 풀 것.

## 3. 2차 회의 확정 (2026-07-04, 오너×팀장) — 미결 5건 해소

**① 이름: "이야기 상자" 확정.** '일기'라는 말은 8세+ 전환 시 처음 도입. 키디 발화 예: "오늘 이야기, 상자에 넣어둘까?"

**② 작성 메커니즘: 그림일기 조립 시스템 (오너 설계 — 핵심 확정).**
자유 대필이 아니라 **질문 → 선택지 칩(+말하기) → 문장 템플릿 조립**. 키디가 조건(질문·보기)을 주고 아이는 고르기만 하면 자동으로 일기 격식이 완성된다. 예: 날씨 질문에서 "맑음" 선택 → "오늘 날씨는 맑았어요."
- v1 문항(5문장): 날씨(신규 질문) / 기분(체크인 재사용 — 재질문 없음) / 한 일(체크인 재사용) / 누구랑(신규 질문) / 마무리(키디 고정 문장 풀)
- 새로 묻는 것은 2개뿐 — 체크인 재사용 원칙(기둥①) 유지
- '그림' 자리 = 그날의 기분 이모지 표지. "오늘의 스티커 고르기"는 v2
- 연령 사다리 적용: 4세 미만 칩만 / 4세+ 칩+말하기 / 8세+ 자유 문장 덧붙이기  *(말하기 게이트 6세+→4세+ 완화 · AD-10 §2)*

**⑤ 모델: v1 = LLM 0 확정 (오너의 Haiku 품질 우려에 대한 해소).**
템플릿 조립은 코드가 100% 결정적으로 수행 — 왜곡·환각 원천 불가, 즉시 생성, 비용 0. "사실은 코드가, 분위기만 LLM이"의 일기 완성형. v1.5(선택)에서 Haiku 윤문 레이어(연결어미·어미 변주만, 재료 밖 창작 금지·문장 순서 유지). 낭독은 TTS.

**② 진입점·보관 (팀장 추천 확정):** 생성 = 체크인 공유 선택 후 → 조립 일기 낭독 → "상자에 간직할래? / 오늘은 안 할래" → 리워드. 보관·열람 = **키디의 방 안에 놓인 상자 오브젝트** (하단 탭 증설 없음, 공간 은유 완성).

> **[재개정 2026-07-07 · 오너]** 자동 연속 폐기는 **'부르지 않은 제안'에 한함** — reward '그림일기 만들어볼까?' **제안 요소만 폐기(ⓐ)**. **명시적 일기 의도**(그림일기 타일 · 키디의 방 초대 '좋아!') 경로의 **브릿지 연속(체크인 완료 후 일기 진행)은 유지(ⓑ)**. 정상 데일리 체크인 종료는 **'영상 보러 가자!' 단일**. 구현 = AD-10 §1(재개정).

**③ 데이터·보존·삭제 (팀장 추천 잠정 확정 → 2026-07-07 개정):** "간직할래" 선택 시에만 저장(기본 미저장), 저장물은 조립 완성문뿐(음성 원문 미저장), 보존 무기한 + **아이의 페이지 단위 '지우기' 즉시 완전 삭제**, 선물한 페이지도 아이가 비밀로 회수 가능.

> **[개정 2026-07-07 · 오너 결정, 팀장 수용]** ⓐ **'찢어버리기' → '지우기'**: '찢다'의 폭력적 어감 제거. 지키려던 건 동사가 아니라 권리(아이가 스스로 지움 · 죄책감 문구 없음 · 정직한 되물음)이므로 은유만 교체. ⓑ **'부모 삭제 불가' → 부모(오너) 삭제 허용**: 그림일기는 비밀 채널(체크인=미저장)이 아니라 **가족 공유 책장**이라는 두 채널 아키텍처와 정합. 원저작자 존중은 '삭제 금지'가 아니라 '삭제의 무게'로 이동 → 부모 확인 팝업 본문 "**아이가 직접 만든 일기예요.** 한 번 지우면 되돌릴 수 없어요."(팀장 스탬프)가 균형추. 구현 = AD-9.

**④ 부모 관리 (팀장 추천 잠정 확정):** 프로필 단위 on/off, 부모 노출은 2층 신호만("이번 주 상자에 이야기 N개"), 연령 사다리 전환은 나이 기반 자동 + 부모 통지 한 줄.

**추가 확정 — 위기 유입 경로:** 자유 발화("말하기")는 체크인과 동일하게 P 위기 스크리닝 경유. 위기 텍스트의 일기 유입 없음을 검증 케이스로 확인할 것.

*상태: 설계 완료 — 고도화 1개월차 즉시 착수 가능. 잔여는 키디 멘트 카피 게이트(구현 직전)와 ③④에 대한 오너 최종 이의 확인뿐.*

## 4. 경쟁·서사 포지션 (기록)

경쟁 서비스는 전부 "로그 전체 열람(감시)" 또는 "아무것도 안 보여줌" 둘 중 하나다. **"부모는 흐름을 보고, 내용은 아이가 선물한다"**는 세 번째 답을 설계한 팀은 없다. 이 회의록의 4층 모델이 그 답이며, 고도화 산출물이자 발표·서면의 '향후 계획' 핵심 재료다.

*착수: 고도화 1개월차. 브리프 변환은 후속 회의(미결 5건 해소) 후 컨트롤타워가 진행.*
