# [설계 문서 v2] 우리 그림일기 — "엄마 아빠와 함께 쓰는 그림일기"

*3차 회의 (2026-07-04, 오너×팀장) · **일기 설계회의록 v1의 '비밀 상자' 개념을 대체** (작성 메커니즘·연령 사다리는 승계)*
*수신: 컨트롤타워 (고도화 백로그 1순위 갱신) · 착수: 고도화 1개월차*

---

## 1. 재개념화의 이유 — 오너가 발견한 함정

기존 설계(기본 비공개 일기)의 치명적 구멍: **아이가 꺼내 볼 수 있는 저장물은, 부모도 아이 프로필을 열면 몰래 볼 수 있다.** 공유 기기의 물리적 현실 앞에서 앱 계층의 비밀은 지켜질 수 없고, 지킬 수 없는 비밀을 아이에게 약속하는 것은 정직 원칙 위반이다.

**해법 = 개념 전환.** 비밀을 기술로 방어하는 대신, **처음부터 가족의 것으로 정의한다**: "엄마 아빠와 함께 쓰는 그림일기 — 쓰고 나서 자랑해요!" 부모가 봐도 '몰래'가 아니게 되고, 아이는 말하고 싶지 않은 날엔 그날 일기를 안 쓰는 것으로 스스로 조절한다 (자기결정의 자연스러운 형태).

## 2. 두 채널 아키텍처 (핵심 원칙)

| 채널 | 성격 | 비밀 보장 방식 |
|---|---|---|
| **체크인** | 마음의 채널 — 아이의 것 | 비밀 선택 가능, 그리고 **저장하지 않기 때문에** 물리적으로도 지켜짐 (O 미저장) |
| **그림일기** | 추억의 채널 — 가족의 것 | 처음부터 공유가 본질이라 '몰래'가 존재하지 않음 |

> 발표용 한 줄: **"말은 아이의 것, 추억은 가족의 것."** 두 채널이 서로의 정직함을 보증한다.

## 3. 작성 시스템 (2차 회의 승계 + 4차 입력 고도화 확정)

**3층 질문 구조 — "고정 뼈대 + 회전 질문 + 그림 참여":**

**1층 · 고정 뼈대 (아이 입력 1번 → 문장 3개):** 날짜·요일 자동 / 날씨 칩(☀️☁️🌧️⛄+말하기) / 기분·한 일은 **체크인 재사용 (재질문 없음)**. 일기 시작 시점에 이미 절반이 완성 — 성취감으로 출발.

**2층 · 오늘의 질문 1개 (회전 풀 8종, 최근 3일 중복 회피 랜덤):**

| 질문 | 조립 문장 | 비고 |
|---|---|---|
| 오늘 누구랑 제일 많이 있었어? | ○○랑 같이였어요 | 칩: 엄마/아빠/친구/할머니·할아버지/혼자 |
| 오늘 제일 맛있었던 건 뭐야? | 제일 맛있었던 건 ○○이었어요 | |
| 오늘 제일 재미있었던 순간은? | 제일 재미있었던 건 ○○이에요 | |
| 오늘 처음 본 거나 새로 안 거 있어? | 오늘 ○○을 처음 봤어요 | |
| 오늘 고마웠던 사람 있어? | ○○가 고마웠어요 | 부모가 오래 간직할 문장 |
| 오늘 무슨 소리가 제일 기억나? | ○○ 소리가 기억나요 | 6세+ 전용 (감각 어휘) |
| 내일 뭐 하고 싶어? | 내일은 ○○을 하고 싶어요 | 미래 소망형 — 예측 아님 |
| 오늘 뭘 제일 잘했어? | 나는 오늘 ○○을 잘했어요 | 자기효능감 |

**3층 · 그림 참여 질문 (매일 고정, 마지막):** "오늘 그림에 꼭 넣고 싶은 거 하나만 골라줘!" — 칩은 그날의 답에서 자동 생성 + 말하기. 답은 문장이 아니라 **그림 프롬프트에 직접 반영** — 아이가 AI 그림의 공동 작가가 된다 (창작 주권, 손그림 v2 전까지의 다리).

**마무리 문장:** 기분 연동 자동 풀 — 좋은 날 "오늘도 참 좋은 하루였어요" / 흐린 날 "내일은 더 웃을 수 있으면 좋겠어요" (소망형 — 미래 약속 금지 원칙과 충돌 없음).

**공통:** 문장 조립은 LLM 0 (코드 결정적) / 연령 사다리: 4~5세 칩만·6~7세 칩+말하기·8세+ 자유 문장 덧붙이기 / "말하기" 자유 발화는 P 위기 스크리닝 경유 (검증 케이스 필수) / 완성 후 키디 낭독(TTS).

**감정 배려 변주:** 체크인이 😢😡인 날은 강요 금지 — 키디: "오늘은 쉬어갈까? 쓰고 싶으면 써도 돼." 작성 시 마지막 버튼도 "자랑하기!" 대신 **"간직하기"**로 톤 변주 (흐린 날도 가족의 하루).

**v2 씨앗:** '누구랑' 칩에 당일 스케줄 자동 반영 (스케줄러 데이터 연동).

## 4. AI 그림 생성 파이프라인 (신규 핵심 기능)

**흐름:** 일기 문장 확정 → **Sonnet이 그림 프롬프트 작성** (일기 내용 → 영어 프롬프트 변환) → 이미지 생성 API 호출 → 그림 완성 → 진짜 '그림일기' 페이지 탄생.

**Sonnet 프롬프트 변환 규칙 (가드레일):**
- 스타일 고정 상수: "크레용·색연필 아동 그림일기 스타일, 따뜻한 톤, 텍스트 없음" — 매일 그림체가 일관되게
- 인물 일반화: 실명·실제 얼굴 묘사 금지. 아이는 프로필 아바타 스타일의 일반 캐릭터로, 키디 동반 가능
- 일기 문장에 있는 소재만 사용 (재료 밖 창작 금지 — FRAME_HINT 계보)
- 위기·부정 소재는 이미 상류(P·체크인)에서 처리됨

**이미지 API 후보 (2026.7 기준 시세 — 고도화 PoC로 확정):**

## 5. 보관·앨범·PDF (오너 설계)

- 날짜별 저장 → **월별로 '한 권'** — 월말에 그 달의 표지(최다 기분 이모지 + 대표 그림)와 함께 묶임
- 보관 은유: **"가족 책장"** (키디의 방 or 부모 대시보드에서 열람 — 위치는 구현 시 결정)
- **PDF 내보내기: 부모 소장용 앨범** — "아이의 한 달을 책으로." 인쇄 서비스 연계는 장기 로드맵
- **BM 직결:** PDF 앨범·고화질 그림은 프리미엄 1순위 후보 — 부모가 구독할 이유가 눈에 보이는 기능

## 6. 팀장 추가 아이디어 (오너 검토용)

1. **자랑 리추얼 + 부모 알림:** 완성 → "자랑하기!" 버튼 → 부모 대시보드에 "🎨 새 그림일기가 도착했어요!" 카드. 공유 여부 게이트가 아니라 **배달의 리추얼** — 아이가 보여주는 행위의 주인공이 됨.
2. **엄마아빠 도장:** 부모가 일기에 도장/스티커 하나로 답하기 (텍스트 코멘트보다 가볍고 4~7세에게 적절). **양방향 다리(P2)의 최소 실현형** — 부모→아이 반응 채널이 처음 생긴다.
3. **대화의 씨앗 연동:** 자랑 알림에 씨앗 결합 — "오늘 '무슨 그림이야?' 하고 물어봐 주세요." 그림일기가 저녁 대화의 세 번째 다리가 됨.
4. **🔥 이어 그리기 (오너 발제, v2 킥 기능 — 손그림 옵션의 완성형):** 아이가 손가락으로 그린 낙서를 **참조 이미지(img2img)**로 받아, 일기 프롬프트와 합쳐 AI가 "이어 그린다". 알아볼 수 없는 낙서여도 상관없음 — 오히려 그 변신이 마법. 설계 원칙 3개:
   - 프레임 = **"네 그림을 보고 키디가 이어서 그렸어!"** (대체가 아니라 협업). 참조 강도는 아이 그림의 구도·형태가 알아보게 남는 수준으로 (변신이 배신이 되지 않게)
   - **아이에게 최종 선택권**: 내 그림 그대로 vs 이어 그린 그림 — 자기 낙서를 고르는 것도 존중
   - **앨범엔 변신 전/후 나란히 보관** — 부모에겐 원본 낙서가 더 보물. PDF 앨범 가치 2배
   - 기술 검증 (✅ 2026-07-04 1차 조사 완료): **mini도 편집 엔드포인트로 참조 이미지 입력 가능** (GPT 이미지 계열 최대 16장). 단 **`input_fidelity` 파라미터는 mini 미지원** (상위 gpt-image-1 전용) — 낙서 보존 강도 제어 불가. → **팀장 권고: 이원화** — 일반 생성 = mini(확정 유지), 이어 그리기만 = gpt-image-1 상위(`input_fidelity: high`, 사용 빈도 낮아 단가 부담 없음). 낙서→크레용 변환 품질은 2라운드 실측 (ChatGPT 수동 테스트로 사전 감 잡기 가능). 캔버스 입력 UI(큰 붓·색 4~6개·되돌리기)는 신규 화면이므로 고도화 2단계
5. **디자인 톤:** 일기 '종이'는 밝은 크림 톤 허용 검토 — 다크 앱 속에서 "책을 펼친" 콘텐츠 영역이라는 예외 논리 (미니게임 다크 정렬과 다른 계층: UI가 아니라 작품 지면).

## 7. 지켜지는 원칙 (기존 결정과의 정합)

- **아이의 찢어버리기 권리 유지** — 가족의 앨범이어도 저자는 아이. 페이지 단위 완전 삭제, 부모 삭제 불가.
- **자랑에 보상 금지** — 자랑 버튼에 배지·리워드 없음 (공유 무보상 원칙과 동일 계보).
- **체크인 비밀 채널 불변** — 그림일기의 등장이 체크인의 비밀 약속을 침식하지 않는다. 두 채널의 구분을 아이 카피에서도 명확히 (카피 게이트).
- **음성 원문 미저장·판정 미저장 철학 유지** — 저장물은 조립 완성문 + 생성 그림뿐.
