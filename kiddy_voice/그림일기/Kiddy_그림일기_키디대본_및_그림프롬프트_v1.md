## 1. 키디 발화 대본 (플로우 순서대로)

### 진입 (체크인 공유 선택 직후)
- 기본: **"오늘 이야기로 그림일기 만들어볼까? 내가 도와줄게! 🎨"** (버튼: "좋아! / 오늘은 안 할래")
- 😢😡인 날 변주: **"오늘은 그냥 쉬어도 돼. 그래도 쓰고 싶으면, 내가 같이 할게."** (버튼: "써볼래 / 오늘은 쉴래")
- "안 할래/쉴래" 응답: **"응, 알겠어! 내일 또 물어볼게."** (아무것도 저장되지 않음, 재촉 알림 없음)

### 질문 단계
- 날씨: **"오늘 날씨는 어땠어?"**
- 오늘의 질문 8종 (설계 v2 §3 표의 문구 그대로 — 발화용 확정):
  1. "오늘 누구랑 제일 많이 있었어?"
  2. "오늘 제일 맛있었던 건 뭐야?"
  3. "오늘 제일 재미있었던 순간은 언제야?"
  4. "오늘 처음 본 거나 새로 안 거 있어?"
  5. "오늘 고마웠던 사람 있어?"
  6. "오늘 무슨 소리가 제일 기억나?" (6세+)
  7. "내일 뭐 하고 싶어?"
  8. "오늘 뭘 제일 잘했어?"
- 그림 참여(고정 마지막): **"오늘 그림에 꼭 넣고 싶은 거, 하나만 골라줘!"**

### 조립·낭독
- 낭독 전: **"다 됐다! 내가 읽어줄게, 들어봐~ 📖"** (TTS로 완성 일기 낭독)
- 낭독 후 (그림 생성 대기 — 동기안 기준): **"이제 키디가 그림을 그릴게. 잠깐만 기다려 줘! ✏️"**
- 그림 완성: **"짜잔! 오늘의 그림이야!"**

### 마무리 선택
- 기본: **"엄마아빠한테 자랑하러 갈까? 💌"** (버튼: "자랑하기! / 그냥 간직할래")
- 😢😡인 날 변주: **"이 이야기, 우리 책장에 간직해 둘까?"** (버튼: "간직하기 / 안 할래")
- 간직 완료: **"우리 가족 책장에 잘 넣어뒀어!"**

### 기타 상태
- 지우기 확인 (정직한 안내): 제목 **"이 일기를 지울까요?"** / 설명 **"한 번 지우면 되돌릴 수 없어요."** (버튼: "응, 지울래 / 아니야, 둘래") ※ 구 '찢어버리기' — '찢다' 폭력적 어감 제거, 오너 확정 2026-07-07
- 지운 후: **"응, 지웠어. 괜찮아!"** (죄책감 유발 문구 절대 금지)
- 부모(오너) 삭제 팝업 [AD-9]: 제목 **"이 일기를 지울까요?"** / 본문 **"아이가 직접 만든 일기예요. 한 번 지우면 되돌릴 수 없어요."** (버튼: "지우기 / 취소") — 팀장 스탬프 2026-07-07, '아이가 직접 만든'이 부모 삭제 허용의 균형추
- 그림 생성 실패 폴백: **"오늘은 그림이 잘 안 그려졌어. 이야기만 먼저 간직해 둘게! 그림은 다음에 다시 그려볼게."** (일기 텍스트는 정상 저장, 그림 재시도 버튼은 책장에서)

### 부모 측 문구
- 자랑 알림 카드: **"🎨 {아이}가 그림일기를 자랑하고 싶어해요! 오늘 '무슨 그림이야?' 하고 물어봐 주세요."** (대화의 씨앗 결합형)
- 도장 기능 채택 시, 도장 후 아이 화면(다음 접속 시 — 푸시 아님): **"엄마아빠가 네 일기에 도장을 찍어줬어! 보러 갈까?"**
  - ⚠️ 원칙 메모: 이 문구는 아이의 자랑에 대한 '응답'의 전달이므로 애착 유발 리텐션과 구분된다. 단 반드시 **다음 자연 접속 시 인앱 표시**로만 — 이걸 푸시 알림으로 보내는 순간 원칙 위반.

### 금지 조항 (대본 전체 공통)
- "보고 싶었어", "왜 안 왔어", "매일 써야 해" 류 애착·죄책감·의무 카피 절대 금지
- 일기 내용에 대한 평가("참 잘 썼네") 금지 — 과정 축하만 ("다 됐다!", "멋진 하루였네"는 허용 경계 — v1은 중립 유지)
- 자랑·간직에 배지·보상 연결 금지

## 1.5 상황 변주·빈틈 규칙 (4차 회의 확정 — 전부 LLM 0 결정적 규칙)

**R1. 질문 풀 감정 태그 (흐린 날의 잔인한 질문 방지):**
- 맑은 날 전용: "제일 재미있었던 순간" / "뭘 제일 잘했어" (2종)
- 전천후: "누구랑" / "맛있었던 것" / "처음 본 것" / "고마웠던 사람" / "무슨 소리" / "내일 하고 싶은 것" (6종)
- 체크인 기분이 😢😡인 날: **전천후 풀에서만 회전.** 흐린 날 전용 질문은 만들지 않는다 (감정 탐색은 체크인의 몫, 일기는 추억 채널).

**R2. "없었어" 출구 상시 제공:** 모든 회전 질문 칩 끝에 `음… 오늘은 없었어!` 상시. 선택 시 해당 문장은 일기에서 생략 (뼈대 3문장으로 일기 성립 보장), 키디 반응(고정): **"그런 날도 있지!"**

**R3. 날씨 "모르겠어" 선택 시:** 날씨 문장 생략 (조립 스킵).

**R4. 마무리 문장 풀:** 최근 2회 사용분 중복 회피 로테이션.

**R5. 진입 조건:** 일기 제안은 **당일 체크인 완료 시에만** (뼈대 재료 보장).

**R6. '말하기' 답변 인용 프레임 (문장 파손 방지):** 자유 발화 원문은 어미·조사 결합 금지, 인용 조립만 —
`{질문 리드}, "{원문}" 하고 이야기했어요.` (따옴표 안 내용과 무관하게 문법 성립. P 위기 스크리닝 경유는 기존 조건 유지.)

**R7. 자랑하기 완료 멘트 (신규 확정 카피):**
> **"엄마아빠한테 살짝 전해뒀어! 오늘도 이야기 들려줘서 고마워."**
(미래 예측 없음·내용 평가 없음 — 과정 감사만.)

**R8. 거절 빈도 하향 (강요 금지의 구현):** "안 할래" 3일 연속 → 격일 제안 / 7일 연속 → 주 2회. 아이가 가족 책장에 자발 방문하면 기본 빈도로 리셋.

**고도화 노트 (v0 비반영, 인지 기록):** ⓐ mood 코드 기준 변주의 한계 — 기분은 😄인데 후속답에 속상한 내용이 있는 날은 감지 못함 ⓑ 비속어 필터 부재 — 인용 프레임(R6)이 1차 방어, 전용 사전은 고도화에서 (P 사전은 위기 전용이므로 혼용 금지).

## 2. Sonnet → 이미지 API 프롬프트 템플릿

**Sonnet의 역할:** 조립된 일기 문장(한국어)을 아래 템플릿의 `{scene_summary}` 등으로 변환. 출력은 JSON `{"prompt": "..."}` 하나만.

### 고정 스타일 블록 (모든 그림 공통 — 그림체 일관성의 핵심)
```
A child's crayon drawing on cream paper, warm pastel colors,
simple joyful hand-drawn style with visible crayon strokes,
flat perspective like a 5-year-old's picture diary.
```

### 장면 변환 규칙 (Sonnet 시스템 프롬프트에 명시)
- **일기 문장에 있는 소재만 사용** — 문장 밖 인물·사물·장소 창작 금지 (FRAME_HINT 계보)
- 날씨 → 하늘 표현 (맑음=해, 비=구름과 빗줄기...)
- 한 일 → 장면의 중심 행동
- 누구랑 → 동반 인물 수 (인물은 전부 일반화된 아이/어른 캐릭터)
- 기분 → 표정과 팔레트 온도 (😢인 날은 차분한 톤 — 단, 어둡거나 무섭지 않게)
- **그림 참여 답({child_pick})은 반드시 화면에 등장** — 아이와의 약속
- 키디(작은 초록 아기공룡) 동반 여부: 기본 포함 (브랜드 각인 + 인물 일반화 보조)

### 안전 제약 (프롬프트 말미 고정)
```
No text, no letters, no numbers, no realistic human faces,
no logos or brands, no scary or dark imagery.
```

### 완성 예시 (일기: 맑음/기분 좋음/엄마랑 바깥놀이/그림에 미끄럼틀)
```
A child's crayon drawing on cream paper, warm pastel colors, simple joyful
hand-drawn style with visible crayon strokes, flat perspective like a
5-year-old's picture diary. A happy little child playing outside on a sunny
day with their mother, a playground slide in the scene, a small green baby
dinosaur friend nearby, bright sun in the sky. No text, no letters, no
numbers, no realistic human faces, no logos or brands, no scary or dark imagery.
```

### 검증 케이스 (구현 시)
1. 왜곡: 일기에 없는 소재가 프롬프트에 등장하지 않는가 (10건 표본)
2. 약속: {child_pick}이 10건 중 10건 그림에 실제로 등장하는가
3. 텍스트 렌더: 생성 그림에 글자가 없는가
4. 흐린 날: 😢 일기의 그림이 차분하되 무섭지 않은가
5. 스타일 일관성: 연속 7일 그림이 같은 그림체로 보이는가 (API 선정 PoC 겸용)

## 3. 남은 연결 고리 (구현 브리프 작성 시)

- 동기/비동기 결정(§설계 v2 8-3)에 따라 "잠깐만 기다려 줘" 구간 대본 확정
- 도장 채택 여부(8-4)에 따라 부모 측 대본 확정
- 이미지 API 선정(8-2) 후 스타일 블록 문구를 해당 모델에 맞게 미세 조정 (PoC에서)
- 8세+ '자유 문장 덧붙이기' 단계의 대본은 연령 사다리 구현 시 추가 (v1은 4~7세 대본만)
