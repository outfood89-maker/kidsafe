```html
<meta property="og:title" content="Kiddy(키디) — 아이의 마음을 매일 안부 확인하는 미디어 친구" />
<meta property="og:description" content="유해 영상을 거르는 데서 멈추지 않아요. 키디가 매일 아이의 마음에 안부를 묻고, 아이가 나누고 싶어한 이야기만 부모님께 전해드립니다." />
<meta property="og:type" content="website" />
```

## 4. 섹션 재배치 — Meet Kiddy + 체크인·리포트 컷을 미션 뒤로 전진

**목표 순서:**
```
① 히어로 → ①-b 정의(새 카피) → ② 공감 → ③ 미션
→ ⑦ Meet Kiddy (:720-735 — 카피 원문 유지, 통째로 이동)
→ ⑧ 중 체크인 컷(:748-766) + 키디의 한 주 리포트 컷(:768-794, 프라이버시 콜아웃 포함) — 이동
→ ④ 공생 → ④-b 비교표 → ⑤ 검수 방식(+도란도란) → ⑤-b 건강한 습관
→ ⑧ 잔여: 스케줄러 컷(:796-826) — 제자리
→ ⑨ 안심 → ⑩ CTA
```

**구현 주의 (동작 보존):**
- ⑧ 섹션이 둘로 쪼개진다 → 전진하는 체크인·리포트 컷은 **새 section + 새 `useInView` ref**로 감싸라 (기존 `previewRef/previewInView`는 스케줄러 잔여 섹션에 남김). 기존 `previewInView` 조건부 트랜지션 클래스 구조를 새 ref 이름으로 **그대로 복제** — 애니메이션 로직 변경 금지.
- ⑦ Meet Kiddy는 `kiddyRef/kiddyInView` 그대로 들고 이동 (ref 선언 위치는 무관, JSX 순서만 이동).
- 이동 시 **삭제되는 줄이 없어야 한다** — diff에서 삭제(−)는 이동 원위치의 것과 1:1 대응하는지 검토.
- 섹션 배경색 교차(어두움 `#0A1E1E` ↔ 밝음 `#0E2A2A`)가 이동 후 연속 두 섹션 같은 색으로 붙으면 배경색만 스왑 가능 (스타일 조정은 이 한정으로 허용).
