# [보충 지시서] N-2 — 배지 개편 보고 3건에 대한 팀장 결정 (N 이어서 진행)

> N 작업 중 "멈추고 보고"한 3건에 대한 **팀장 확정 답변**을 코드 앵커와 함께 정리. 이대로 N을 마무리하면 된다.

---

## 1. 미니게임 배지 — ✅ 조건부 승인 → 구현

**판단 근거(팀장):** 제거한 건 '화면 소비량 보상'이지 빈도 자체가 아님(마음 개근왕도 체크인 '빈도'). 미니게임은 시청의 반대편(배움·마무리 리추얼). 단 game_bonus가 시청시간을 늘려주는 구조라 "많이 하기" 유인이 되면 안 됨.

**조건 3개 (그대로 지킬 것):**
1. **누적 1회성 마일스톤 1개만** — 통산 10판. ⚠️ 10/50/100판식 사다리 **금지**(그라인딩 루프).
2. **카피는 배움·성취 프레임** — 가칭 **"놀이 척척박사"** 승인(최종 카피는 팀장 검수 — 보고 시 최종안 목록에 포함). ⚠️ "시간을 벌었어요" 류 표현 **절대 금지**.
3. **최소 스코프 트리거** — 기존 game_bonus 저장 성공 지점([MiniGame.jsx:153](../client/src/pages/MiniGame.jsx#L153) 부근)에 `checkBadges` 호출 한 줄만. **획득 즉시 팝업은 생략 가능** — 컬렉션 화면에 뜨면 충분. 팝업 인프라 이식이 무거우면 하지 마라.

**판정 데이터 (컨트롤 타워 확인):** `game_bonus` 테이블에 한 판 완료마다 행 기록됨(`POST /game-bonus`, [game_bonus.py:73](../server/routers/game_bonus.py#L73)). → [badges.py `check_badges`](../server/routers/badges.py#L294)에서 해당 프로필의 game_bonus **행 수만** 조회해 클로저 주입(favorites/searches 패턴), `>= 10` 판정.

```python
{
    "id": "game_scholar",          # id 가칭 — 자유
    "name": "놀이 척척박사",        # 최종 카피는 팀장 검수 대상
    "emoji": "🧩",
    "description": "미니게임을 열 판이나 해냈어요!",  # 배움·성취 프레임. '시간' 언급 금지.
}
```

## 2. 약속 지킴이 — ⏸️ 보류 확정 (고도화 1개월차로 이관)

- 판정 데이터 없음 + D-13에 새 테이블 = 스코프 크리프. **history 어림 계산으로 때우는 것도 금지** — 오판정 배지는 "정직함이 무기" 원칙과 자기모순.
- 고도화 1개월차에 '준수 이벤트 로깅 설계'와 함께 제대로 구현 예정. → **코드 작업 없음.** (신청서 '향후 계획' 재료로만 — 컨트롤 타워가 로드맵 문서에 기록)
- 신설 서사는 **마음 개근왕 + 놀이 척척박사** 둘로 충분.

## 3. 기존 획득 데이터 — 오너가 SQL 실행 후 분기

**Freddie 액션:** Supabase 대시보드에서 실행 → 결과를 컨트롤 타워에 전달:
```sql
-- 제거 대상 배지의 획득 행 (배지별 개수 + 어느 프로필인지)
select badge_id, profile_id, count(*) from badges
where badge_id in ('sprout_explorer','watch_master','attendance_king',
                   'early_bird','evening_explorer','channel_regular','fav_master')
group by badge_id, profile_id;
```

**분기 (팀장 확정):**
- **0건** → 상황 종료. 정의 주석처리로 N 완결.
- **있음** → 전부 테스트 데이터 전제, **삭제 승인.** 단 ⚠️ **데모 계정 컬렉션이 새 배지 체계와 일치해야 시연 영상이 깔끔** → 삭제는 **시드 재구성(R 브리프, 예정)과 같은 커밋**으로 처리. 지금은 삭제하지 말고 대기.
- **출처 불명 행 발견**(테스트가 아닐 가능성) → 삭제 말고 **재보고.**
