# [Claude Code 작업 지시서] I — 키디 음성 발음 자연화 (글은 "네가", 소리는 "너가")

> 문제: 화면 글자는 맞춤법대로 **"네가"**인데, CLOVA가 "네가"를 그대로 읽어 발음이 어색하고 "내가"와 혼동된다.
> 해결: **화면 글자는 "네가" 그대로 두고, TTS로 보내는 텍스트만 "너가"로** 바꾼다. 글↔말 분리.
> 원칙: "표기는 표준(네가), 발음은 자연스럽게(너가)." **화면 표시는 절대 건드리지 않는다.**
>
> ⚠️ **단순 `replace("네가","너가")` 금지.** 오탐 있음(동네가→동너가, 그네가→그너가, 언니네가→언니너가). 아래 어절 경계 규칙대로.
> **막히면 임의로 우회하지 말고 멈추고 보고.**

---

## 0. 확정 사실 (실제 코드 검증됨)

- **삽입 지점:** `server/services/tts.py`의 `synthesize()` 함수, `clean = strip_emoji(text)`(73줄) **직후**, `if not clean:`(75줄) 앞.
- **왜 이 한 곳이면 되나:** `synthesize()`는 **모든** 키디 대사가 CLOVA 합성으로 가기 전 거치는 유일 깔때기다.
  - 고정 대사: `client/src/utils/kiddyLines.js` (인사·받아주기·공유질문 등)
  - LLM 생성 대사: `server/routers/checkins.py`의 react/greet 엔드포인트(Haiku)에서 "네가"가 자유롭게 생성됨
  - → 서버 한 지점에서 정규화하면 **고정+생성 대사 둘 다** 커버. (프론트에 넣으면 LLM 생성분이 새어 빠짐.)
- **화면 안 건드림 근거(검증됨):** 프론트 `Typewriter`는 원본 텍스트(kiddyLines 결과/스트리밍 원문)를 그대로 화면에 그린다. 음성용 텍스트는 `voice.speak(...)` → `synthesizeKiddyVoice` → `POST /tts/kiddy`로 **따로** 서버에 간다. 서버 정규화는 **mp3 바이트에만** 영향을 주고 React state·화면으로 되돌아오지 않는다. → **화면엔 "네가", 소리만 "너가".**
- **라우터(`routers/tts.py` 46줄)의 `strip_emoji`는 '빈 텍스트 체크'용**이라 정규화 불필요. 정규화는 `synthesize()` 한 곳만.

## 1. 구현 — `naturalize_for_tts()` 추가 (결정론적 규칙)

`server/services/tts.py`에 아래 함수를 추가하고, `synthesize()` 안에서 `strip_emoji` **직후** 호출한다.

```python
import re

# '네가' 앞이 한글이 아니고(문두·공백·따옴표·괄호 뒤), 뒤가 어절 끝일 때만 = 독립 2인칭 대명사.
# 앞 글자가 한글이면 '-네+가' 합성(동네가·그네가·언니네가)이므로 건드리지 않는다.
_YOU_SUBJECT = re.compile(r"(?<![가-힣])네가(?=\s|[.,!?~\"')\]]|$)")

def naturalize_for_tts(text: str) -> str:
    """음성 발음 자연화 — TTS로 보내는 텍스트 전용. 화면 표시엔 절대 쓰지 말 것.
    독립 2인칭 '네가' → '너가' (발음이 '내가'와 안 헷갈리게).
    '동네가/그네가/언니네가' 같은 -네+가 합성명사는 보존."""
    return _YOU_SUBJECT.sub("너가", text)
```

`synthesize()` 안:

```python
    clean = strip_emoji(text)
    clean = naturalize_for_tts(clean)   # ← 추가: 발음 자연화(화면 미영향, 음성만)
    if not clean:
        raise TTSConfigError("합성할 텍스트 없음")
```

- ⚠️ **이 함수는 TTS로 보내는 텍스트에만.** 프론트·화면·kiddyLines엔 절대 적용 금지. (화면 글자 "네가"가 표준 맞춤법이다.)
- ⚠️ **이번 범위는 "네가"(주격)만.** 소유격 "네 "(네 편이야·네 마음 등)는 **이번에 건드리지 말 것** — 오탐 표면이 크다(네 살=4살, 네 개, 네모, 네 번…). 별도 결정 대기.
- 함수는 **확장 가능한 자리**로 둔다. 나중에 다른 "글↔말" 규칙이 생기면 여기에 추가.

## 2. 검증 (구현 후 반드시)

**정탐(바뀌어야 함):**
- `"네가 골랐지?"` → 음성 `"너가 골랐지?"`
- `'"네가 최고야"'`(따옴표 안) → `"너가 최고야"`
- 문두 `"네가 좋아하는 건 뭐야?"` → `"너가 좋아하는 건 뭐야?"`

**보존(절대 안 바뀌어야 함 — 오탐 회귀):**
- `"우리 동네가 좋아"` → 그대로 (동네가)
- `"그네가 재밌어"` → 그대로 (그네=놀이터)
- `"언니네가 왔어"`, `"친구네가 놀러 왔어"` → 그대로 (-네 집단접미사)

**회귀 체크(중요):**
- **화면 회귀:** 화면에 표시되는 키디 대사가 여전히 **"네가"** 인지(정규화가 프론트로 안 샜는지). 화면이 "너가"로 바뀌면 실패.
- **폴백 회귀:** 키 미설정·빈 텍스트 → 204, CLOVA 오류 → 502 가 **그대로** 동작하는지(정규화 추가가 기존 폴백 경로를 안 깼는지).

**값 조정:**
- "너가"가 귀에 어색하면 "니가"로 — 함수 안 치환값 한 곳(`"너가"`)만 바꾸면 됨. 데모 때 들어보고 결정.
