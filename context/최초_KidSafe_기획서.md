# KidSafe 세부 기획서
**AI Media Guardian for Kids**
> 작성일: 2026년 5월 26일 | 개발 기간: 3주 | 개발자: 1인

---

## 1. 프로젝트 개요

### 한 줄 소개
> AI가 어린이 콘텐츠의 안전도를 실시간으로 검수하고, 아이의 연령과 관심사에 맞는 영상을 추천해주는 웹 플랫폼

### 기획 배경
YouTube Kids 등 기존 어린이 콘텐츠 플랫폼은 이미 검수된 라이브러리를 제공하는 방식이라, 실시간 안전도 분석이나 항목별 점수 시각화 기능이 없다. KidSafe는 LLM 기반 실시간 검수 + 안전도 점수 시각화 + 키워드 기반 맞춤 추천을 결합해 기존 서비스와 차별화한다.

### 타겟 사용자
- 1차: 부모 (7~10세 자녀를 둔 학부모)
- 2차: 아이 (5~10세)

### 핵심 차별점
| 기능 | 기존 서비스 | KidSafe |
|------|------------|---------|
| 콘텐츠 필터링 | 사전 검수된 라이브러리 | LLM 실시간 검수 |
| 안전도 점수화 | 없음 | 항목별 점수 + 시각화 |
| 키워드 맞춤 추천 | 없음 | 있음 |
| 부모 대시보드 | 기본 수준 | 시청 분석 + 알림 |

## 2. 기능 명세

### 2-1. AI 검수 엔진 (핵심)
- **입력**: 유튜브 영상 제목, 설명, 썸네일 텍스트
- **처리**: Anthropic API (Claude) 호출 → 항목별 분석
- **출력**: 안전도 점수 (0~100) + 항목별 세부 점수 + 한 줄 요약

#### 검수 항목
| 항목 | 설명 |
|------|------|
| 폭력성 | 공격적 장면, 위험 행동 묘사 여부 |
| 언어 | 비속어, 부적절한 표현 포함 여부 |
| 선정성 | 성적 표현, 과도한 노출 여부 |
| 교육성 | 학습 가치, 창의성 포함 여부 |

#### 안전도 등급
- 90점 이상: 안전 (초록)
- 70~89점: 주의 (노랑)
- 70점 미만: 위험 (빨강)

---

### 2-2. 키워드 기반 콘텐츠 추천
- 아이가 키워드 입력 (예: "공룡", "우주", "동요")
- YouTube Data API v3로 관련 영상 검색
- AI 검수 엔진이 결과를 안전도순 정렬
- 안전도 70점 미만 영상은 자동 필터링

---

### 2-3. 부모 트랙 (Parent Mode)

#### 대시보드
- 오늘 시청 시간 / 평균 안전도 / 시청 영상 수 요약 카드
- 시청 기록 리스트 (영상명, 시청 시간, 안전도 배지)
- 항목별 안전도 막대 그래프

#### 연령 설정
- 3세 / 5세 / 7세 / 10세 프리셋
- 연령별 필터 강도 자동 조정 (7세 이하: 90점 미만 차단)

#### 시청 시간 관리
- 일일 한도 설정 (슬라이더, 15분 단위)
- 한도 도달 시 알림 표시
- 취침 전 시간대 잠금 (선택)

---

### 2-4. 아이 트랙 (Kid Mode)

#### 메인 화면
- 친근한 캐릭터 AI 친구 (텍스트 챗봇 UI)
- 키워드 입력창 ("어떤 영상 볼까?")
- 오늘의 추천 콘텐츠 카드 3개

#### 콘텐츠 카드
- 썸네일 + 제목 + AI 한 줄 요약
- 안전도 마스코트 표시 (별 개수)
- 클릭 시 YouTube 영상 링크 열기

#### 배지 시스템
- 공룡 박사 / 우주 탐험가 / 음악 천재 등 주제별 배지
- 영상 시청 완료 시 배지 포인트 적립
- 배지 컬렉션 화면

## 9. 이 프로젝트를 만든 이유 (개발 동기)

### 문제 인식
유튜브는 전 세계에서 가장 많이 쓰이는 영상 플랫폼이다. 그리고 대한민국 아이들의 하루 평균 영상 시청 시간은 2시간을 넘는다. 문제는 부모가 아이 옆에 항상 있을 수 없다는 것이다.

YouTube Kids가 있지만, 이 플랫폼은 두 가지 한계가 있다.

첫째, **사전 검수된 콘텐츠만 노출**한다. 즉, 새로운 콘텐츠가 등록되어도 검수 전까지는 보여주지 않는다. 빠르게 변하는 콘텐츠 생태계를 따라가기 어렵다.

둘째, **왜 안전한지 설명해주지 않는다.** 부모 입장에서는 "이 영상이 왜 안전한가?"를 알 수 없다. 그냥 허용되거나 차단될 뿐이다.

### 해결 아이디어
> "AI가 영상을 보고, 부모 대신 판단해주면 어떨까?"

단순한 키워드 필터링이 아니라, LLM이 영상의 맥락과 내용을 이해하고 **항목별 이유를 설명해주는** 검수 시스템을 만들면 부모의 신뢰를 얻을 수 있다고 생각했다.

## 10. KidSafe만의 유일성 (Uniqueness)

### 시장에 없는 것들

| 유일한 요소 | 설명 |
|------------|------|
| **LLM 실시간 맥락 검수** | 단어 필터링이 아닌 문맥 이해 기반 분석. "싸움" 장면도 교육적 맥락이면 안전으로 분류 |
| **항목별 이유 설명** | "폭력성 92점 — 공격적 표현 없음" 처럼 점수의 근거를 부모에게 제공 |
| **투 트랙 UI 분리** | 부모 화면과 아이 화면을 완전히 다른 UX로 설계. 기존 앱은 하나의 UI를 공유 |
| **연령별 필터 강도 자동 조정** | 3세는 90점 이상만, 10세는 70점 이상 허용 — 연령에 맞는 기준 자동 적용 |
| **학습 배지 시스템** | 콘텐츠 시청을 게임처럼 만들어 교육적 동기 부여. 안전한 콘텐츠를 더 보고 싶게 만드는 구조 |

### 포지셔닝 한 줄 요약
> "YouTube Kids는 콘텐츠를 필터링한다. KidSafe는 콘텐츠를 이해하고 설명한다."

## 11. KidSafe의 강점 (Strengths)

### 기술적 강점

**① AI를 도구로 제대로 활용했다**
단순히 "AI로 만든 서비스"가 아니라, AI가 없으면 구현 불가능한 핵심 기능(맥락 기반 검수)을 중심에 놓았다. LLM의 언어 이해 능력을 실제 문제 해결에 연결한 설계다.

**② 풀스택 경험을 자연스럽게 보여준다**
React (UI) + Node.js (서버) + 외부 API 2개 (Anthropic + YouTube) 를 유기적으로 연결했다. "이 기술을 왜 썼는가?"에 대한 답이 설계 안에 녹아있다.

**③ 데이터를 시각화했다**
점수를 숫자로만 보여주지 않고, Recharts를 활용해 막대그래프·배지·게이지 형태로 표현했다. 사용자 경험과 개발 역량을 동시에 어필할 수 있다.

### 비즈니스·사회적 강점

**④ 명확한 사용자 페인포인트를 해결한다**
"내 아이가 어떤 영상을 보는지 모른다"는 부모의 실제 불안을 직접 해소한다. 면접에서 "왜 이걸 만들었어요?" 질문에 한 문장으로 답할 수 있다.

**⑤ 확장 가능한 구조다**
현재는 YouTube 기반이지만, 검수 엔진은 어떤 콘텐츠 소스에도 붙일 수 있다. 넷플릭스, 네이버 TV, 카카오TV 등으로 확장 가능하다는 점을 면접에서 어필할 수 있다.

**⑥ 사회적 가치가 있다**
어린이 미디어 리터러시는 전 세계적으로 주목받는 이슈다. 단순한 토이 프로젝트가 아닌, 실제 사회 문제를 해결하려는 의도가 담긴 프로젝트라는 점에서 면접관의 기억에 남는다.

### 포트폴리오로서의 강점

**⑦ 스토리가 있다**
문제 발견 → 기존 서비스 분석 → 차별화 포인트 도출 → 기술 선택 → 구현의 흐름이 자연스럽다. 포트폴리오는 결과물만큼이나 "왜 이렇게 만들었는가"의 스토리가 중요하다.

**⑧ 혼자 3주 만에 완성한 프로젝트다**
기획부터 배포까지 1인 개발로 완성했다는 사실 자체가 실행력과 문제 해결 능력을 증명한다.

## 12. 면접 예상 Q&A

**Q. 왜 이 프로젝트를 만들었나요?**
> 아이들의 영상 시청은 늘어나는데, 부모가 모든 콘텐츠를 확인할 수 없다는 현실적인 문제에서 출발했습니다. 기존 서비스들이 '허용/차단'만 할 뿐 이유를 설명해주지 않는다는 점에 착안해, AI가 맥락을 이해하고 부모에게 근거를 제공하는 서비스를 기획했습니다.

**Q. YouTube Kids와 뭐가 다른가요?**
> YouTube Kids는 사전 검수된 라이브러리를 제공하고, 왜 안전한지 설명하지 않습니다. KidSafe는 LLM이 실시간으로 영상의 맥락을 분석하고, 항목별 점수와 이유를 부모에게 시각적으로 제공합니다.

**Q. AI를 어떻게 활용했나요?**
> Anthropic API를 통해 영상 제목과 설명을 입력하면, 폭력성·언어·선정성·교육성 4가지 항목을 분석해 0~100점으로 점수화합니다. 단순 키워드 매칭이 아니라 문맥을 이해하는 LLM의 특성을 활용했습니다.

**Q. 어렵거나 힘들었던 부분은요?**
> YouTube API와 Anthropic API를 동시에 연동하면서, 검색 결과를 실시간으로 검수하고 정렬하는 플로우를 설계하는 것이 가장 어려웠습니다. API 호출 비용과 응답 속도를 고려해 캐싱 전략을 적용하는 과정에서 많이 배웠습니다.

**Q. 더 발전시킨다면요?**
> 현재는 영상 메타데이터 기반 검수이지만, 실제 영상 내용(자막, 음성)을 분석하는 방향으로 고도화할 수 있습니다. 또한 교사용 교실 모드나, 콘텐츠 제작자가 업로드 전에 사전 검수받는 B2B API도 구상하고 있습니다.
