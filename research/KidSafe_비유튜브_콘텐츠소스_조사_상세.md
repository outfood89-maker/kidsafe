## 1. 개요 — 조사 방법·규모

### 1-1. 무엇을 왜 조사했나
KidSafe(한국 4~7세 아동용 "아이의 첫 미디어 경험 공간" 앱, 현재 YouTube Data API 기반 영상 검수 + AI 캐릭터 '키디' 데일리 대화 + 부모 주간 감정 리포트, 초기 1인 개발자)가 **유튜브 외 아동 콘텐츠 소스**를 도입할 수 있는지, 세 축으로 조사했다.
- **축 1** — 유튜브 대체 vs 보완 (트레이드오프)
- **축 2** — 콘텐츠 종류별(동화·그림책 / 학습·교육 / 음악·동요 / 오디오) 실제 소스와 접근 방식(공개 API·제휴·오픈 라이선스·스크래핑 불가), 국내외 모두
- **축 3** — 라이선스·저작권·수익모델·규제(CC·퍼블릭도메인·공공누리·OER, 상업 라이선싱/신디케이션, COPPA·한국 아동 규제·임베드 정책)

## 2. 축 1 — 유튜브 대체 vs 보완 (상세)

### 2-1. "완전 대체"가 1인 개발자에게 비현실적인 이유
유튜브의 강점은 **콘텐츠 양(무한)·검색·무료**다. 단일 소스로 대체하려면 (a) 방대한 카탈로그, (b) 스트리밍 인프라, (c) 라이선스 정리를 전부 자체 확보해야 하는데 1인 개발자엔 불가능에 가깝다. **[검증됨]** 다만 "영상"을 고집하지 않고 콘텐츠 종류를 그림책·오디오·(부분적으로)영상 큐레이션으로 넓히면, 유튜브 없이도 "첫 미디어 경험"을 구성할 오픈 소스가 존재한다.

### 2-2. 대체형 후보로 발견됐으나 요약본에서 빠진 것들
요약본은 "완전 대체는 비현실적"으로 결론짓고 대체형 실제 사례를 거의 지웠다. 그러나 fetch 단계에서는 아래 두 개의 **대체형 실제 사례**가 실제로 추출됐다(둘 다 검증 컷오프에 밀림):

- **TKSST — The Kid Should See This (thekidshouldseethis.com)** **[상세본 보강확인]**
  - 정체: **사람이 직접 큐레이션한** "호기심 자극 스마트 영상" 컬렉션. 알고리즘이 아니라 교육 미디어 큐레이터(본인이 엄마)가 한 편씩 선별. Webby 수상. 8,300편+ (초기 추출 스냅샷은 7,000편+).
  - KidSafe 관련성: **"알고리즘이 아닌 사람 큐레이션"** 이라는 편집 모델의 벤치마크. KidSafe의 "검수·큐레이션" 정체성과 철학이 같다.
  - 접근 방식: 웹사이트/뉴스레터 형태. 영상 자체는 YouTube·Vimeo 등에서 **임베드**(자체 호스팅 아님). **공개 API는 확인되지 않음** → 콘텐츠 소스로 프로그램 연동은 어렵고, 벤치마크/큐레이션 참고용.
  - 라이선스·비용: 사이트 자체는 큐레이션 저작물. 개별 영상 권리는 원저작자에게 있음(임베드 경유).
  - 아동 안전: 사람 선별이라 품질 균질. 단 임베드 소스가 유튜브면 유튜브 임베드 제약(광고·추적) 그대로 상속.
  - 티어: ①즉시(벤치마크·큐레이션 참고) / 콘텐츠 소스로는 ②(연동 불명확).
  - 출처: thekidshouldseethis.com

- **Kidoodle.TV** **[미검증·fetch추출 / blog 출처]**
  - 정체: **완전 무료(광고 기반)** 12세 이하 아동 스트리밍. 부모+공인 교육자가 참여하는 큐레이션/모니터링 시스템으로, "YouTube Kids의 부모통제 강화판"으로 포지셔닝.
  - KidSafe 관련성: "무료+부모통제+큐레이션"이라는 **유튜브 키즈 대안 스트리밍**의 실제 사례. 다만 광고 기반이라 KidSafe의 "광고 없음" 정체성과는 상충 가능.
  - 접근 방식: 완성된 앱/서비스(직접 붙일 콘텐츠 API 아님) → **경쟁·벤치마크 사례**.
  - 라이선스·비용: 시청자 무료(광고 수익 모델로 추정).
  - 아동 안전: "Safe Streaming™" 표방, COPPA 준수 마케팅.
  - 출처: zugucase.com 블로그(2차, 낮은 신뢰도) — 사실 재확인 필요. **[불확실]**

> **정리:** 완전 대체형에서도 "사람 큐레이션(TKSST)"과 "무료 키즈 스트리밍(Kidoodle)"이라는 실제 모델이 존재한다. 다만 전자는 콘텐츠 소스라기보다 **큐레이션 철학 벤치마크**, 후자는 **경쟁 서비스**로 봐야 한다.

### 2-3. 유튜브 유지 + 보완(+α) — KidSafe에 가장 현실적 **[검증됨]**
유튜브를 영상 축으로 유지하되 그림책·오디오·동요를 별도 소스로 애그리게이션. 근거: (1) **안전성** — 오픈 소스는 광고 없고 콘텐츠 고정이라 유튜브 임베드의 광고·알고리즘 리스크 우회(유튜브는 API 클라이언트의 광고 제거·차단을 금지). (2) **차별화** — 키디가 그림책/오디오를 함께 읽어주는 경험이 정체성(B)에 직접 기여. (3) **비용/난이도** — 오픈 라이선스는 계약·수수료 없음.

### 2-4. 접근별 트레이드오프
| 접근 | 콘텐츠 양 | 안전성 | 비용 | 기술 난이도 | 차별화 |

| 유튜브 완전 대체(자체 카탈로그) | 낮음 | 높음 | 높음 | 높음 | 높음 |
| 유튜브 유지 + 오픈 소스 보완 | 높음 | 중~높음 | 낮음 | 중간 | 높음 |

  - 출처: storyweaver.org.in/about · /en/prathambooks · /en/story-weaver-and-you (원 라이선스 문구는 prathambooks.org/cc 에서 검증)

### 3-B. 부차·미확정 소스 (요약본에서 빠진 "애매했던 놈들")

> 아래는 fetch에서 추출됐으나 검증 컷오프에 밀린 소스들. 이 상세본에서 개별 WebFetch로 사실 확인했다.

#### African Storybook (afrikanstorybook / Saide, 남아공) **[상세본 보강확인]**
- **정체:** 아프리카 언어권 아동용 **오픈 라이선스 그림책** 디지털 아카이브. 사용자가 읽기·창작·다운로드·번역·각색 가능. StoryWeaver와 유사한 "오픈 그림책 프로젝트" 계열.
- **규모:** 2023년 3월 기준 **원작 3,800종 + 번역 7,266종, 236개 언어**. (StoryWeaver보다 작지만 "번역/각색 워크플로"가 강함.)
- **라이선스:** Creative Commons 기반 오픈 라이선스(위키 서술). 자유 접근·수정 허용. ⚠️ 개별 스토리 라이선스는 African Storybook 사이트에서 재확인 권장.
- **운영:** 남아공 비영리 **Saide**(요하네스버그). 창립자 Tessa Welch(2020년 별세). Comic Relief 등 후원.
- **KidSafe 관련성:** 콘텐츠 자체보다 **한국어 무관**(236개 언어에 한국어·아시아어 없음, 아프리카 언어 중심)이라 **직접 도입 가치는 낮음**. 대신 "저자원 언어 그림책을 오픈 라이선스로 대량 확보하는 모델"의 참고 사례.
- **접근성 티어:** ①즉시(오픈 라이선스) — 단 한국어 콘텐츠 부재로 실효성 제한.
- 출처: en.wikipedia.org/wiki/African_Storybook

#### Epic! (getepic.com) **[상세본 보강확인]**

### 4-2. 공공누리(KOGL) — 제1~4유형 차이 (요약본에서 빠진 핵심)

**공공누리(KOGL, Korea Open Government License)** 는 저작권법 제24조의2에 근거해 국가·지자체·공공기관 보유 저작물(사진·영상·음원·보고서 등)을 국민이 자유이용하도록 하는 **한국형 오픈 라이선스**. 문체부 산하 **한국문화정보원**(및 한국저작권위원회 연계)이 운영. **[미검증·fetch추출 + 상세본 보강확인]**

| 유형 | 출처표시 | 상업적 이용 | 변경/2차저작 | 요지 |
|---|---|---|---|---|
| **제1유형** | 필수 | **허용** | **허용** | 가장 자유(≈CC-BY). 출처만 표기하면 상업·변형 자유 **[보강확인]** |
| **제2유형** | 필수 | **금지** | 허용 | 비상업 한정, 변형은 가능 |
| **제3유형** | 필수 | 허용 | **금지** | 상업 가능하나 원형 유지(변경 불가) |
| **제4유형** | 필수 | **금지** | **금지** | 가장 제한적. 비영리+원형유지. 단 **별도 이용허락 받으면 상업 가능** **[보강확인]** |

- 제1유형 원문(요지): "이용자는 … **상업적 활용 여부에 관계없이 무료로 자유롭게 이용**", "2차적 저작물로 **변경하여 이용 가능**". (단 저작인격권 존중 — 왜곡·명예훼손 변형 금지) **[보강확인: kogl.or.kr/info/licenseType1.do]**
- 제4유형 원문(요지): "저작물의 출처를 표시", "**비영리 목적으로만** 이용", "저작물을 **변경하거나 2차적 저작물 등으로 작성할 수 없습니다**", 다만 "별도의 이용허락을 받아 … 상업적으로 이용하는 것은 가능". **[보강확인: kogl.or.kr/info/licenseType4.do]**

> **KidSafe 실무 함의:** 국내 공공저작물을 상업 서비스(KidSafe)에 넣을 수 있는지는 **KOGL 유형이 결정한다.** **제1유형 = 그대로 상업 사용 가능**, **제2·4유형 = 상업 불가(별도 허락 필요)**, **제3유형 = 상업은 되나 원형 변경 불가.** 소스마다 유형 배지를 확인해야 하며, "공공누리니까 다 자유"라고 뭉뚱그리면 안 된다.

### 4-3. 공유마당(gongu.copyright.or.kr) — 국내 공공저작물 실물 창고 **[상세본 보강확인]**
- **정체:** **한국저작권위원회**(문체부 산하) 운영 공공저작물 공유 플랫폼.
- **콘텐츠 유형:** 이미지(미술·사진), 영상, **음악(음원)**, 어문(텍스트), 글꼴, 3D, **전자책(e-Book)** — 미디어 유형이 넓다.
- **라이선스 체계(3종):** ① **공공누리(KOGL)** 1~4유형, ② **CCL**(크리에이티브 커먼즈), ③ **만료저작물**(저작권 만료 = 퍼블릭 도메인).
- **상업 이용:** 라이선스 유형에 따라 다름(KOGL 제1유형=상업 OK, 제2·4=상업 금지).
- **KidSafe 관련성:** 국내에서 "**본문/실물 콘텐츠(음원·전자책·삽화)**"를 무료로 구할 수 있는 몇 안 되는 창구. 특히 §3-B의 "국내 동요 부재" 문제를 부분 해결할 잠재 후보(음악 카테고리). ⚠️ **아동 특화 콘텐츠가 실제로 얼마나 있는지, 각 콘텐츠 KOGL 유형이 무엇인지는 개별 검색 필요.** **[불확실 — 추가 실사]**
- 출처: gongu.copyright.or.kr/gongu/main/contents.do?menuNo=200097

### 4-4. 상업 라이선싱/제휴 모델

### 4-5. 규제 — 유튜브 임베드 의무 (Google 공식 정책) **[검증됨]**
KidSafe가 유튜브를 API/임베드로 계속 쓰면 아래가 하드 의무:
1. **Made-for-Kids 상태 조회 의무:** 임베드하는 각 영상의 MFK 상태를 `videos.list`의 `status.madeForKids`로 조회하고, MFK 영상은 **추적 오프 + 모든 데이터 수집을 COPPA·GDPR 준수**. (API 문서 원문: MFK 영상 임베드 시 "turn off tracking … compliant with … COPPA"). **[보강확인: made_for_kids_status 가이드]**
2. **Child-Directed 통지 + 광고 제약:** 자녀 대상 API 클라이언트임을 Google에 통지, **개인화·리마케팅 광고 타게팅 금지**.
3. **쓰기 액션 전면 금지:** 업로드·댓글·재생목록 생성/공유 등 write 액션을 앱도 사용자도 수행/유도 불가.
4. **플레이어·광고 개조 및 게이팅 금지:** 플레이어의 어떤 부분도 수정·차단·재구축 불가, 광고 수정·차단·대체 불가, **재생 버튼 외 행동을 요구하는 접근 게이팅 금지** → **광고 제거·플레이어 교체·시청 전 강제 절차 삽입 불가.**
5. **[신규] AV 콘텐츠 추출·재배포 불가:** YouTube API 약관은 "API 서비스를 통하지 않고 시청각 콘텐츠를 복제·배포할 권리를 부여하지 않음" → **유튜브 영상을 플레이어 밖으로 빼내 독립 재배포 불가.** **[미검증·fetch추출 / api-services-terms-of-service]**

> **시사점:** 유튜브 임베드로는 "광고 없는 순수 안전 경험"을 구조적으로 만들 수 없다 → 광고 없는 비유튜브 오픈 소스 병행이 필요.

### 4-6. 규제 — 한국 아동 개인정보 (요약본에서 빠진 국내 규제)

KidSafe는 4~7세 아동 + 부모 리포트라 **아동 개인정보 처리자**다. 다음이 직접 적용된다:

- **개인정보보호법 제22조의2(아동의 개인정보 보호)** **[상세본 보강확인]**
  - 제1항: **만 14세 미만** 아동 개인정보 처리에 동의가 필요하면 **법정대리인(부모)의 동의**를 받고 동의 여부를 확인해야 함.
  - 제2항: 법정대리인 동의를 받기 위한 최소 정보(법정대리인 성명·연락처)는 동의 없이 아동에게서 직접 수집 가능.

## 6. 반박된 주장 상세 (killed 2개)

3중 적대검증에서 **2건이 반박(1-2 표결, 즉 3명 중 2명이 반박)** 됐다.

### 반박 ① — Gutenberg "아동 고전 인기 다운로드 직접 노출"
- **원 주장:** "퍼블릭 도메인 아동 고전(이상한 나라의 앨리스, L.M. 몽고메리 *The Blue Castle* 등)이 **인기 다운로드 목록에 직접 올라 있어** 아동 콘텐츠 앱에 바로 쓸 수 있는 PD 소스다."
- **왜 반박됐나:** 검증자들이 "**인기 다운로드 목록에 아동 고전이 직접 노출된다**"는 **구체적 노출 형태**를 확인하지 못함. Gutenberg에 아동 고전이 **존재**하는 것은 사실이나, "top downloads에 아동물이 직접 걸려 있다"는 세부는 근거 부족. (*The Blue Castle*은 아동물로 보기도 애매.)
- **올바른 이해:** Gutenberg에 아동 적합 PD 텍스트가 **있긴 하다.** 그러나 "인기 목록을 그대로 아동 카탈로그로 쓸 수 있다"고 단정하면 안 되고, **아동 적합작을 개별 선별**해야 한다. 또 텍스트라 삽화·오디오화 가공이 필요하다.
- 출처: gutenberg.org

### 반박 ② — 국립어린이청소년도서관 API "라이선스 범위 무제한"
- **원 주장:** "이 OpenAPI는 무료이고 **이용 라이선스 범위에 제한이 전혀 없어** 소규모 개발자가 라이선스료·계약 없이 바로 상업 이용할 수 있다."
- **왜 반박됐나:** "라이선스 범위에 **제한이 없다**"는 부분이 근거 부족. data.go.kr 데이터셋은 **이용약관/라이선스 조건**이 붙는 경우가 많고, "무제한 상업 재사용"은 확인되지 않음.
- **올바른 이해:** API가 **무료로 제공되는 것은 맞지만**, "제한 없음"은 아님. **data.go.kr 실제 데이터셋 페이지에서 이용약관·라이선스 범위를 재확인**해야 하며, 이것이 **서지 메타데이터**라는 점(본문 아님)도 함께 명심. 무제한 상업 재사용 단정 금지.
- 출처: data.go.kr/data/15104976

### 7-2. urlDupes 3 — 중복 URL로 걸러진 것
검색 각도별로 동일 URL이 여러 번 나와 **3건이 중복 제거**됐다(예: StoryWeaver가 broad/practitioner/open-license/case-study 여러 각도에서 반복 등장). 콘텐츠 손실은 아니며 **동일 소스의 중복 카운트를 제거**한 것. 다만 이는 "StoryWeaver가 여러 각도에서 최상위로 반복 추천됐다"는 **신호 강도**로 해석 가능(그만큼 적합).

- **국내 상용 그림책/동요/에듀테크(웅진·교원·핑크퐁/스마트스터디 등)** 는 fetch 대상에 없었음 → 국내 상용 IP·제휴 조건 전면 미조사.

2. **Book Dash = 보조 그림책.** CC-BY 원본 파일로 한국어 리레벨링 가능.
3. **LibriVox = "자기 전 오디오" 축.** API 쉬움·무료. 아동 적합작 선별 + 국내 저작권 확인.
4. **공유마당(KOGL 제1유형) = 국내 실물 콘텐츠 후보.** 특히 **국내 동요/음원 공백을 메울 유일한 무료 창구 후보**. 단 KOGL 유형·아동물 규모 개별 확인 필요.
5. **국립도서관/Open Library 메타데이터 = 큐레이션 신뢰.** 사서 추천으로 "무엇을 보여줄지" 정당화 → 부모 신뢰. 본문 아님 명확화.
6. **TKSST = 큐레이션 철학 벤치마크**(사람이 고른다). Kidoodle·Epic!·Yoto = 경쟁/포지셔닝 벤치마크.
7. **유튜브 = 영상 축 유지 + §4-5 의무 이행.** 광고 없는 경험은 오픈 소스로 보완.
8. **규제 준수 설계:** 만 14세 미만 법정대리인 동의, 알기 쉬운 언어, 타게팅 광고 최소화, 다크패턴 금지, privacy-by-design (§4-6).

**추천 로드맵:** 1단계(지금) StoryWeaver 한국어 그림책 + 키디 낭독/질문 → 2단계 Book Dash + LibriVox 아동 오디오 선별(국내 저작권 확인) → 3단계 사서추천/Open Library 메타데이터로 큐레이션 신뢰 + **공유마당 국내 동요/음원 실사** → 4단계(제휴) 국내 상용 동요/그림책(티어②), 장기 브랜드 제휴(티어③).

## 10. 불확실·주의(Caveats) + 남은 질문 (전체)

### 10-1. Caveats
- **반박 ①:** Gutenberg 아동 고전 "인기 다운로드 직접 노출" 반박 → 아동 적합작 **개별 선별**(§6).
- **반박 ②:** 국립도서관 OpenAPI "라이선스 무제한" 반박 → data.go.kr **약관 재확인**, 무제한 상업 단정 금지, 메타데이터임(§6).
- **퍼블릭 도메인 관할:** LibriVox·Gutenberg PD는 **미국 기준.** 한국(사후 70년 등)에서 보호 대상일 수 있어 **국내 저작권 상태 확인**(LibriVox가 사용자 책임 명시).
- **StoryWeaver 스토리별 라이선스** 개별 확인 / "Read Along" 영상 = CC-BY-NC-ND(상업 불가).
- **한국어 물량:** StoryWeaver 100권+ 보조적, African Storybook은 **한국어 없음** → 주력 한국어는 국내 소스 별도 확보.
- **메타데이터 vs 콘텐츠 혼동 금지:** Open Library·사서추천 = 서지, 공유마당·StoryWeaver·Book Dash·LibriVox = 실물.
- **KOGL 유형 확인 필수:** 제1유형만 상업·변형 자유. 제2·4유형은 상업 불가(별도 허락). "공공누리=다 자유" 금지.
- **상용/구독 소스 직접 소싱 불가:** Epic!·KITABOO·Yoto 브랜드 콘텐츠는 공개 API·오픈 라이선스 없음 → 벤치마크로만.
- **Storyberries 자체 콘텐츠는 앱 삽입 불가.** 파트너(Book Dash·StoryWeaver) 콘텐츠는 **원천에서 직접** 받을 것.
- **2차 출처 주의:** Kidoodle(zugucase 블로그)·Yoto Creators(kidscreen 기사)는 2차라 사실 재확인 필요.
- **원본 fetch 제약:** StoryWeaver·LibriVox HTML 페이지는 직접 fetch 시 403(봇 차단) → 공식 미러/독립 출처/API 응답으로 교차 검증. 본 상세본에서도 storyberries·gongu는 정상 fetch, kitaboo는 홈으로 우회, thekidshouldseethis는 403이라 WebSearch로 보강.

### 10-2. 남은 질문 (추가 실사)
1. **국내 상용 그림책/에듀테크 제휴 조건**(최소 계약·매출 배분·1인 접근성) — 미조사(핑크퐁/스마트스터디·웅진·교원 등).
2. **동요/키즈 음악 오픈 소스** — 확정 부재. **공유마당 음악 카테고리에서 국내 동요/전래 음원 KOGL 유형·규모 실사** 우선.
3. **공유마당·KOGL 아동 콘텐츠 실물 범위** — 제도만 확인, 실물 목록·유형별 분포 미확인.
4. **일반 OER(OER Commons·MERLOT)** 아동 콘텐츠 실물 — fetch 안 됨, 미확인.
5. **국내 저작권법상 미국 PD 작품의 한국 지위** — 작품별 사후 70년 등 법률 검토.
6. **국립도서관 OpenAPI 이용약관 세부**(상업 재사용 범위) — 재확인.
7. **아동 개인정보 준수 실무**(법정대리인 동의 UI, 만14세 미만 고지 언어, 데이터 최소수집) — KidSafe 부모동의/멀티테넌시 설계에 반영 실사.
8. **Kidoodle.TV 사실 재확인**(무료 모델·안전 기능·COPPA) — 2차 블로그뿐이라 1차 확인 필요.
9. **TKSST 콘텐츠 연동 가능 여부**(피드/API·재사용 허가) — 벤치마크 이상으로 쓸 수 있는지.
