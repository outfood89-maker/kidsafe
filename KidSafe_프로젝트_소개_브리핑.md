## 0. 한 줄 요약 (Executive Summary)

> **KidSafe는 "AI가 영상을 직접 검수해서, 아이에게 안전한 YouTube 영상만 보여주는" 어린이 미디어 안전·추천 플랫폼이다.**

부모는 안심하고 아이에게 영상을 맡기고, 아이는 캐릭터 친구('키디')와 함께 안전하게 영상을 찾아보고, 학습 게임으로 보너스 시청 시간을 번다. 단순 키워드 필터링을 넘어, **영상의 실제 내용(자막)을 AI가 분석**하고 그 결과를 캐싱해 **비용 없이 운영**하는 계층형 검수 아키텍처가 핵심 경쟁력이다.

- **형태**: 웹 앱 (모바일 우선 반응형)
- **프론트 배포**: https://kidsafe-eight.vercel.app

## 1. 왜 만들었나 (문제 정의 & 배경)

### 1.1 부모가 겪는 진짜 문제
- 아이에게 YouTube를 보여주고 싶지만, **알고리즘이 어떤 영상을 다음에 틀어줄지 통제할 수 없다.** 안전해 보이던 채널도 갑자기 자극적·폭력적·선정적 콘텐츠로 빠지는 경험이 흔하다.
- YouTube Kids가 있지만, **"왜 이 영상이 안전한지"에 대한 근거가 불투명**하고 부모가 세밀하게 통제하기 어렵다.
- 시청 시간 관리가 어렵고, 아이는 갑자기 영상이 꺼지면 떼를 쓴다.

### 1.2 KidSafe의 접근
1. **사전 검수**: 아이에게 보여주기 전에 AI가 영상 내용을 분석해 안전도 점수를 매긴다. 기준 미달 영상은 재생 자체를 막는다.
2. **투명한 근거**: 폭력·언어·선정성 등 항목별 점수와 AI 요약을 부모에게 보여줘 "왜 안전/위험한지"를 설명한다.
3. **부드러운 시간 관리**: 시청 시간이 끝나기 전 키디가 "10분 남았어, 슬슬 마무리하자"라고 **미리·친절하게** 알려준다. 학습 게임을 풀면 보너스 시간을 얻는 **긍정적 동기**를 설계했다.
4. **아이 친화적 경험**: 캐릭터 '키디'가 검색·검수·시청 전 과정을 안내해, 아이에게는 통제가 아니라 **친구와 함께하는 놀이**처럼 느껴지게 한다.

### 1.3 핵심 가치 (제품 의사결정의 기준)
> **"검색엔진 완성도 + 영상 분석 능력"** — 모든 기능 우선순위는 이 두 축을 기준으로 판단한다.

## 2. 무엇인가 (제품 개요 & 사용자)

KidSafe는 하나의 부모 계정 아래 **여러 아이 프로필**을 두고, 각 프로필마다 독립된 시청 환경·시간 제한·부모 페이지를 갖는 **멀티테넌시(다중 사용자)** 구조의 웹 앱이다.

### 사용자 유형
| 사용자 | 역할 | 주요 화면 |
|---|---|---|
| **아이** | 영상 검색·시청, 미니게임 플레이 | 아이 홈(KidHome), 영상 플레이어, 미니게임 |
| **부모** | 프로필·시간 제한 관리, 시청 분석 확인, AI 코치 받기 | 부모 대시보드(ParentDashboard), 계정 관리 |
| **관리자** | 검수 룰 운영, 회원·통계·감사 로그 관리 | 관리자 대시보드(AdminPage) |

### 사용 흐름 (한눈에)
```
로그인(부모 계정) → 아이 프로필 선택 → [아이 모드]
   ├─ 키디와 함께 영상 검색  →  AI 검수 → 안전한 영상만 카드로 노출
   ├─ 영상 시청 (시간 제한·남은시간 알림·연속재생 시 다음 영상 자동 검수)
   └─ 미니게임으로 보너스 시청 시간 획득
[부모 모드] (PIN 보호)
   ├─ 시청 기록·습관 분석(차트) 확인
   ├─ AI 코치에게 "우리 아이 시청 습관" 코칭 받기
   └─ 시간 제한·안전 기준·프로필 관리
```

## 3. 주요 기능 (사용자별 상세)

### 3.1 아이 경험
- **키디(Kiddy) 캐릭터 가이드**: 검색·검수·시청 로딩 등 모든 순간에 등장하는 마스코트. 정적 포즈 13종 + 투명 배경 애니메이션 영상(검색/대화/인사)으로 살아있는 느낌을 준다. 이름 받침에 맞춰 호격조사를 자동 처리("해인아"/"호이야")하는 디테일까지 적용.
- **안전한 영상 검색**: 음성 검색 지원, 검색어 히스토리, 차단 키워드 자동 필터링. 검색 결과는 모두 안전도 점수와 함께 카드로 노출.
- **영상 플레이어**:
  - 안전 기준 미달 영상은 **재생 게이팅**(차단).
  - **연속재생**: 부모가 켜두면 다음 영상을 자동으로 **사전 검수**한 뒤(통과 시) 재생, 미달이면 차단.
  - **시청 시간 관리 + 남은시간 알림**: 하루 제한 남은 시간을 분·초로 표시하고, 임계값(예: 10·5·1분)마다 키디가 부드러운 토스트로 미리 알림(영상은 멈추지 않음).
- **미니게임 6종**: OX 퀴즈, 수학 퀴즈, 분류 게임, 단어 맞추기, 퍼즐, 메모리 카드. **게임 한 판 완료 시 +3분** 보너스 시청 시간(하루 상한 적용). 학습과 보상을 연결.
- **즐겨찾기·뱃지**: 좋아하는 영상 저장, 활동에 따른 뱃지 수집으로 재방문 동기 부여.

### 3.2 부모 경험
- **프로필별 부모 페이지(PIN 보호)**: 넷플릭스식으로 프로필마다 부모 영역을 분리해 프라이버시 보장.
- **시청 분석 대시보드**: 시청 기록을 조인·집계(pandas)해 안전도 분포(파이차트), 최다 시청 채널, 시간대 분포, 최근 7일 안전도 추이(라인차트) 등을 시각화.
- **AI 코치('키디 코치')**: Claude가 아이의 실제 시청 데이터를 바탕으로 "이번 주 시청 습관"을 분석하고 등급·코칭 멘트를 생성.
- **세밀한 통제**: 하루 시청 시간 제한, 안전도 기준(threshold) 설정, 연속재생 on/off, 프로필 추가·수정.

### 3.3 관리자 경험
- **검수 룰 운영**: 안전 검수에 쓰이는 예외·가산·감점 규칙을 데이터로 관리(코드 배포 없이 운영).
- **회원·권한 관리**: 프리미엄 권한 부여/해제, 관리자 지정.
- **통계 & 감사 로그**: 서비스 사용 통계, 관리자 행위 감사 로그(audit log) 기록.

## 4. 핵심 기술 ① — 영상 검수 파이프라인 (이 서비스의 심장)

KidSafe의 가장 큰 기술적 차별점. **"단계별로 거르고, 비싼 AI는 마지막에 딱 한 번만, 그리고 결과는 영구 캐싱"** 하는 계층형 구조다.

```
영상 검수 요청 (videoId)
   │
   ▼ [캐시 확인]  이미 분석한 영상인가?
   │   └─ 있음 → 저장된 결과 즉시 반환 (비용 0, 즉시) ✅
   │
   ▼ [Tier 0] 무료·즉시 — 공식 신호 + 차단 키워드
   │   • YouTube madeForKids 플래그 / 카테고리(교육·키즈)
   │   • 차단 키워드 매칭 → 명백한 위험이면 즉시 '위험' 확정
   │
   ▼ [Tier 1] 무료·가벼움 — 채널 신뢰도 + 통계 휴리스틱
   │   • 검증된 키즈 채널 화이트리스트 → 신뢰도 가산점
   │   • 채널 신뢰 임계값(예: 90+ 판정 N회) 충족 시 자동 신뢰 채널 등록
   │
   ▼ [Tier 2] AI·정밀 — 자막 + 메타데이터 → Claude
   │   • youtube-transcript-api로 자막 추출(없으면 메타데이터 기반)
   │   • Claude가 구조화된 안전 리포트 생성
   │
   ▼ [결과 캐싱]  videoId를 키로 영구 저장 → 같은 영상은 두 번 분석하지 않음
```

### 왜 이 구조인가
- **비용 절감**: AI 호출은 가장 비싼 단계라, 캐시·무료 계층에서 최대한 걸러 **실제 AI 호출 빈도를 최소화**한다. 한 번 분석한 영상은 모든 사용자가 캐시 결과를 공유.
- **속도**: 이미 본 영상은 즉시 응답.
- **정확도**: 키워드 매칭의 오탐을 AI의 문맥 판단으로 보완. (예: "납치"·"변태" 같은 동화 단골 단어나 동음이의어는 키워드로 단정하지 않고 Tier 2 AI가 문맥으로 판단)

## 5. 핵심 기술 ② — 안전도 점수 체계

영상은 **폭력 안전 / 언어 안전 / 선정성 안전** 3개 핵심 축으로 평가되며, 채널 신뢰도 가산점이 더해져 **총점(0~100점)** 이 결정된다. (교육성 등은 참고 지표로, 총점엔 미반영)

- **키워드 위험도 가중치**: 심각(-30) / 보통(-15) / 경미(-5)
- **총점 ≈ (폭력+언어+선정성 점수 평균) + 채널 신뢰 가산점** (상한 100)
- **오탐 방지 설계**: 한글 부분일치의 함정을 피하기 위해 모호한 한 글자 키워드는 의도적으로 제외(예: "피"→피아노·커피 오탐). 명확한 복합어만 매칭.

### 안전 등급 (UI 색상 기준)
| 점수 | 등급 | 색 |
|---|---|---|
| 90점 이상 | 안전 | 🟢 Green |
| 70 ~ 89점 | 주의 | 🟡 Yellow |
| 69점 이하 | 위험 | 🔴 Red |

기준 미달 영상은 재생 게이팅으로 차단된다(총점·위험 카테고리·비상업성 등 복합 룰).

## 6. 기술 스택 & 아키텍처

### 6.1 스택 요약
| 구분 | 기술 |
|---|---|
| 프론트엔드 | React 19, React Router v7, Tailwind CSS v4, Axios, Recharts, react-icons, react-youtube |
| 백엔드 | **FastAPI (Python)** · uvicorn (포트 3000) |
| 데이터베이스 | **Supabase (PostgreSQL)** — 전 데이터 DB 이전 완료 |
| 인증 | Supabase Auth (이메일/비밀번호, JWT) |
| AI | Anthropic Claude API — 챗봇/코치/정밀 검수 (claude-haiku-4-5) |
| 영상 데이터 | YouTube Data API v3 |
| 자막 추출 | youtube-transcript-api |
| 결제 | 토스페이먼츠(예정/설계 완료) |
| 배포 | Vercel(프론트) + Railway(백엔드) |
| 마스코트 | 로컬 PNG 포즈 + 투명 WebP 애니메이션(ffmpeg 크로마키 제작) |

### 6.2 아키텍처 개요
```
[브라우저 - React SPA (Vercel)]
        │  Axios (REST, JWT 헤더)
        ▼
[FastAPI 백엔드 (Railway)]
   ├─ 라우터: search / analyze / chat / recommend / history /
   │         profiles / favorites / badges / game-bonus /
   │         alerts / blocked-keywords / feedback / reports /
   │         me / admin(users·stats·audit) ...
   ├─ 외부 연동: YouTube Data API, youtube-transcript-api, Anthropic
   └─ DB 접근: PostgREST(httpx) 헬퍼 (service 키, 서버 전용)
        │
        ▼
[Supabase (PostgreSQL + Auth)]
   └─ 멀티테넌시: user_id 기준 행 단위 소유권 검증
```

### 6.3 주목할 설계 결정 (배경)
- **Express → FastAPI 전환**: Python 기반 AI/데이터 처리(pandas, 자막 분석)와의 친화성을 위해 백엔드를 FastAPI로 이전. (Express 원본은 `server_backup/`에 보존)
- **JSON 파일 → Supabase 전면 이전**: 초기엔 JSON 파일로 데이터를 저장했으나, 배포 환경(Railway) 재시작 시 데이터가 초기화되는 문제로 PostgreSQL(Supabase)로 전면 이전. 멀티테넌시·동시성·영속성 확보.
- **비동기 우선**: AI/외부 API 호출은 `AsyncAnthropic`·httpx 비동기로 처리해 블로킹 방지.
- **응답 호환성 유지**: 프론트 코드 수정 없이 백엔드를 교체하기 위해 API 응답 형태를 100% 동일하게 유지.

## 7. 데이터·보안·멀티테넌시

- **계정-프로필 분리**: 하나의 부모 계정 아래 여러 아이 프로필. 각 프로필은 독립된 시청 기록·시간 제한·부모 페이지(PIN)를 가진다.
- **소유권 검증**: 모든 데이터 접근은 `user_id` 기준으로 소유권을 확인해 타 계정 데이터 접근 차단.
- **권한 시스템**: 일반/프리미엄/관리자 권한을 분리. 관리자 전용 라우터는 백엔드에서 `require_admin`으로 보호.
- **감사 로그**: 관리자 행위를 audit log로 기록(운영 추적성).
- **키 보안**: Supabase service 키 등 민감 정보는 서버 전용. 프론트에는 노출 키만 사용.

## 8. 수익 모델 (설계 완료, 단계적 적용)

- **프리미엄 구독**: 무료는 하루 정밀(AI) 검수 횟수 제한 등 일부 기능 제한, 프리미엄은 제한 해제 및 고급 기능 제공.
- **결제**: 토스페이먼츠 연동 설계 완료. Supabase의 구독(subscription) 테이블로 프리미엄 상태 관리.
- **운영 비용 최적화**: 검수 결과 캐싱·계층형 필터링으로 AI 호출 비용을 구조적으로 억제 → 사용자 증가에도 한계비용을 낮게 유지.

### 10.2 용어집
- **키디(Kiddy)**: KidSafe의 마스코트 캐릭터(초록 아기 공룡). 아이 경험 전반을 안내.
- **검수(Inspection)**: 영상을 안전도 기준으로 평가하는 과정. 간이(키워드)와 정밀(AI) 두 단계.
- **Tier 0/1/2**: 검수 파이프라인의 계층. 0(공식신호·키워드)·1(채널/통계)·2(AI 자막 분석).
- **게이팅(Gating)**: 안전 기준 미달 영상의 재생을 차단하는 것.
- **멀티테넌시**: 한 계정 아래 여러 아이 프로필을 독립적으로 운영하는 구조.
- **보너스 시간**: 미니게임 완료로 얻는 추가 시청 시간(완료 시 +3분, 하루 상한 존재).
