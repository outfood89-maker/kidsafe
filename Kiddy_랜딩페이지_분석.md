# Kiddy 랜딩페이지 전략 분석

> 레퍼런스(weenote)의 설득 구조를 분해하고, Kiddy 랜딩페이지에 적용한 설계 기록.
> 목적: "스크롤을 내려가는 동안 사용자가 점점 쓰고 싶어지는" 랜딩 구조 만들기.

---

## 0. 분석의 출발점

좋은 랜딩페이지는 예쁜 게 아니라 **설득력 있는** 페이지다.
핵심 질문은 하나: **"사용자가 첫 화면부터 끝까지 내려오는 동안, 점점 더 쓰고 싶어지는가?"**

이 문서는 그 구조를 레퍼런스에서 역설계(reverse-engineering)하고,
우리(B2C 신생 서비스)의 조건에 맞게 재설계한 기록이다.

---

## 1. 레퍼런스 분석 — weenote (위노트)

위클래스 상담시스템. B2B(학교·상담교사 대상) SaaS.
랜딩페이지를 **"내용(주제) + 역할 + 답하는 질문"** 으로 분해하면 다음과 같다.

| # | 내용 (주제) | 역할 | 사용자 머릿속 질문 |
|---|---|---|---|
| 1 | 위클래스 상담시스템, 도입 1위 | 정체성 + 즉시 신뢰 | "이게 뭔데?" |
| 2 | 전국 학교 40+ 로고 | 사회적 증거 | "남들도 써?" |
| 3 | 누적 기관·선생님 숫자 | 규모 증명 | "믿을 만해?" |
| 4 | 선생님 후기 ("3일 → 3분") | 감정 공감 + 효과 체감 | "진짜 효과 있어?" |
| 5 | Before/After 96% 단축 | 핵심 가치 각인 | "얼마나 좋아져?" |
| 6 | 문제 → 해결 + UI 스크린샷 | 내 문제 인식 + 시연 | "어떻게 생겼는데?" |
| 7 | 직무별 맞춤 가치 | 개인화 | "나한테 맞아?" |
| 8 | 카드결제 1분 | 구매 장벽 제거 | "사기 복잡하지?" |
| 9 | 자문단 · 보안 · FAQ | 마지막 불안 제거 | "안전해?" |
| 10 | CTA "오늘부터 줄여보세요" | 행동 유도 | "그럼 해볼까" |

### 핵심 인사이트 — "질문-답변 사다리"

weenote의 진짜 무기는 "비교를 썼다", "후기를 넣었다" 같은 **개별 장치**가 아니다.
**사용자 머릿속에 다음으로 떠오를 질문을, 그 섹션이 미리 알고 답해준다**는 것이다.

```
"이게 뭔데?"      → 1위 상담시스템      (정체성)
"믿을 만해?"      → 학교 N개 + 숫자      (사회적 증거)
"효과 있어?"      → 후기 + 96%          (증명)
"어떻게 생겼어?"  → UI 스크린샷          (시연)
"나한테 맞아?"    → 직무별 가치          (개인화)
"사기 복잡하지?"  → 1분 결제            (장벽 제거)
"안전해?"        → 보안 + FAQ          (불안 제거)
"해볼까"         → CTA                 (행동)
```

의심이 생길 틈을 주지 않는다. 이것이 "끝까지 읽히는" 구조의 본질이다.

### weenote 설득의 두 축
- **시간 절약** (구체적 가치): "3일 → 3분", "96% 단축"
- **개인정보 보안** (최대 우려 해소): 로컬 저장 + 인증 배지

## 2. 우리(Kiddy)의 조건 — 무엇이 다른가

| 항목 | weenote (B2B) | Kiddy (B2C, 신생) |
|---|---|---|
| 대상 | 상담교사 (전문가, 논리적 구매) | 부모 (감정적 + 신중한 구매) |
| 신뢰 자산 | 도입 1위, 로고, 숫자, 후기 | **없음** (신생 서비스) |
| 구매 동기 | 업무 시간 절약 | 아이에 대한 불안 해소 |
| 핵심 가치 | 효율 (정량) | 안심 (정성) |

### 결론: 사회적 증거의 빈자리를 우리 강점 3개로 대체한다

weenote의 신뢰 자산(로고·숫자·후기)이 우리에겐 없다.
그 자리를 다음으로 채운다.

1. **공감** — 현실 인정("늦게 보여주고 싶었죠")으로 죄책감을 덜어주는 정서적 신뢰
2. **투명성** — 검수 방식을 그대로 공개("이렇게 거릅니다")해서 쌓는 신뢰
3. **캐릭터** — 마스코트 키디가 스크롤 전체를 안내. 친근함이 곧 신뢰
   - 캐릭터 기반 서비스는 리텐션·신뢰가 더 높다는 업계 데이터 존재
   - 키디 = 우리의 "일관된 신뢰 신호" (weenote의 로고 그리드 역할을 감성 버전으로)

### 포지셔닝 원칙 — "막는 도구"가 아니라 "함께하는 친구"

> 유튜브는 적이 아니다.
> "유튜브가 위험하니 이걸 써야 한다"가 아니라,
> **"이미 쓰고 있는 콘텐츠 플랫폼(유튜브) 위에서, 키디가 활약한다"** 로 간다.

유튜브엔 좋은 영상이 많다. 다만 그 안에서 고르는 수고가 있을 뿐.
키디는 그 수고를 대신 덜어주는 조력자다. (공포 마케팅 ❌ / 조력 메시지 ⭕)

## 3. Kiddy 랜딩 구조 설계

부모 머릿속 질문 순서에 맞춰 섹션의 역할을 배치한다. (최종 구현 기준 11블록)

| # | 섹션 | 역할 | 부모 질문 | 키디 |
|---|---|---|---|---|
| ① | 히어로 | 정체성 + 감성 후킹 | "이게 뭐야?" | 인사 영상 |
| ② | 공감 | 현실 인정 + 죄책감 덜기 | "내 얘기네" | 🤔 think |
| ③ | 미션 | 우리가 왜 시작했나 | "그래서 뭘 해?" | 🙌 help |
| ④ | 공생 | 막는 게 아니라 골라준다 | "막는 앱이야?" | 📖 reading |
| ④-b | 비교 표 | 유튜브 vs 키디 차이 | "유튜브랑 뭐가 달라?" | — |
| ⑤ | 검수 방식 | 안전 보장 방법(투명성) + 점수/차단 실제 캡쳐 | "진짜 어떻게 해?" | 🔍 search |
| ⑤-b | 건강한 습관 | 시간관리 + 미니게임(4박자) | "보다 끝나면?" | — |
| ⑦ | 키디 소개 | 친구이자 부모-아이 다리 | "키디가 뭔데?" | 👉 point |
| ⑧ | 기능 쇼케이스 | 실제 화면 4컷(채팅·리포트·한주·스케줄) | "어떻게 생겼어?" | — |
| ⑨ | 안심 포인트 | 마지막 불안 제거 | "믿어도 돼?" | 🛡 로고 |
| ⑩ | 최종 CTA | 행동 유도 | "해볼까" | 🎉 jump |

> ⑥ "한 걸음 더"는 ⑧과 중복되어 ⑦로 통합·삭제됨. 번호는 작업 이력상 유지.
> **카피 전문(모든 글 원문)은 별도 문서 [Kiddy_랜딩페이지_구조와_카피전문.md] 참고.**

### 설계 원칙
1. **질문-답변 사다리** 유지 — 각 섹션이 다음 질문을 미리 답한다.
2. **시각 리듬 변화** — 가운데정렬 텍스트 블록을 연속 배치하지 않는다.
   (텍스트 → 표 → 단계 카드 → 좌우 교차 캡쳐 → 4박자 플로우 → 카드 그리드)
3. **키디를 스크롤 가이드로** — 섹션마다 감정에 맞는 포즈로 등장.
4. **공포가 아닌 조력** — 유튜브를 부정하지 않는다.
5. **사실만 말한다** — 못 하는 것(유튜브 자체 광고 차단 등)은 솔직히 명시.

## 4. 적용된 개선 (변경 이력)

### ① 검수 섹션에 키디 배치
"내용을 먼저 살펴봐요" 섹션 제목 위에 **돋보기 포즈(search)** 키디 추가.
메시지를 캐릭터가 직접 보여주는 효과.

### ② 비교 섹션 신규 — "유튜브 + 키디" 표
weenote의 Before/After 비교 무기를 우리 버전으로 차용.

**초안의 문제점 (수정 전):**
- "키디 없이 혼자서" vs "키디와 함께" → 키디가 없으면 위험하다는 공포 프레이밍
- 카드 두 장, 빨강 ✕ 사용 → 유튜브를 적으로 묘사

**수정 후:**
- 헤드라인: "유튜브를 막지 않아요, 한 가지를 더해요"
- "일반적인 시청 vs 키디와 함께" **비교 표** (키디 열만 강조)
- 일반 시청 열은 중립 회색(위험 ❌ 아님, 그저 "직접 해야 함")

| 항목 | 일반적인 시청 | 🛡 키디와 함께 |
|---|---|---|
| 영상 고르기 | 직접 하나씩 확인해요 | 키디가 미리 다 살펴봐요 |
| 안전 판단 | 끝까지 봐야 알 수 있어요 | 안전도 점수로 한눈에 |
| 검색 결과 | 관련 없는 영상도 섞여요 | 위험 키워드는 자동으로 걸러요 |
| 영상을 본 뒤 | 시청으로 끝나요 | 아이 마음까지 이어드려요 |

> 하단 카피: "유튜브는 그대로, 고르는 일만 키디가 도와드려요."

### ③ 기능 쇼케이스(⑧) — 손그림 목업 → 실제 앱 캡쳐로 전환
- 직접 그린 목업은 실제 앱과 미묘하게 달라 신뢰도↓ → **실제 화면 캡쳐를 폰 베젤(PhoneShot)에 그대로** 삽입.
- 캡쳐는 회색 바깥 여백을 자동 검출·크롭하고, 모달 너비를 맞춰 **여러 폰의 줌 배율을 동일하게** 통일.
- 4컷 구성: 채팅(아이) / 분석 리포트(부모, +실천 To-Do 말꼬리표) / 키디의 한 주(부모, 프라이버시 콜아웃) / 스케줄러(부모).
- **프라이버시 콜아웃**: "아이가 '같이 보고 싶어' 고른 것만 공유, 몰래 보는 게 아니에요" — '부모가 비밀 엿본다' 오해 차단.

### ④ 건강한 습관 섹션(⑤-b) 신규 — 시간관리 + 미니게임
- 4박자 플로우(부모 시간설정 → 키디 알림 → 교육 미니게임 → 보너스/성취)를 실제 캡쳐로.
- ⚠️ **함정 회피**: "게임하면 시청시간 더!"가 "더 보게 하는 앱"으로 오해될 위험 → 하단에 *"'더 보여주는 장치'가 아니에요. 끝맺음을 배움과 성취의 기회로"* 명시.

### ⑤ ⑥⑦ 중복 정리 + 통일
- ⑥ "한 걸음 더"(⑧과 중복) 삭제, 핵심 메시지는 ⑦로 통합. 중간 CTA 버튼 정리.
- 기능 컷 칩(태그) 전부 제거, ⑨ 안심 카드를 ⑤ STEP 카드와 동일 스타일로 통일.

### ⑥ 사실 정정 — 광고 (중요)
- 초안 "광고 없이 깨끗하게"는 **거짓**(유튜브 자체 광고는 못 막음) → **"광고 영상은 걸러요 + 단, 유튜브가 트는 광고는 막을 수 없어요(프리미엄 권장)"** 로 정정.
- 우리가 막는 건 '광고가 목적인 영상', 못 막는 건 '유튜브가 송출하는 광고'. 신뢰가 생명이라 반드시 구분 명시.

### ⑦ 리브랜딩 + 다듬기
- 앱 이름 KidSafe → **Kiddy**, 투명 키디 심볼 로고 제작·전 화면 적용, 파비콘 교체.
- 섹션 간 간격 확대(주제 전환 호흡), 키디 포즈 섹션별 배치.

## 6. 핵심 요약 (한 줄)

> **"스크롤은 사용자의 질문 흐름이다.
> 각 섹션이 다음 질문을 미리 답하면, 사용자는 끝까지 내려오고, 끝에서 행동한다."**

우리의 차별점: 신생이라 사회적 증거가 없는 대신,
**공감 · 투명성 · 캐릭터(키디)** 로 신뢰를 쌓고,
유튜브를 부정하지 않고 **그 위에서 키디가 활약하는** 포지셔닝으로 간다.
