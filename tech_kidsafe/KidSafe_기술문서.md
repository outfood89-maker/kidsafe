## 1. 프로젝트 개요

- 프로젝트명: KidSafe (AI Media Guardian for Kids)
- 한 줄 소개: AI가 어린이 콘텐츠의 안전도를 실시간으로 검수하고, 연령과 관심사에 맞는 영상을 추천해주는 웹 플랫폼
- 개발 기간: 진행 중 (1인 개발)
- 목적: 프론트엔드 취업 포트폴리오

## 3. 핵심 기능 및 구현 기술

### 3-1. AI 콘텐츠 안전도 검수
- **사용 기술**: Anthropic API (claude-sonnet-4-5)
- **분석 항목**: 폭력성, 언어, 선정성, 교육성 (각 0~100점)
- **입력**: 영상 제목 + 설명 텍스트
- **출력**: totalScore + 항목별 점수 + AI 한 줄 요약
- **특징**: 단순 키워드 필터링이 아닌 LLM 기반 맥락 이해 분석

### 3-2. 연령별 콘텐츠 필터링
- **사용 기술**: safetyFilter.js (JavaScript 로직)
- **기준점수**:
  - 3세 → 90점 이상
  - 5세 → 85점 이상
  - 7세 → 80점 이상
  - 10세 → 70점 이상
- **특징**: 프로필 나이에 따라 자동으로 필터링 강도 조정

### 3-3. 나이별 가중치 점수 계산
- **사용 기술**: safetyFilter.js `calculateWeightedScore`, `sortByWeightedScore`
- **가중치 기준**:
  - 3세: 안전도 20%, 폭력/선정 50%, 언어 20%, 교육성 10%
  - 5세: 안전도 20%, 폭력/선정 40%, 언어 20%, 교육성 20%
  - 7세: 안전도 20%, 폭력/선정 30%, 언어 20%, 교육성 30%
  - 10세: 안전도 20%, 폭력/선정 20%, 언어 20%, 교육성 40%
- **특징**: 나이가 어릴수록 폭력/선정 비중 높게, 나이가 많을수록 교육성 비중 높게

### 3-4. 추천 콘텐츠 시스템 (YouTube API 실제 연동)
- **사용 기술**: YouTube Data API v3, `/search/recommend` 엔드포인트
- **나이별 추천 키워드**:
  - 3세: 동요, 뽀로로, 자장가, 율동, 타요
  - 5세: 동화, 유아 애니메이션, 키즈 캐릭터, 동물원, 인형놀이
  - 7세: 공룡, 에그박사, 곤충, 과학 실험, 키즈 다큐
  - 10세: 다큐멘터리, 과학 상식, 역사, 우주 탐험, 수학
- **특징**: 프로필 진입 시 자동으로 나이 맞춤 추천 영상 로드

### 3-5. 쇼츠/게임 콘텐츠 차단
- **사용 기술**: YouTube Data API v3 `videos.list`, ISO 8601 duration 파싱
- **차단 방식**:
  - 쇼츠 차단: `videoDuration: 'short'` + 실제 영상 길이 조회 후 60초 이하 제거
  - 게임 차단: 제목에 게임 키워드(로블록스, 마인크래프트 등) 포함 시 필터링
  - 유해 콘텐츠: `safeSearch: 'strict'` 적용
- **특징**: 동요(2~4분)는 포함하면서 쇼츠(60초 이하)만 정확히 차단

### 3-6. Anti-Bias 편식 방지 알고리즘
- **사용 기술**: safetyFilter.js `extractTags`, `applyAntiBias`, `getTopKeyword`
- **동작 방식**:
  - 영상 제목에서 키워드 자동 추출 (`extractTags`)
  - 최근 10개 시청 기록 태그 빈도 분석
  - 특정 태그 비율 70% 초과 시 해당 영상 점수 15% 감점
  - 새로운 영역 영상 +20점 보너스 부여
- **특징**: 아이가 특정 콘텐츠만 반복 시청하는 편식 현상 방지

### 3-7. 시청 기록 기반 개인화 추천
- **사용 기술**: `/search/history-recommend` 엔드포인트, `getTopKeyword`
- **동작 방식**:
  - 시청 기록에서 가장 많이 본 키워드 추출
  - 해당 키워드로 YouTube 검색
  - Anti-Bias 적용해서 편식 방지
- **UI**: "내가 좋아할 것 같아요" 섹션으로 별도 표시
- **특징**: 나이별 추천 + 개인화 추천 2개 섹션으로 구성

### 3-8. HTML 특수문자 디코딩
- **사용 기술**: search.js `decodeHtml` 함수
- **문제**: YouTube API가 반환하는 제목에 `&#39;`, `&amp;` 등 HTML 엔티티 포함
- **해결**: 백엔드에서 디코딩 후 프론트에 전달

### 3-9. 자녀 프로필 시스템
- **사용 기술**: DiceBear API, localStorage, JSON 파일 DB
- **기능**: 최대 4개 프로필 생성, 성별별 아바타 자동 생성
- **저장 데이터**: 이름, 나이, 성별, 아바타 시드, 시청 시간 제한

### 3-10. 시청 기록 저장 및 필터링
- **사용 기술**: Express REST API, history.json
- **기능**: 영상 시청 시 자동 저장, 프로필별 필터링
- **저장 데이터**: videoId, title, channelTitle, thumbnail, totalScore, violence, language, sexual, educational, summary, profileId, watchedAt

### 3-11. 배지 시스템
- **사용 기술**: Express REST API, badges.json
- **배지 종류**: 10종 (첫 발걸음, 새싹 탐험가, 시청 대장, 안전 보안관 등)
- **기능**: 조건 달성 시 자동 배지 부여 + 획득 팝업 표시

### 3-12. 부모 대시보드
- **사용 기술**: Recharts, React state
- **기능**:
  - 요약 카드 (총 시청 영상, 평균 안전도, 오늘 시청 시간)
  - 안전도 분포 막대 차트
  - 프로필별 시청 기록 탭 필터링
  - 시청 시간 제한 설정

### 3-13. AI 요약 모달 (VideoModal)
- **사용 기술**: React 컴포넌트, props 패턴
- **기능**: 영상 카드 클릭 시 항목별 점수 + AI 요약 모달 표시
- **UI**: 점수 바 (색상으로 안전도 시각화), 말풍선 스타일 요약

### 3-14. 반응형 디자인
- **사용 기술**: Tailwind CSS 반응형 클래스
- **적용**: 모바일 1열 → 태블릿 2열 → PC 3열 그리드
- **테스트**: 실제 모바일 기기에서 확인 완료

### 3-15. 공통 NavBar 컴포넌트
- **사용 기술**: React 컴포넌트, React Router useNavigate
- **기능**: 뒤로가기 버튼, 페이지 타이틀, KidSafe 로고
- **특징**: 모바일/PC 반응형, sticky top 고정

## 6. 면접 예상 질문 & 답변

**Q. AI를 어떻게 활용했나요?**
> Anthropic API를 통해 영상 제목과 설명을 입력하면 폭력성, 언어, 선정성, 교육성 4가지 항목을 분석해 0~100점으로 점수화합니다. 단순 키워드 매칭이 아닌 문맥을 이해하는 LLM의 특성을 활용했습니다.

**Q. YouTube Kids와 뭐가 다른가요?**
> YouTube Kids는 사전 검수된 라이브러리를 제공하고 왜 안전한지 설명하지 않습니다. KidSafe는 LLM이 실시간으로 영상의 맥락을 분석하고 항목별 점수와 이유를 부모에게 시각적으로 제공합니다.

**Q. 추천 시스템은 어떻게 동작하나요?**
> 4단계 파이프라인으로 동작합니다. 1단계에서 쇼츠/게임을 차단하고, 2단계에서 나이별 가중치를 적용해 점수를 계산하고, 3단계에서 편식 방지 알고리즘을 적용하고, 4단계에서 시청 기록 기반 개인화 추천을 제공합니다.

**Q. Anti-Bias 로직이 뭔가요?**
> 아이가 특정 콘텐츠만 반복 시청하는 편식 현상을 방지하기 위해 설계했습니다. 특정 태그의 누적 시청 비율이 70%를 초과하면 해당 태그 영상 점수를 15% 감점하고, 새로운 영역의 교육성 높은 영상에 +20점 보너스를 부여해 균형 있는 시청을 유도합니다.

**Q. 반응형은 어떻게 구현했나요?**
> Tailwind CSS의 반응형 클래스(md:, lg:)를 활용해 모바일 1열, 태블릿 2열, PC 3열 그리드를 구현했습니다. 실제 모바일 기기에서 직접 테스트하며 UI를 개선했습니다.
