# KidSafe UI 대개편 — 디자인 시스템 & 작업 문서

> 최초 작성: 2026-06-22 · 작업 모델: Claude Opus 4.8
> 이 문서는 UI/UX 대개편의 **단일 기준(SSoT)** 입니다. 화면 작업 시 항상 이 문서의 토큰·원칙·색 매핑을 따릅니다.

---

## 1. 개편 배경 & 목표

- **동기**: 기존 UI가 "올드/평범 + 디자인 일관성 없음 + 포트폴리오 임팩트 부족"
- **목표**: 전 화면을 **하나의 디자인 시스템**으로 통일 + 트렌디하게 + 포트폴리오 임팩트와 실사용 품질의 균형
- **방식**: 화면을 부수지 않고, **디자인 시스템(폰트·색·모서리·그림자)을 세우고 전 화면에 입힘**. 앱 실행 순서대로 진행.

### "올드함"의 5가지 원인 (진단)
1. 커스텀 폰트 없음 (시스템 기본 폰트) → ✅ 해결(Pretendard)
2. 디자인 토큰이 선언만 되고 인라인 hex 하드코딩 → 점진 마이그레이션 중
3. 모서리·그림자 값 제각각(10·12·14·16·20·36) → 토큰 스케일로 통일
4. 시각적 깊이감/트렌드 요소 부족 → 글로우·그라데·레이어드 섀도우 추가
5. 타이포 위계 약함(다 font-medium) → font-black 제목 + 위계 강화

## 2. 디자인 컨셉

### 레퍼런스
- **Netflix · Wavve** (OTT 콘텐츠 카탈로그). 사용자가 직접 제시.
- 핵심: "다크 테마"가 아니라 **"콘텐츠(영상 썸네일)가 주인공이 되는, 끝없이 탐색하게 만드는 몰입형 브라우징"**

### 확정 방향
- **레이아웃/UX**: 넷플릭스식 OTT — 풀블리드 **히어로 배너** + 가로 스크롤 **캐러셀 로우** + **호버 시 카드 확대/그림자**
- **톤**: **전체 다크** (랜딩·로그인·프로필·부모·아이홈 모두). 아이홈도 다크 확정.
- **색**: 전부 **키디 캐릭터(에메랄드 공룡)에서 추출**. 올리브 그린(#6DAB60) 폐기.
- **캐릭터**: 키디를 **큐레이터**로 (히어로·말풍선·로우 안내)

### 왜 다크인가
밝은(흰) 배경은 알록달록 유튜브 썸네일과 채도 싸움 → 산만.
다크는 모든 썸네일을 중립적으로 받쳐 "액자 속 그림"처럼 정돈. (OTT가 다크인 이유)
사용자가 다크 vs 밝은 직접 비교 후 다크 선택.

## 3. 디자인 시스템 토큰

> 정의 위치: `client/src/index.css` 의 `@theme` 블록. Tailwind 클래스로 사용(`bg-ks-base`, `text-ks-emerald`, `rounded-ks-lg`, `shadow-ks-card` 등).

### 색 (키디에서 추출)
| 토큰 | HEX | 용도 |
|---|---|---|
| `ks-emerald` | `#18C49A` | 메인 (버튼·강조) |
| `ks-teal` | `#14B8C4` | 청록 (그라데 끝) |
| `ks-spring` | `#16BEAF` | 중간값 (단색 필요 시) |
| `ks-accent-gold` | `#F5B829` | 별/포인트 |
| `ks-accent-coral` | `#F2655C` | 경고/하트 |
| `ks-base` | `#0B1F1B` | 최하단 배경 (딥 틸-그린, 순검정 아님) |
| `ks-surface` | `#0F2A24` | 카드/본문 영역 |
| `ks-surface-2` | `#16352E` | 칩·보더·올라온 면 |
| `ks-ink` | `#EAF5F1` | 본문 텍스트 |
| `ks-ink-sub` | `#8FA89F` | 보조 텍스트 |

**그라데이션 표준**: `linear-gradient(135deg, #18C49A, #14B8C4)` (버튼·로고·활성칩)
**히어로 배경 표준**: `radial-gradient(120% 90% at 50% 0%, #123129 0%, #0B1F1B 55%, #08160F 100%)`
**제목 그라데 글자**: `linear-gradient(110deg, #2BE0B4, #18C49A 45%, #14B8C4)` + `WebkitBackgroundClip: text`

### 폰트
- **Pretendard** (CDN: jsdelivr, `index.html`에 로드 + `index.css` body/`--font-sans`에 적용)
- 위계: 제목 `font-black` + `tracking-tight`, 강조 `font-bold/extrabold`, 본문 `font-medium`

### 모서리 스케일
`rounded-ks-sm`(10) · `rounded-ks-md`(14) · `rounded-ks-lg`(18) · `rounded-ks-xl`(24) · `rounded-ks-pill`(999)

### 그림자 스케일
`shadow-ks-card`(0 8px 22px /.35) · `shadow-ks-pop`(0 16px 40px /.45) · `shadow-ks-glow`(0 12px 36px teal/.30)

## 4. 디자인 원칙 (5가지)

1. **다크 우월**: 베이스는 항상 다크(`ks-base`/`ks-surface`). 섹션은 딥(#0B1F1B) ↔ surface(#0F2A24) 번갈아 깔아 단조로움 방지.
2. **색 레이어 분리**: 브랜드색(에메랄드/청록)은 **UI 크롬(버튼·칩·텍스트)에만**. 썸네일은 콘텐츠 레이어로 분리 → 알록달록 썸네일과 안 싸움.
3. **타이포 위계**: 제목은 크고 굵게(`font-black` + `tracking-tight`). 핵심 문구는 그라데 글자.
4. **깊이감**: 글로우(blur 원) + 레이어드 섀도우 + 그라데로 평평함 제거.
5. **키디 큐레이터**: 키디가 화면을 안내(히어로·말풍선). 단 어색한 멘트는 제거(예: 랜딩 말풍선).

### 썸네일 안전점수 뱃지 — "다크 글래스" (B안 확정)
`background: rgba(0,0,0,0.72)` + 흰 글씨 + 작은 브랜드 색점.
→ 어떤 색 썸네일 위에서도 안정적. (단색 에메랄드 뱃지는 ✗ — 초록/빨강 썸네일과 충돌)

## 9. 작업 원칙 (필수)

- **기존 코드 삭제 금지** — 비활성화는 주석처리(복구 가능). 레거시 토큰도 보존.
- **화면 단위로 진행** + 각 화면 변경 전/후 사용자 확인.
- **목업(아이폰 미리보기)은 해당 실제 화면 완성 후 동기화** — 두 번 작업 방지.
- 모델: 토대·핵심 화면은 Opus, 단순 반복 적용은 Sonnet 가능.
- 커밋/푸시는 사용자가 요청할 때만.
